# P6 Context

> P6 = 單人電子卡牌戰術遊戲,以「一擊必殺(OHK, One-Hit Kill)」為核心美學。
> 玩家用卡牌作為資源語言,在不完整資訊下對單一 commit 做出戰術規劃。
>
> 本檔給 AI agent 與新接手者:**領域語言 + 概念地圖**。機制細節請進 `docs/`,以 `docs/A__Index_and_meta/文件目錄.md` 為入口。

---

## 領域術語表

### 戰鬥核心

| 詞 | 意思 | 不要叫成 |
|---|---|---|
| **本擊**(Strike) | 玩家在單場戰鬥中,從手上挑出並**一次性提交**的卡牌組合。設計上的最小單位,不是個別卡。 | ❌ 回合 / turn / attack |
| **OHK** | One-Hit Kill。每場戰鬥結算只有兩個結果:OHK 或失敗。沒有 HP 條、沒有血量遞減。 | ❌ kill / damage |
| **Underkill** | 本擊未達敵人任何需求 → 結算失敗。 | ❌ miss / fail |
| **commit moment** | 玩家提交本擊的瞬間,所有效果同時結算。 | ❌ submit / turn end |

### 三子階段(Place / Lock / 結束本擊)

| 詞 | 意思 |
|---|---|
| **Place** | 玩家拖卡到本擊區,可隨時編輯,系統不結算、不消耗資源。 |
| **Lock** | 玩家對單張卡點「鎖定」;`required` 卡必須 lock 才能結束本擊;lock 不可撤回。 |
| **lock_class** | 卡牌欄位,3 值:`none` / `optional` / `required`,決定 lock 行為。 |
| **結束本擊** | 玩家點按鈕,系統鎖定剩餘卡 → 計算類型次數總和 → 比對需求 → 判定結果。 |

### 結算模型

| 詞 | 意思 |
|---|---|
| **類型計數模型** | P6 結算採此模型:本擊內每種類型(衝擊/穿刺/燃燒/...)的卡片**次數**加總,任一條件達標即 OHK。 |
| **需求表** | 每個敵人定義「各類型需要多少次數」+「混合任意需求」。 |
| **混合任意需求** | 不限類型只比總次數;= 該敵人最高類型需求 × 1.5 取整(2026-05-13 定案)。 |
| **本擊上限** | 防止 dump 全資源策略;普通敵人 = 最低需求 ×2,精英 ×2.5,BOSS ×3+5。 |
| **flexible** | 部分卡的可分配資源(如「+2 任意」),由玩家主動分配到具體類型。 |
| **contribution** | 卡牌貢獻到本擊類型計數的數值。 |

### 敵人系統

| 詞 | 意思 |
|---|---|
| **敵人四類** | 兔子 / 狼 / 豹 / 熊。視覺直覺直接傳達「失敗會怎樣」。 |
| **失敗代價** | 兔子=卡牌消耗+逃跑;狼=變精英回歸;豹/熊=GAME OVER。 |
| **精英化** | 狼型敵人失敗後變精英回歸,需求更高。**不是隨機觸發**,是設計合約。 |
| **四層資訊架構** | 層1模板永久顯性 / 層2視覺線索 / 層3即時狀態 / 層4隱藏狀態(僅情報卡可揭)。 |
| **戰鬥狀態** | 先攻 / 標準 / 被襲;每場戰鬥開始時的處境本身是戰術變數。 |
| **猜弱點** | 玩家不投資情報,純靠視覺線索判斷弱點 → 精算成本。 |

### 卡牌系統

| 詞 | 意思 |
|---|---|
| **兩軸正交分類** | 資源類別(工具/爆發/角色)× 功能類別(戰鬥/情報/複合)獨立填寫。 |
| **資源池卡 vs 個體卡** | 池卡 = 同類多張共享;個體卡 = 單張獨立效果。 |
| **一張=一次使用** | 2026-05-14 取消 `use_limit` 欄位;同一張卡用完即消耗。 |
| **special_effect** | 卡牌的特殊效果(揭露 / 試探 / 控制 / 等)。 |
| **教學重來** | 序章前半第一連戰專用:卡組+狀態回滾,對話衰減,無上限。 |

### 連戰 / 戰役 / 主冒險

| 詞 | 意思 |
|---|---|
| **本擊 → 戰鬥 → 連戰 → 戰役 → 主冒險** | 5 層遞進結構。 |
| **連戰** | 一條線串接的多場戰鬥;中間有整備期。 |
| **戰役** | 約 30 分鐘(獵人模式)或 8-15 分鐘(騎兵模式);由多場連戰組成。 |
| **主冒險** | 3-6 場故事連續的戰役;1-2 小時總時長。 |
| **Hub** | 戰役間的中樞場景,作為敘事連結點。 |
| **整備期** | 連戰之間的資源回補階段;序章採劇情驅動固定補給(非玩家選擇)。 |
| **追擊** | 連戰中的特殊狀態,影響下一場戰鬥起手條件。 |

### 戰後系統

| 詞 | 意思 |
|---|---|
| **戰後校準對話** | 取代「數值結算畫面」的呈現層;玩家+NPC 雙聲道對話 + 敵方反應。 |
| **冒險手記** | 戰鬥紀錄系統的呈現載體;雙頁書頁;玩家行動造圖。 |

### 模式

| 詞 | 意思 |
|---|---|
| **獵人模式** | 慢節奏、深戰術(每場 30 分鐘)。 |
| **騎兵模式** | 快節奏、輕戰術(每場 8-15 分鐘)。 |

## 核心概念地圖

```
主冒險 (1-2h)
 └─ 戰役 (30min 獵人 / 8-15min 騎兵)
     └─ 連戰 (4-6 場戰鬥串接)
         ├─ 整備期 (回補資源)
         └─ 戰鬥 (3-5min)
             └─ 本擊 (玩家的單一 commit)
                 ├─ Place (排卡,可改)
                 ├─ Lock (鎖定,不可撤回)
                 └─ 結束本擊 → 類型計數結算 → OHK / Underkill
```

---

## 設計上必須拒絕的(裁判依據)

P6 **不是**:
- ❌ 牌組構築(沒有抽牌、沒有 Deck Building)
- ❌ 位置戰術(沒有棋盤、沒有移動)
- ❌ RPG(沒有等級、裝備、技能樹)
- ❌ Roguelite Deckbuilder(沒有 HP、沒有抽牌)

提案任何機制 / 卡 / 敵人前,先檢查是否違反上述任一條。
