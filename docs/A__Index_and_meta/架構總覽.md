## 二、層 ① 場景流

6 個 scenes,分玩家流程 + dev。

| 場景 | 入口 | 出口 |
|---|---|---|
| **[title_screen](../../scenes/title_screen.tscn)** | `project.godot` main_scene(啟動點) | 任意輸入 → main_menu |
| **[main_menu](../../scenes/main_menu.tscn)** | ← title | new game / continue → hub;quit |
| **[hub](../../scenes/hub.tscn)** | ← main_menu / ← m2 完成 | 點書架 → m2_campaign;DevPanel → main_menu;點日誌 → adventure_journal **overlay** |
| **[m2_campaign](../../scenes/m2_campaign.tscn)** | ← hub(EnterButton)| 戰役完成 / 接受失敗 → hub;`RecordButton` → adventure_journal overlay |
| **[adventure_journal](../../scenes/adventure_journal.tscn)** | overlay(從 hub 或 m2)| ESC / close → `queue_free` 回呼叫者 |
| **[m1_battle](../../scenes/m1_battle.tscn)** | F5/F6 dev only | 三大 dev 用途:戰鬥機制驗證 / 新卡片驗證 / 新敵人驗證 |

**關鍵**:`adventure_journal` 是 **overlay**(`add_child` 進父場景,close 時 `queue_free`),所以從 m2 戰役中開手記**不會打斷戰役 state**。

## 三、層 ② 跨場景持久層

| Autoload | 職責 | 持久化路徑 | 內容 |
|---|---|---|---|
| **ResourceLibrary** | 靜態資源目錄(read-only) | 無 | 啟動時載入,之後唯讀 |
| **GameState** | 玩家進度 + 紀念品解鎖 | `user://progress.cfg` | `completed_campaigns: Array` / `unlocked_souvenirs: Array` |
| **SaveSystem** | 戰役自動存檔(整備節點) | `user://savegame.tres` | campaign_id / chain_index_completed / deck / supply_applied_history |
| **AdventureRecord** | 冒險手記 append-only log | `user://battle_records.json` v2 | battles[] + prep_nodes[] + 3 indices |
| **CampaignContext** | 預留 | — | 目前空殼,M5-S3+ 會用 |

**ResourceLibrary 規則(ADR-0001)**:所有靜態 `.tres` 從 `ResourceLibrary.<type>(id)` 取;新增資源 = 加一行 id → 路徑;**其他模組從不直接 `load("res://resources/...")`**。

**AdventureRecord v2 機制(ADR-0003)**:啟動先比 disk version,不符 → 自動清檔 + push_warning。未來 schema 變動 bump `VERSION` 即可,不必手動 `rm`。

## 五、層 ④ 戰鬥執行層

純邏輯模組,**不碰任何 Node / UI**,可被任何 host 驅動。

```
┌─────────────────────────────────────────────────────┐
│  BattleEngine (RefCounted)                          │
│  ─────────────────────────                          │
│  state: phase / strike / enemy / deck / library     │
│         result: StrikeResult                        │
│         revealed_info: Dictionary                   │
│                                                     │
│  API:                                               │
│    place_card(id) / unplace_card_at(i)             │
│    lock_card_at(i)                                  │
│    can_commit() / commit_strike() → StrikeResult    │
│    available_count(id) → int     ◀── ADR-0002 §C2  │
└────────────┬────────────────────────────────────────┘
             │ owns
             ▼
       ┌──────────────┐         ┌─────────────────┐
       │  Strike      │         │  Resolution     │
       │  ──────      │  ←──    │  ──────         │
       │  placed_cards│  resolve│  static fn 計算  │
       │  locked_cards│         │  比對 vs 敵人    │
       │  get_type_   │         │  需求 → 回傳     │
       │  counts()    │         │  StrikeResult    │
       └──────────────┘         └─────────────────┘
                                          │ produces
                                          ▼
                                ┌─────────────────┐
                                │  StrikeResult   │  ◀── ADR-0003
                                │  (Resource)     │
                                │  6 個 @export   │
                                │  欄位 + to/from │
                                │  dict           │
                                └─────────────────┘
```

**契約**:
- BattleEngine **不知道** 自己在 m1 / m2 / 未來 replay 哪裡跑;只接 `(enemy_instance, enemy_template, deck, card_library)` 四個輸入
- Strike 是 BattleEngine 內部,不對外開放修改(只能透過 engine 的 place/lock method)
- Resolution 是 **pure static function**,可單獨測試
- StrikeResult 跨 engine ↔ view ↔ record 三層共用同型別

## 六、層 ⑤ 結算分類層

結算結果出來後,**五個 helper class** 各司其職:

| Class | 職責 | 純度 |
|---|---|---|
| `CalibrationClassifier` | 把 StrikeResult + weakness_range + total_committed 分類成 6 種對話狀態(精準擊殺 / 大幅 overkill / 勉強過 / 差一步類型錯 / 差一步量不足 / 大幅不足) | static,純邏輯 |
| `FailureHandler` | 失敗時依 enemy_class 決定處置(教學重來 / 兔子連鎖逃 / 狐狸獨逃 / 狼菁英化 / GAME OVER) | static,純邏輯 |
| `SupplyHandler` | 整備時把 SupplyPhase 套用到 deck(`full_restore` / `add_cards`) | static,純邏輯 |
| `TutorialRetry` | 取對話衰減 tier(encouraging / neutral / sarcastic)依 retry_count | static,純邏輯 |
| `DeckManager` | 卡組工具(snapshot / restore / add_cards / total_count / find_entry / build_starting_deck / restore_to_starting) | static,純邏輯 |

**全部 static + RefCounted**,完全沒有 UI / autoload 副作用,可單獨單元測試。

## 七、層 ⑥ 戰鬥-zone UI 層(BattleView)

ADR-0002 鎖定的 seam。**純 script controller,沒有自己的 .tscn**。

```
host 場景 (m1 或 m2)
  ├── @onready var pile_slot, strike_slot, status_content,
  │       enemy_figure, enemy_info_slot, end_strike_button,
  │       advance_button (m2 only), progress_slot (m2 only),
  │       strike_info_label
  │
  ├── _ready():
  │     view = BattleView.new()
  │     view.attach({ ...node refs... })          ◀── 傳入 host 已 lay 好的 nodes
  │     view.set_preview_source(callable)         ◀── m2 註冊;m1 不註冊
  │     view.strike_committed.connect(_on_...)
  │     view.advance_pressed.connect(_on_...)    (m2 only)
  │
  └── 切換戰鬥時:
        engine = BattleEngine.new(...)
        view.set_engine(engine)                   ◀── view 重新渲染
        view.set_enemy_display(label, elite, n)   (m2 only)
```

view 內部 **吞掉**:
- pile 點擊 → `engine.place_card` → refresh
- unplace / lock 按鈕
- end_strike 按鈕 → `engine.commit_strike` → emit `strike_committed(result)`
- enemy_figure 點擊 → 情報欄展開/收合
- progress_slot 點擊 → 連戰預覽切換(`preview_source.is_valid()` gate)

view **對外只 emit 2 signal**:`strike_committed(result: StrikeResult)` + `advance_pressed`。

## 八、層 ⑦ Widget 庫

`scripts/ui/widgets/` 下的純視覺元件,各自 `class_name X extends PanelContainer / HBoxContainer / Control`,風格無關佔位版(美術 pass 後替換內部視覺,介面不動):

| Widget | 用途 |
|---|---|
| `CardWidget` | 卡牌色塊(本擊區用),可掛動作按鈕 |

| `PortraitPair` | 雙頭像(誰在說話)|

| `SupplyChip` | 整備卡塊(卡名 + ±N 類型色邊框) |
| `UiPalette` | 配色 + 樣式工廠 + 顯示字串集中地(static)|

## 十、一條完整 trace:玩家在 m2 點「結束本擊」

```
1. user 點 EndStrikeButton
   └─→ BattleView._on_end_strike_button_pressed()
       ├─→ engine.commit_strike()  ── BattleEngine
       │   ├─→ 對 placed 中的 optional 卡觸發 spec_unlocked
       │   ├─→ strike.finalize_lock_all()  ── Strike
       │   ├─→ 消耗卡組 (each entry.consume(1))
       │   ├─→ result = Resolution.resolve(strike, enemy_instance)  → StrikeResult
       │   └─→ phase = RESOLVED
       │
       ├─→ view._refresh_all()
       │   ├─→ enemy_widget.set_revealed(engine.revealed_info)
       │   ├─→ _refresh_piles  (engine.available_count)
       │   ├─→ _refresh_strike  (lock 區 + place 區)
       │   ├─→ _refresh_status (RESOLVED 分支:通用 OHK/Underkill verdict + RequirementBar)
       │   └─→ _refresh_buttons (end_strike.disabled = !can_commit)
       │
       └─→ strike_committed.emit(result)
           │
           ▼  ── host (m2) 接手 ─────────────────────────────
2. m2._on_strike_committed(result: StrikeResult)
   ├─→ cal_state = CalibrationClassifier.classify(result, weakness_range, total_committed)
   ├─→ last_calibration_text = _build_calibration_text(cal_state)
   │
   ├─→ if result.ohk:
   │     ├─→ _record_battle(entry, result, cal_state, -1)
   │     │   ├─→ rec = BattleRecord.new()
   │     │   ├─→ rec.result = result  ← typed StrikeResult 直接賦值
   │     │   ├─→ rec.(metadata + enemy + revealed + strike_cards + cal + timestamp)
   │     │   └─→ AdventureRecord.save_battle(rec)
   │     │       ├─→ battles.append(rec)
   │     │       ├─→ _update_indices_for_battle (by_enemy / last_failure / by_campaign)
   │     │       └─→ _save_to_disk()  → JSON file
   │     ├─→ enemy_queue.pop_front()
   │     ├─→ chain_defeated_count += 1
   │     └─→ _after_battle_result(true, entry, "")
   │         ├─→ status_content 重寫:✓ 擊敗 X + RequirementBar
   │         ├─→ portrait_pair.set_speaking("npc")
   │         ├─→ dialogue_bubble.show_line(校準文字)
   │         ├─→ advance_button.disabled = false
   │         ├─→ view.set_advance_label("繼續" / "結束連戰")
   │         └─→ _refresh_chain_sequence (progress_indicator 劃掉剛擊敗的)
   │
   └─→ else:  // Underkill
         outcome = FailureHandler.resolve_failure(template, instance, is_elite, tutorial_retry)
         _record_battle(entry, result, cal_state, outcome.outcome)
         _handle_failure(outcome, entry)
           ├─→ TUTORIAL_RETRY  → _trigger_tutorial_retry
           ├─→ RABBIT_CHAIN_FLEE  → 過濾 queue
           ├─→ FOX_FLEE  → pop_front
           ├─→ WOLF_ELITE_PROMOTION  → pop + 末尾追加 elite
           └─→ GAME_OVER  → _trigger_game_over

3. user 點 Continue
   └─→ view.advance_pressed (signal)
       └─→ m2._on_advance_pressed()
           ├─→ enemy_queue.is_empty()?  → _complete_chain()  → POST_CHAIN phase
           └─→ otherwise → _start_next_enemy()  ── 新 engine,view.set_engine, ...
```

## 十一、ADR 鎖定的契約

| ADR | 鎖定什麼 | 對寫程式的意義 |
|---|---|---|
| **[ADR-0001](adr/0001-resource-library-autoload.md)** | ResourceLibrary 採 autoload(非 DI) | 所有靜態 `.tres` 從 `ResourceLibrary.<type>(id)` 取;測試時 fixture override autoload |
| **[ADR-0002](adr/0002-battle-view-script-controller.md)** | BattleView 採純 script controller(非 self-contained scene) | 新加戰鬥 host 場景 = `.tscn` lay widgets + script attach + 兩個 signal handler |
| **[ADR-0003](adr/0003-strike-result-embedded-in-battle-record.md)** | StrikeResult 內嵌進 BattleRecord(Route B nested) | 結算所有路徑都吃 typed StrikeResult;schema 變動 bump `AdventureRecord.VERSION` 即可自動 migration |

**反向決定(已決定但未寫 ADR)**:不採用 EventBus(2026-05-16,5 emit / 0 listener 觀察後刪)。跨模組通訊用 **explicit signal**(`view.strike_committed` 為範本)。
