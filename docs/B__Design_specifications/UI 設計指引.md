## 一、UI 設計原則

P6 的 UI 設計直接服務於遊戲設計概念,以下原則必須在每個畫面、每個元件落實:

### 原則一:系統可整理資訊,但不做判斷

對應 [遊戲核心系統機制.md](../C__Implementation_benchmarks/遊戲核心系統機制.md) §13 的 A 類 vs B 類 區分:

| 允許(A 類) | 禁止(B 類) |
|----------|----------|
| 類型篩選按鈕(玩家選類型,UI 過濾) | UI 自動推薦「這個敵人用這張卡」 |
| 玩家自定義釘選 | UI 自動把弱點對應卡推到前列(玩家未揭露時) |
| 緊湊 / 展開切換 | UI 顯示「最佳本擊組合」 |

**檢驗法**:任何 UI 行為若依賴「**系統知道但玩家不知道的資訊**」進行視覺強調,就是 B 類,**必須拒絕**。

### 原則二:緊湊呈現 + 點擊展開

降低初始視覺密度——卡牌、敵人、紀錄等的預設呈現要簡潔,玩家點擊才展開詳細資訊。

### 原則三:雙模式適配(獵人 / 騎兵)

每個 UI 元件都需要考慮「**獵人模式下如何呈現** vs **騎兵模式下如何呈現**」。具體規格見 §5。

### 原則四:呈現層原則的視覺實踐

對應 [遊戲核心系統機制.md](../C__Implementation_benchmarks/遊戲核心系統機制.md) §12:
- 顯示「+2 衝擊」而非「+15% 攻擊力」
- 顯示「中等強度的目標」而非「HP: 47/50」
- 顯示「需求 bar 填充進度」而非「目前傷害總和 = 5」

### 原則五:風格無關性

本文件規範**佈局、結構、互動**,不規範**顏色、字體、圖示風格**。後者待美術風格決定後再做。

## 二、螢幕地圖

P6 第一期需要實作的所有畫面:

### 主流程畫面

| 畫面 | 用途 | 關鍵元件 |
|------|------|--------|
| **啟動畫面** | Logo + 點擊進入 | Logo、開始按鈕、設置入口 |
| **主選單** | 新遊戲 / 繼續 / 設置 / 結束 | 選單按鈕列表 |
| **Hub(中樞)** | 戰役選擇 + NPC 對話 + 紀念品展示 | 戰役選擇器、NPC 對話泡、紀念品展示位 |
| **戰役選擇** | Hub 內子畫面 / 戰役清單與選擇 | 戰役卡片、難度標示(若有)、開始按鈕 |
| **戰役開場劇情** | 劇情段落播放 | 對話 / 文字呈現器、跳過按鈕(重玩時) |
| **戰役末尾劇情** | 結局劇情 | 同上 |
| **整備節點** | 卡牌回補 + 校準對話 + 小劇情 + 暫停退出 | 卡組狀態檢視、繼續按鈕、暫停退出按鈕 |
| **戰鬥介面** | Place / Lock / 結束本擊 | 卡牌庫存區、本擊區、敵人 widget、結束按鈕 |
| **結算演出** | 需求 bar + 敵方反應 | 需求 bar、擊殺動畫區、結算結果 |

### 輔助畫面

| 畫面 | 用途 |
|------|------|
| **戰鬥紀錄查閱** | 玩家主動查 / 精英敵人對戰前自動彈出 |
| **紀念品展示** | Hub 內子畫面,展示已解鎖紀念品 |
| **設置介面** | 動畫速度 / 教學 UI / 音量 / 雙模式 |
| **暫停選單** | 整備節點時可用,暫停退出 |
| **GAME OVER 選擇** | 重新挑戰 vs 接受失敗 |
| **存檔讀檔** | 第一期單存檔,可省略複雜介面 |

### 畫面流程圖

```
[啟動] → [主選單] ─── [新遊戲] ───→ [Hub(初始)] 
                  └── [繼續]     ───→ [Hub(已進)] ──┐
                  └── [設置]                          │
                                                     ↓
   ┌──────────── [Hub] ←─── [戰役末劇情] ←─ [最終連戰]
   │                ↓                          ↑
   │     [戰役選擇 / NPC 對話 / 紀念品]        │
   │                ↓                          │
   │      [戰役開場劇情] ────────→ [連戰 1]
   │                                  ↓
   │                          [整備節點 1]
   │                                  ↓
   │                              [連戰 2]
   │                                  ↓
   │                          [整備節點 2]
   │                                  ↓
   │                              [連戰 3]
   │                                  ↓
   │            (失敗時:[GAME OVER] → 重新挑戰 / 回 Hub)
   │
   └─ [紀念品展示 / 戰鬥紀錄] ← Hub 子畫面
```

## 三、元件清單

每個元件規範:**用途 / 狀態 / 互動 / 風格無關規格**。

### 3.1 卡牌 widget

**用途**:呈現單張卡的所有玩家可見資訊。

**狀態**:
- 預設(緊湊):縮起的卡面
- 展開:顯示完整描述、特殊效果
- 在庫存:可拖曳到本擊區
- 在本擊區(Place):可拖回庫存、可點 Lock 按鈕
- 在本擊區(Lock):**不可移動**(視覺定格)
- 灰階(教學期):不被建議使用
- 高亮(教學期):被建議優先嘗試

**互動**:
- 點擊:展開 / 收起
- 拖曳:從庫存到本擊區 / 從本擊區到庫存(Lock 前)
- 點 Lock 按鈕:鎖定該卡

**📚 舊專案參考**:`old_game_plan/scripts/Ui/CardView.cs`(198 行)提供卡牌 widget 完整結構參考——PanelContainer + StyleBoxFlat 邊框、類型 band、名稱顯示、`_GetDragData` 拖曳啟動、`_GuiInput` 點擊偵測。**P6 屬性欄位不同**(無 Cost,改用 lock_class / contribution 等),但結構直接可借鏡。

**結構欄位**(全部來自卡牌資料層):
- 名稱
- 資源類別圖示(工具 / 爆發 / 角色)
- 功能類別圖示(戰鬥 / 情報 / 複合)
- 弱點類型圖示(衝擊 / 穿刺 / 燃燒 / 通用 / 無)
- 貢獻文字(「+2 衝擊」或「+2 任一」)
- 強度(★ / ★★ / ★★★)
- 使用次數點數(●●○)
- lock_class 標示(無 / 雙模式 / 🔒)
- 特殊效果 display_text(緊湊只顯示關鍵字)

### 3.3 本擊區 / 庫存區

**本擊區雙子區設計**:

```
┌─────────────────────────────────────────┐
│  本擊區                                  │
│  ┌─────────────┐  ┌──────────────────┐ │
│  │ Place(可動)│  │ Lock(已鎖定,定格)│ │
│  │ [卡 A]      │  │ [卡 X]            │ │
│  │ [卡 B]      │  │                    │ │
│  └─────────────┘  └──────────────────┘ │
│                                          │
│  即時本擊狀態:                            │
│  衝擊 +X / 穿刺 +Y / 燃燒 +Z            │
│                                          │
│  [結束本擊] (若有 required 卡未 lock 則禁用) │
└─────────────────────────────────────────┘
```

**互動**:
- 拖曳卡進入 Place 子區
- 點 Lock 按鈕 → 該卡進入 Lock 子區
- Lock 卡視覺上「定格」(灰底、邊框加粗、不可拖動)

**📚 舊專案參考**:`old_game_plan/scripts/Ui/PlayCardZone.cs`(200 行)提供 drop zone 完整實現——`_CanDropData`(驗證 + hover 視覺反饋)、`_DropData`(接受邏輯 + 觸發效果),拖放 Dictionary 格式如下:
```gdscript
{
  "kind": "action_card",
  "card_id": "...",
  "card_name": "...",
  "card_type": "...",
}
```
P6 可擴充 Dictionary 加入 `lock_class` 等欄位作為 drop 驗證依據。

### 3.4 庫存區

**用途**:呈現玩家當前可用的所有卡。

**結構**:
- 按類別分區(資源池 / 個體卡)
- 類型篩選按鈕(衝擊 / 穿刺 / 燃燒 / 通用 / 情報)
- 緊湊 / 展開切換
- 釘選功能(老手用)

**資源池卡呈現**:堆疊式縮略圖 + 剩餘次數(例如「直擊 ●●●●●●●●● 9 次」)。

**📚 舊專案參考**:`old_game_plan/scripts/Ui/HandDock.cs`(114 行)提供 Hand 容器結構——HBoxContainer + 動態 spawn 子節點(`OnHandCardsChanged(cardIds, cardNames, cardTypes, cardCosts)`)。**P6 移除 deck/discard 計數**,庫存區改顯示類型篩選 + 緊湊/展開切換。

### 3.5 敵人 widget

**用途**:呈現當前敵人的四層資訊。

**結構**:

```
┌──────────────────────────┐
│  [敵人視覺立繪]            │
│                            │
│  名稱:[敵人模板名]         │
│  類別:[兔子/狼/豹/熊圖示]   │
│  戰鬥狀態:[先攻/標準/被襲]  │
│                            │
│  弱點範圍:[類型圖示群]      │
│  需求層級:[低/中/高]        │
│                            │
│  [若已揭露需求表] 需求:    │
│   衝擊 X / 穿刺 Y / 燃燒 Z │
│                            │
│  [若為精英化] ⚠ 精英化     │
│  [戰鬥紀錄查閱按鈕]         │
└──────────────────────────┘
```

**互動**:
- 點擊立繪 → 展開層 2 視覺線索的「解讀提示」(教學期)
- 點擊精英化標記 → 開啟戰鬥紀錄查閱

### 3.6 需求 bar

**用途**:結算演出時視覺化「填滿哪個需求」。

**結構**:每個類型一個 bar,顯示玩家本擊貢獻次數 vs 敵人需求。

```
衝擊  [▓▓▓▓▓░░░░░] 5/8  ← 未達
穿刺  [▓▓░░░░░░░░] 2/5  ← 未達
燃燒  [▓▓▓▓▓▓▓▓▓▓] 10/6 ← 達標!
混合  [▓▓▓▓▓▓░░░░] 8/12  ← 未達
```

**動畫**:結算時 bar 逐漸填充(0.3-1 秒,可調速),逼近達標線時稍頓,然後填滿 / 不滿。

### 3.7 flexible 分配介面

**觸發**:玩家對 flexible 卡點 Lock 時跳出。

```
┌──────────────────────────────┐
│  分配 flexible:+2 任一        │
│                                │
│  衝擊 [-] [○ ●] [+]            │
│  穿刺 [-] [● ○] [+]            │
│  燃燒 [-] [○ ○] [+]            │
│                                │
│  總計:2 / 2 ✓                 │
│                                │
│  [確認 Lock]                  │
└──────────────────────────────┘
```

**規則**:總和必須 = N(本卡的 flexible 數)才能確認。

### 3.8 對話框

**兩種形態**:

**Modal 對話**(用於失敗校準、重要劇情):
- 全螢幕或半透明背景遮罩
- 玩家必須點擊 / 按鍵推進
- 不能繞過

**異步飄過字幕**(用於 OHK 校準、Hub NPC、道中對話):
- 螢幕底部或側邊顯示
- **不暫停遊戲**
- 自動淡入 → 持續 → 淡出
- 玩家可選忽略

詳見 §7。

### 3.9 教學提示元件

**用途**:教學期顯示「該怎麼做」。

**呈現**:
- 高亮元件(亮邊框)
- 旁邊浮出文字提示(短句)
- 箭頭指示(若需要)
- 教學期過後**全部關閉**

### 3.10 進度指示器

**用途**:讓玩家知道在連戰 / 戰役 / 主冒險中的哪個位置。

```
連戰 2: [✓] [✓] [▶ 當前] [ ] [ ]
戰役進度: 連戰 2 / 3
```

### 3.11 NPC 對話泡(Hub 內)

**用途**:Hub 中 NPC 跟玩家對話。

**結構**:
- NPC 立繪 + 表情
- 對話泡 / 文字框
- 「下一句」按鈕(若多句)
- 不需要選項分支(P6 無分支對話)

### 3.12 紀念品展示位

**用途**:Hub 中已解鎖紀念品的展示。

**關鍵設計**:**只顯示已解鎖**,**不顯示未解鎖位**(對應第一原則)。

```
┌────────────────────────────────┐
│  紀念品                        │
│                                │
│  [明信片 A] [小物件 B] [...]   │
│                                │
│  (沒有「???」未解鎖位)         │
└────────────────────────────────┘
```

點擊紀念品 → 展開描述 / 大圖。

## 四、互動規範

### 主要輸入裝置

- **滑鼠**:主要互動(拖曳、點擊)
- **鍵盤**:快捷鍵(可選但建議)
- **手把**:**第一期不支援**,但 UI 設計時預留(避免使用 mouse-only 互動)

### 互動模式

| 操作 | 動作 | 對應 |
|------|------|------|
| 拖曳 | 滑鼠左鍵長按移動 | Place 卡到本擊區 |
| 點擊 | 滑鼠左鍵單擊 | Lock 卡、推進對話、選擇選單 |
| 雙擊 / 右鍵 | 滑鼠雙擊 / 右鍵單擊 | 卡牌詳細展開 |
| 長按 | 滑鼠左鍵長按 | 動畫加速 / 跳過 |
| Hover | 滑鼠停留 | 提示 tooltip(若有) |

### 鍵盤快捷鍵建議(熟手用,可後加)

| 按鍵 | 功能 |
|------|------|
| ESC | 暫停選單 / 取消 |
| Space | 結束本擊 / 推進對話 |
| Enter | 確認 / Lock |
| 1-9 | 快速 Place 對應卡 |
| F | 動畫加速 |
| H | 顯示 / 隱藏教學提示 |

### 控制器預留

- 不依賴**精確指標互動**(滑鼠 hover 顯示提示這類)
- 互動目標**足夠大**(手把可選擇)
- 雙模式 UI 應該都能用手把操作

## 五、雙模式 UI 具體規格

### 解鎖時機

| 條件 | 解鎖內容 |
|------|--------|
| **未完成序章** | **強制獵人模式**,設置中無切換選項 |
| **完成序章後** | 設置中出現「精簡 UI」開關 |
| 開關開啟 | 切換到騎兵模式 UI |

### 兩模式對照

| UI 面向 | 獵人模式 | 騎兵模式 |
|--------|---------|---------|
| 動畫速度 | 標準(1.0x) | 快速(2-3x) |
| 校準對話 | Modal 暫停遊戲 | 異步字幕飄過 |
| 道中對話 | 顯示 | 預設隱藏 |
| 教學提示 | 顯示 | 全部隱藏 |
| 戰鬥紀錄詢問 | 精英敵人對戰前自動跳出 | 不跳出(玩家可手動查) |
| 卡牌詳細描述 | 預設展開狀態多 | 預設緊湊 |
| 結算演出 | 完整 bar 填充 | 縮短演出 |

### 切換的視覺呈現

設置中:

```
┌─────────────────────────────┐
│  介面模式                    │
│                              │
│  ◉ 獵人模式(完整教學提示)  │
│  ◯ 騎兵模式(精簡介面)      │
│                              │
│  [說明]                      │
└─────────────────────────────┘
```

切換後**立即生效**,不需重啟。

## 六、教學期 UI 規範

### 灰階規則

**灰階卡** = 該卡可使用,但「**不建議當前嘗試**」:
- 視覺處理:降低飽和度 + 半透明
- 可點擊(玩家叛逆使用)
- 不出現在篩選結果(除非玩家明確切換)

### 高亮規則

**高亮卡** = 該卡「**建議優先嘗試**」:
- 視覺處理:邊框加亮 / 微微浮起
- 不限制其他卡可用

### 教學工具逐場解鎖

對應 [遊戲核心系統機制.md](../C__Implementation_benchmarks/遊戲核心系統機制.md) §13:

| 戰鬥 | UI 預設 | 解鎖工具 |
|------|--------|--------|
| 教學戰 1(哨兵) | 灰階所有不相關卡,只剩 6 張 | 基礎本擊區 + 庫存區 |
| 教學戰 2 | 灰階範圍縮小 | + 類型篩選按鈕 |

## 七、對話異步呈現規格

### Modal 觸發條件(強制停下)

- 重要劇情(戰役開場 / 中段 / 末段)
- 戰後校準(失敗時)
- 教學戰 1 的關鍵教學節點

### 異步飄過觸發條件(不暫停)

- 戰後校準(OHK 成功)
- 道中對話
- Hub 進入時 NPC 招呼(若玩家直接移動則跳過)

### 異步字幕呈現

**位置**:**螢幕底部**(寬度 60-80%,留邊距)

**結構**:

```
┌────────────────────────────────────────┐
│                                          │
│  [遊戲主畫面持續運作]                    │
│                                          │
│                                          │
│  ─────────────────────────────────────  │
│  「不錯。一箭判斷對,比兩箭判斷錯更值錢。」 │  ← 對話字幕區
│  ─────────────────────────────────────  │
└────────────────────────────────────────┘
```

**時序**:
- 淡入:0.2 秒
- 顯示:每句 2-4 秒(依字數)
- 淡出:0.3 秒
- 多句:逐句替換

**玩家可選操作**:
- 點擊字幕區 → 加速顯示 / 跳到下一句
- 設置中可調整顯示速度

### 角色標示

異步字幕需區分「玩家視角」與「NPC 視角」:

```
[玩家內心] 「我用了兩支箭,但一支就夠了」     ← 樣式 A
[NPC] 「不錯。一箭判斷對,比兩箭判斷錯更值錢」  ← 樣式 B
```

具體區分方式(風格相關)等美術風格決定後設計。

## 九、風格無關 vs 風格相關的清楚邊界

### 風格無關(本文件規範)

- **佈局**:元件位置、相對大小、間距比例
- **元件結構**:卡牌欄位、敵人 widget 結構
- **互動行為**:拖放、點擊、Lock 機制
- **狀態變化**:Place → Lock → 結算的視覺過渡
- **資訊密度**:緊湊 / 展開的區分

### 風格相關(美術風格決定後規範)

- 顏色:主色、次色、警示色、類型色
- 字體:選字、大小階層、行距
- 圖示:類別圖示、狀態圖示、按鈕圖示
- 動畫細節:緩動曲線、特效粒子
- 立繪:NPC、敵人視覺
- 背景:戰鬥場景、Hub 場景

### 第一期實作期間的處理

**用「佔位視覺」先 build 出機制**:
- 卡牌用 Godot 預設按鈕外觀 + 純文字
- 敵人用佔位色塊 + 文字標籤
- 動畫用最簡 Tween
- 對話用預設 Label 字體

**這讓 M0-M5 可以在無美術的情況下完成機制驗證**,等美術風格決定後在 M6 統一套表皮。
