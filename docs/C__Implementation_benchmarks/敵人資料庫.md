## 二、兔子(兔子型 A,教學基準)

> **2026-05-17 起序章前半全 5 連戰均不再使用**;由灰兔(§六)以 spawn_clone 機制全面取代。
> 兔子的「連鎖逃跑」機制(RABBIT_CHAIN_FLEE)程式留著,留作後續期數可能用到的設計工具
> (例如「群體會驚動而集體逃竄」的草食動物群)。

**一句話定位**:森林中的小型獵物,定義「弱點對應 vs 不對應」的次數差距感。

**敵人類別**:**兔子(A)** — 無威脅普通,失敗時逃走,**會引發其他兔子連鎖逃跑**

### 層 1 — 模板永久顯性

- 大類識別:無裝甲弱小目標
- 敵人類別:**兔子**
- 弱點範圍:穿刺(衝擊不在範圍,只能透過混合任意路徑計入)
- 弱點需求層級範圍:1(低)
- 戰鬥狀態傾向:預設標準戰

### 層 2 — 次級視覺線索

- 小體型、毛茸茸的耳朵與尾巴 → 對應視覺詞彙表 D.1 兔子型
- 警戒姿態、隨時準備逃跑 → 兔子型 D.1
- 無武器、無裝甲 → 弱點低需求
- 配色偏柔和(土色、灰色) → 不傳達威脅

### 層 3 — 即時狀態

- 戰鬥狀態標籤
- 上一場用偵查揭露的相關資訊

### 層 4 — 隱藏狀態(僅情報牌可揭)

完整需求表:
- **衝擊 3 / 穿刺 1 / 混合 5**

### 實例變異規則

序章前半此模板**不變異**,所有兔子實例均為上述需求表(教學基準)。

### 失敗後處理(類別:兔子)

- **失敗時**:卡牌消耗,該兔子逃走
- **連鎖逃跑**:**任一兔子逃走 → 連戰中「所有兔子」(含當前這隻)一起逃走,移出連戰序列**
- **不影響其他敵人**:狐狸、狼、異變的巨狼等**非兔子敵人不受連鎖影響**,留在連戰中
- **連戰不算失敗**:兔子逃走不會導致整場連戰失敗;連戰繼續打剩下的非兔子敵人,打完即連戰結束
- **第一連戰專屬**:連戰 1 全為兔子且啟用**教學重來機制** — 任一兔子失敗 → 卡組+狀態回滾 → 重來,直到三連勝(詳見 [遊戲核心系統機制.md](遊戲核心系統機制.md) §17)

### 設計合約

- **教玩家**:弱點對應的次數差距感(穿刺 1 對應 vs 衝擊 3 非對應)
- **老手能做到**:純看物種直接出 1 張射箭
- **新手失敗時能理解**:對話會傳遞「動作太大、嚇跑了」的方向感

### 登場時機建議

- **適合出現階段**:序章前半第一連戰(必出)、後續連戰可作過渡
- **不應出現的情境**:任何 BOSS 戰

## 三、狐狸(兔子型 A,無弱點偏好)

**一句話定位**:中等敏捷度的獵物,定義「沒有弱點時,要投注足夠資源」。

**敵人類別**:**兔子(A)** — 無威脅普通,失敗時逃走,**無連鎖**

### 層 1 — 模板永久顯性

- 大類識別:無裝甲弱小目標
- 敵人類別:**兔子**(歸入兔子類,但與兔子實例不同)
- 弱點範圍:衝擊 + 穿刺(兩者皆可,**無偏好**)
- 弱點需求層級範圍:2(低)
- 戰鬥狀態傾向:預設標準戰

### 層 2 — 次級視覺線索

- 中等體型、靈活姿態、尾巴蓬鬆 → 兔子型 D.1(但體型略大於兔子)
- 偏紅褐色或灰色,森林保護色 → 不傳達特定弱點偏好
- 蹲伏觀察姿態 → 警戒中,但仍是兔子型

**設計反差**:狐狸看起來「應該有弱點」(尖牙、爪、奔跑速度),但機制上**無弱點偏好** — 兩種類型都需要投注同等資源。

### 層 3 — 即時狀態

- 戰鬥狀態標籤
- 上一場用偵查揭露的相關資訊

### 層 4 — 隱藏狀態(僅情報牌可揭)

完整需求表:
- **衝擊 2 / 穿刺 2 / 混合 3**

> 即使揭露弱點,結果是「沒有弱點偏好」 — 偵查卡的價值在於「確認狐狸沒有偏好,所以不需要堅持某類型」。

### 實例變異規則

序章前半此模板**不變異**,所有狐狸實例均為上述需求表。

### 失敗後處理(類別:兔子)

- **失敗時**:卡牌消耗,該狐狸逃走
- **無連鎖逃跑**:狐狸的逃跑不會引發其他敵人逃走
- 連戰繼續進行下一目標

### 設計合約

- **教玩家**:「沒有弱點差異時要投注足夠資源」 — 偵查卡 lock 揭露「沒有偏好」,讓玩家確定投資路徑
- **老手能做到**:不用偵查,直接出 2 張任一類型
- **新手失敗時能理解**:NPC 會指出「使用力量不夠(投注資源不夠)」

### 登場時機建議

- **適合出現階段**:序章前半第二連戰(首次引入)、第三連戰繼續出現
- **不應出現的情境**:第一連戰(教學節奏要從單一弱點開始)

- 弱點範圍:穿刺
- 弱點需求層級範圍:2(低)

## 五、異變的巨狼(狼型 B·終末形,序章前半 BOSS)

**一句話定位**:序章前半的最終對手,身上有抓痕咬痕,左臂呈現極深的黑色,**預示森林異變**。

**敵人類別**:**狼(B)·終末形** — 機制上仍屬狼類(失敗菁英化),但視覺與敘事承擔「BOSS 級威脅」

> 註:此敵人原稱「熊」,因敘事需要更名為「異變的巨狼」。**機制上歸屬狼類**,失敗代價是菁英化(非 GAME OVER),敘事上承擔連戰高潮。

### 層 1 — 模板永久顯性

- 大類識別:精英重甲(肌肉發達 + 詛咒外觀)
- 敵人類別:**狼**
- 弱點範圍:衝擊
- 弱點需求層級範圍:5(中)
- 戰鬥狀態傾向:預設標準戰

### 層 2 — 次級視覺線索

- **壓倒性體型**(明顯大於普通狼) → 高需求暗示
- **身上有抓痕與咬痕** → 「曾與其他生物搏鬥過」的敘事線索
- **左臂明顯不祥氣息,呈現極深的黑色** → 「異變」的視覺標記,暗示某種詛咒
- 厚實肌肉與緊繃皮膚 → 衝擊弱點(鈍器擊倒厚實生物;穿刺難以穿透)

### 層 3 — 即時狀態

- 戰鬥狀態標籤
- 上一場用偵查揭露的相關資訊
- 若為菁英狀態:UI 標籤「菁英化 ⚠」+ 戰鬥紀錄可調出上次資料

### 層 4 — 隱藏狀態(僅情報牌可揭)

完整需求表:
- **衝擊 5 / 穿刺 8 / 混合 12**

### 實例變異規則

序章前半此模板**不變異**,單一實例 = 序章前半 BOSS。

### 失敗後處理(類別:狼·終末形)

- **第一次失敗(預期會發生)**:菁英化,排到本連戰末段。卡組消耗保留
- **菁英化失敗**:GAME OVER

> **設計意圖**:**預期玩家首戰會戰敗**,然後菁英化版本進入正式 BOSS 戰。這是教學「菁英化的運作」的核心場景。

### 設計合約

- **教玩家**:菁英化的運作(失敗 → 同一場連戰末段再戰一次)+ BOSS 級資源投入

- **新手失敗時能理解**:對話會傳遞「資源量根本不夠 / 方向錯了」的份量感

### 劇情敘事

### 登場時機建議

- **僅出現於**:序章前半第五連戰(最終戰)
- **不應出現的情境**:其他任何位置

## 六、灰兔(兔子型 A 變體,2026-05-16 新加)

**一句話定位**:外形似兔的小型獵物,定義「失敗代價的具體化 —— 沒擊中會留下複製體,下一場立刻面對」。**取代兔子作為 chain_1 教學基準**。

**敵人類別**:**灰兔(A 變體)** — 自成 enemy_class `gray_rabbit`;default 失敗時 spawn 一個 clone 到 queue 緊接後位置,clone 自己失敗則無連鎖。

### 兩個 instance

| instance | display_name | special_effect | 失敗代價 |
|---|---|---|---|
| `gray_rabbit_default` | 灰兔 | `{ on_fail: { action: spawn_clone, clone_instance_id: "gray_rabbit_clone" } }` | 逃走 + clone 插入緊接後 |
| `gray_rabbit_clone` | 灰兔(複製) | `{}`(空) | 單獨逃走,連戰繼續(類似狐狸) |

### 層 1 — 模板永久顯性(共用模板 `gray_rabbit`)

- 大類識別:無裝甲弱小目標
- 敵人類別:**灰兔**(自成新 class,顏色:`#6e7e8f` 偏藍灰)
- 弱點範圍:穿刺(衝擊不在範圍)
- 弱點需求層級範圍:1(低)
- 戰鬥狀態傾向:預設標準戰

### 層 2 — 次級視覺線索

- 小體型、灰色毛 → 區別於常規兔子的草綠系
- 警戒姿態 → 兔子型 D.1
- 無武器、無裝甲 → 弱點低需求
- 配色偏冷灰 → 不傳達威脅,但跟兔子有色差(教導玩家「灰色 = 失敗有代價」)

### 層 4 — 隱藏狀態(僅情報牌可揭)

- **衝擊 3 / 穿刺 1 / 混合 5**(跟兔子相同需求數值)

### 失敗後處理

**default 失敗(`gray_rabbit_default`)**:
- 卡牌消耗,該灰兔逃走
- **spawn 1 隻 clone(`gray_rabbit_clone`)插入到 queue 的下一個位置** —— 下一場戰鬥就會面對它
- 連戰繼續

**clone 失敗(`gray_rabbit_clone`)**:
- 卡牌消耗,clone 單獨消失
- **無進一步連鎖**(clone 的 special_effect 為空)
- 連戰繼續

**硬天花板**:每 default 最多 spawn 1 clone,clone 不再 spawn。chain_1 默認 3 default → 最壞 3 default + 3 clone = 6 場。

### 設計合約

- **教玩家**:失敗代價是具體的「多一隻敵人」,不是抽象的「重來次數」
- **老手能做到**:純看物種直接出 1 張射箭,OHK 化解失敗代價
- **新手失敗時能理解**:對話會引導「上次太用力了嗎,結果跑出第二隻」

### 登場時機建議

- **適合出現階段**:序章前半第一連戰(必出,取代兔子的教學基準角色)
- **不應出現的情境**:任何 BOSS 戰、高難度連戰(失敗代價過於溫和)

## 七、序章前半連戰配置

### 連戰 1:教學起點(2026-05-16 重設)

```
灰兔 + 灰兔 + 灰兔  (default × 3)
```

- **核心目標**:教學「選卡出卡結算基本邏輯」+ 建立 NPC 信任 + **失敗代價具體化**
- **預期消耗**:3-8 張攻擊卡(全 OHK)~ 6-12 張(連 fail 觸發 clone)
- **劇情要求**:**通過全 queue** — default + 失敗造成的 clone 都打完才算過連戰 1
- **特殊規則**:**灰兔 spawn_clone 機制**(§六)— default 失敗 → clone 插入緊接後位置;clone 失敗即消失,不再連鎖
- **失敗代價邊界**:每 default 最多 spawn 1 clone → 最壞 3 default + 3 clone = 6 場戰鬥
- **整備 1**:完全休息,卡片全部恢復
- **教學重來機制**:**已停用**(`chain_1.tutorial_retry = false`);機制碼留在 FailureHandler + m2 + TutorialRetry,待灰兔機制驗證跑順後評估清除

### 連戰 2:無弱點教學

```
灰兔 + 狐狸 + 狐狸
```

- **核心目標**:辨識是否有弱點 + 「沒有弱點時要投注足夠資源」
- **預期消耗**:5-8 張攻擊卡(全 OHK)~ 6-9(灰兔 fail 觸發 1 clone)
- **失敗代價**(2026-05-17):
  - 灰兔失敗 → default 逃走,**spawn 1 clone 插入緊接後**(下一場是 clone,然後才到狐狸)
  - 狐狸失敗 → 僅該狐狸單獨逃走,NPC 指出「使用力量不夠(投注資源不夠)」,連戰繼續
- **整備 2**:+2 射箭、+2 投石
- **劇情敘事**:外面出現狼嚎,停止休息確認周遭安全

### 連戰 3:狼登場

```
灰兔 + 狐狸 + 狼
```

- **核心目標**:辨識狼的弱點 + 卡牌會被消耗的概念
- **預期消耗**:6-10 張攻擊卡(全 OHK)~ 7-11(灰兔 fail 觸發 1 clone)
- **失敗代價**(2026-05-17):
  - 灰兔失敗 → default 逃走,spawn 1 clone 緊接後
  - 狐狸失敗 → 單獨逃走
  - 狼失敗 → 菁英化排末(連戰末段)
- **整備 3**:+2 射箭、+2 投石
- **劇情敘事**:出現更多狼嚎,狼身上有爪痕(暗示異變)

### 連戰 4:狼群驗收

```
狼 + 狼 + 狼
```

- **核心目標**:確認玩家對狼的運作完全理解
- **預期消耗**:8-12 張攻擊卡
- **整備 4**:+3 射箭、+3 投石(父親將備用裝備全部給兒子)

### 連戰 5:BOSS 戰

```
灰兔 + 灰兔 + 異變的巨狼(+ 菁英化異變的巨狼)
```

- **核心目標**:確認菁英化運作 + BOSS 級資源投入
- **連戰順序**:灰兔 → 灰兔 → 異變的巨狼(巨狼排在最後)
- **失敗代價**(2026-05-17):
  - 灰兔失敗 → default 逃走,spawn 1 clone 緊接後(延後巨狼出場)
  - 巨狼首戰失敗 → 菁英化 → 排到連戰末段(預期玩家首戰會敗)
  - 菁英化巨狼失敗 → GAME OVER
- **連戰長度上限**:最壞 6 場(2 灰兔 default + 2 clone + 巨狼首戰 + 菁英巨狼)
- **預期流程**:灰兔打完(或失敗 → clone)→ 巨狼首戰失敗 → 菁英化 → 菁英化巨狼再戰一次
- **設計考量**:灰兔 spawn_clone 會延後 BOSS 出場,**測試時要注意這段節奏是否合適**;若拉長感覺不對,後期可考慮 chain_5 改回兔子或減少灰兔數量
- **劇情敘事**:擊敗後父親拿走斷臂,留下主角獨自前行

## 七、後續期數參考:舊版 5 模板

> **以下為 2026-05-13 版本的 5 隻原型敵人模板**,作為**後續期數 / 序章後半 / 主冒險敵人擴充**的設計參考保留。
> 完整 instance 變異 / 視覺線索 / 失敗代價詳情參考 git 歷史 v3.x 版本。

### 7.1 哨兵(兔子型,輕甲偵察)

- **大類**:輕甲偵察(對應後續期數的城鎮 / 領主軍勢)
- **弱點範圍**:衝擊 / 穿刺
- **需求層級**:1 / 2
- **後續期數定位**:序章後半或第二期登場時,作為「比兔子更專業的兔子型敵人」(有武器 + 輕甲)

### 7.2 巡守(狼型,戰鬥狀態變體)

- **大類**:中型近戰
- **弱點範圍**:衝擊 / 穿刺 / 燃燒
- **需求層級**:2 / 3 / 4
- **戰鬥狀態**:60% 標準 / 30% 先攻 / 10% 被襲
- **後續期數定位**:第二期主力敵人,引入「戰鬥狀態」概念

### 7.3 重甲衛兵(狼型,弱點配對價值)

- **大類**:重甲近戰
- **弱點範圍**:衝擊 / 穿刺
- **需求層級**:2 / 3 / 4
- **後續期數定位**:第二期重點敵人,教學「弱點對應的投資價值」

### 7.4 伏擊者(豹型,情報遮蔽)

- **大類**:輕甲刺客
- **弱點範圍**:穿刺 / 燃燒
- **需求層級**:1 / 2
- **戰鬥狀態**:80% 被襲
- **失敗代價**:**GAME OVER**(預設精英)
- **後續期數定位**:第二期或第三期引入,需要先建立玩家的視覺判讀能力

### 7.5 領主(熊型,核心 BOSS)

- **大類**:精英重甲
- **弱點範圍**:衝擊 / 穿刺
- **需求層級**:4 / 6
- **失敗代價**:**GAME OVER**(劇情 BOSS)
- **後續期數定位**:主冒險的章節 BOSS(非序章 BOSS,序章 BOSS 改為異變的巨狼)

### 7.6 後續期數的引入順序建議

| 期數 | 引入新敵人 | 理由 |
|------|----------|------|
| 序章後半 | 哨兵類(人類偵察)、巡守標準變體 | 視覺從動物轉為人類 / 異形,引入戰鬥狀態 |
| 第二期 | 重甲衛兵、巡守全變體 | 引入弱點對應投資價值教學 |
| 第三期+ | 伏擊者、領主 | 引入豹熊型失敗即 GAME OVER 概念 |
