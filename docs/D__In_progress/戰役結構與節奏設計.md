## 一、核心發現:雙模式節奏

P6 的節奏應該支持**兩種模式並存**,由玩家當下的熟練度與資源狀態決定處於哪個模式。

### 兩種模式

| 模式 | 玩家狀態 | 節奏 | 戰鬥行為 | UI/敘事呈現 |
|------|---------|------|---------|-----------|
| **獵人模式** | 新手 / 資源不足 / 不熟敵人 | 慢、謹慎 | 投資情報、試探驗證、仔細推理、Place 重排多次 | 戰後校準完整、NPC 觀察豐富、道中對話可見 |
| **騎兵模式** | 熟手 / 資源充足 / 熟悉敵人 | 快、致命 | 不投情報、直接 OHK、Place 即 Lock、無重排 | 戰後校準最小化、NPC 沉默、道中對話跳過 |

### 關鍵原則

**這不是兩種遊戲,是同一場戰鬥的兩種展開方式**。決定權在玩家身上;**元知識的累積就是從獵人到騎兵的滑動軸**。

### P6 已內建的雙模式基礎

回頭看主線文件,雙模式的基礎其實已經在:

| P6 既有機制 | 如何支持雙模式 |
|------------|--------------|
| 「猜弱點」是可選的 | 騎兵直接猜,獵人投資情報 |
| 情報卡 optional lock(廣度 vs 深度) | 騎兵不用,獵人精算 |
| 試探卡作為「驗證猜測」工具 | 獵人用、騎兵不用 |
| 老手 vs 新手的「設計合約三問」 | 每個敵人模板就為兩種模式設計 |
| NPC 自適應教學(早期多話、後期沉默) | 自然從獵人 → 騎兵的過渡 |
| 視覺線索老手可讀(層 2) | 騎兵眼睛讀 = 不投情報 |

→ 雙模式不是「新需求」,**是 P6 設計的正確命名**。「獵人 vs 騎兵」是「**理解即進度**」原則的具體形象化。

## 二、戰役結構模型

### 標準模型

```
戰役(大段落)= 
  開場重要劇情
  →  [連戰 + 整備節點(含小劇情)] × N
  →  最終挑戰(boss + 戰役末劇情)
```

整備節點扮演「**呼吸點**」:小劇情 + 校準對話 + 卡牌回補。N 是設計變數。

### 30 分鐘預算估算

| 元素 | 單次時長 | 一場戰役總量 | 預算佔比 |
|------|---------|------------|---------|
| 戰鬥 | 3-5 分(獵人模式)/ 30 秒-1 分(騎兵) | 5-8 場 | 20-30 分 |
| 整備節點 | 1-2 分 | 1-2 次 | 1-4 分 |
| 重要劇情 | 2-3 分 | 3-4 段 | 6-12 分 |
| 道中對話 | 30 秒 | 5-7 次 | 2.5-3.5 分 |

**真正的時長瓶頸是重要劇情數量 + 整備次數**。道中對話可以撒得頻繁不破預算。

### 雙模式下的時長差異

| 模式 | 一場戰役總時長 | 主要差異來源 |
|------|------------|------------|
| 獵人模式(首次玩) | 25-35 分 | 完整看劇情、戰鬥慢、校準對話多 |
| 騎兵模式(熟手) | 8-15 分 | 跳過劇情、戰鬥快、動畫加速 |

→ 同一場戰役,**時長差約 3-4 倍**——這是雙模式設計的具體威力。

## 三、四種循環配比

### α 直行式(1 整備 + 中段重要劇情)

```
開場(2分) → 連戰 1:3 場 + 道中對話 ×2 → 中段重要劇情(2分) → 連戰 2:4 場 含 boss + 道中 ×3 → 戰役末(2分)
```
- 1 整備 / 3 段重要劇情 / 約 5 次道中對話
- 節奏:像一部短篇電影,中段轉折明確

### β 兩幕式(平衡型)

```
開場(2分) → 連戰 1:4 場 + 道中 ×3 → 整備 + 中段劇情(3分) → 連戰 2:4 場 + 道中 ×3 + 戰役末(2分)
```
- 1 整備 / 3 段重要劇情 / 約 6 次道中對話
- 適合**旅程敘事**(白天行程 + 夜晚紮營 + 隔日行程)

### γ 三段升級(教學戰役專用)

```
開場(2分) → 短連戰:2 場 + 道中 → 整備 + 小劇情(1.5分) → 中連戰:3 場 + 道中 → 整備 + 重要劇情(2分) → 長連戰:3 場含 boss + 道中 + 戰役末(2.5分)
```
- 2 整備 / 4 段劇情 / 約 5 次道中對話
- 對應**教學進程**(哨兵→巡守→重甲→伏擊者→領主)
- 漸進負擔最友善

### δ 持續推進(沉浸型)

```
開場(2分) → 一條長連戰:6-7 場 + 道中對話 ×6 + 中段微停頓(30秒內心獨白)→ Boss + 戰役末(3分)
```
- 0-1 整備 / 2 段重要劇情 / 6+ 次道中對話
- 沒有結構性停頓,沉浸感最強
- ⚠ 缺點:狼型失敗的「進未解決池 + 追擊」機制空間被壓縮

### 對雙模式的契合度

| 結構 | 獵人模式 | 騎兵模式 | 適用時機 |
|------|--------|--------|--------|
| α 直行式 | ✓ | △ 中段斷點打斷 | 標準戰役 |
| β 兩幕式 | ✓✓ | △ 中段整備 = 下馬 | 旅程式敘事 |
| **γ 三段升級** | ✓✓ **完美** | ✗ 太多停頓 | **教學戰役、首次玩** |
| **δ 持續推進** | ✗ 沒有思考喘息 | ✓✓ **完美** | **重玩、騎兵模式專用** |

### 關鍵洞察:同一場戰役、兩種結構呈現

**第一次玩 = γ 模式呈現,第 N 次重玩 = δ 模式呈現**——同樣的戰役,**結構自動隨玩家熟練度滑動**。

像 Sekiro 同一個 boss,新手打 30 分鐘、老手打 30 秒,但**底層機制完全沒變**——是玩家自身的進化讓節奏改變。

### 不對稱循環(進階)

| 配比 | 模式 | 適合敘事 |
|------|------|---------|
| **a-b-c-F** | 2 → 3 → 4 → Boss(漸增) | 教學戰役、能力成長 |
| **a-b-a-F** | 3 → 5 → 3 → Boss(中段最重) | 商人護衛(山路艱險 → 出山緩和 → 終點意外) |
| **A-B-F** | 4 → 5 → Boss(大循環+更大循環) | 古墓探險(淺層 → 深層 → 守護者) |
| **a-a-A-F** | 2 → 2 → 5 → Boss(累積) | 退役戰士(零星遭遇 → 主力會戰) |

不對稱循環比對稱更有故事感——對稱容易感覺機械,不對稱讓玩家潛意識感受到「敘事弧」。

## 四、敘事分層

P6 的敘事採取四層密度配置,**每層必須跟戰鬥節奏共存,不能打斷**:

| 層 | 內容 | 觸發時機 | 時長 | 騎兵模式處理 |
|----|------|--------|------|----------|
| 1 戰鬥中道中對話 | 短句、戰鬥開場/結算瞬間 | 戰鬥同場 | < 5 秒 | 異步呈現,可忽略 |
| 2 戰後校準對話 | 雙聲道反思 | OHK 動畫期間 | 5-15 秒 | 字幕疊加動畫,不暫停 |
| 3 整備節點小劇情 | NPC 對話 / 環境描寫 | 連戰之間 | 1-2 分 | 可跳過 |
| 4 戰役重要劇情 | 開場 / 中段轉折 / 結尾 | 戰役邊界 | 2-3 分 | 首次必看,重玩可跳 |

### 整備節點的「小劇情」具體可能

整備節點 ≠ 強制互動選項。內容可以是:

| 類型 | 內容 |
|------|------|
| 紮營對話 | NPC 與玩家的簡短對話 |
| 環境變化 | 場景轉換的視覺敘事 |
| 敵方反應延續 | 上輪敵人在「世界」中的回聲(暗示精英化敵人歸來) |
| 戰利品/紀錄整理 | 玩家「翻看」戰鬥紀錄的視覺呈現 |

整備節點 ≈ **校準對話 + 場景轉場 + 卡牌回補**,**不需要做選擇分支**。

## 五、雙模式設計的實現

### 必須做的(UI / 體驗層)

| 調整 | 描述 |
|------|------|
| 動畫速度可調 | 設置中 normal / fast,或點擊跳過動畫 |
| 校準對話可跳過 | hold 鍵跳過,或點「下一場」直接前進 |
| 道中對話可隱藏 | 第二次玩同戰役預設關閉,設置可重新打開 |
| 「結束本擊」即播放 | Place/Lock 不限時(獵人慢慢來),按下執行後全部結算一次跑完 |
| 重要劇情可跳過 | 首次必看,重玩可跳 |

### 機制層的雙模式支持(其實已有)

| 機制 | 雙模式支持方式 |
|------|------------|
| 資源池快用 | 獵人慢慢挑、騎兵一次拖 3 張下去 |
| 情報卡完全可選 | 獵人投資、騎兵不用 |
| 試探卡可不用 | 獵人用驗證、騎兵省略 |
| flexible 分配可即決 | 獵人想三秒、騎兵立刻分配 |

### 不要做的(會破壞模式滑動)

| 反設計 | 為什麼不要 |
|--------|---------|
| 強制 modal 對話 | 任何「按確認才能繼續」的對話都會把騎兵拉下馬 |
| 強制動畫時長 | 看了 100 次的擊殺動畫不該每次都 2 秒 |
| 把教學寫進機制本身 | 教學是 UI 預設 + NPC 對話(可關),不是強制流程 |

## 六、競技場模式作為次要內容

雙模式設計天然支持「**無敘事重玩**」模式——即競技場。

### 設計判準

| 競技場模式特性 | 設計選擇 |
|--------------|---------|
| 劇情 | **無或極簡**(只有結算對話) |
| 連戰結構 | 連續波次,無明顯「連戰邊界」 |
| 整備 | **強化版**(波次間補給更明確) |
| 卡組 | 可能隨機 / 可挑選 / 解鎖式 |
| 解鎖時機 | **打完第一場主戰役後**(玩家已熟悉機制) |
| 設計目的 | 「mastery 玩法」——讓老手用同套系統挑戰更純粹的戰術問題 |

### 對個人開發者的優勢

**內容複用率高**——同一套敵人、同一套卡牌,只是抽掉敘事與教學包裝,變成「純機制體驗」。可作為主戰役完成後的延長遊玩。

→ 對應 OTK / Slay the Spire 的設計位置。

## 七、參考遊戲對標

### 雙模式節奏的最佳範例

| 遊戲 | 雙模式表現 | 可借鏡的細節 |
|------|---------|------------|
| **Sekiro** | 新手謹慎格擋 vs 老手連格擋處決 | 「同一機制兩個世界」是 P6 的理想 |
| **Slay the Spire** | 新手讀牌慢出 vs 老手 1 分鐘清層 | **動畫速度可調、enemy turn 可跳過** |
| **Into the Breach** | 思考模式 vs 執行模式分離 | Place(思考)/ 結束本擊(執行)的切換已對應 |
| **Hades** | 對話異步播放、不打斷遊戲 | **校準對話可疊加動畫播放** |

### 結構參考

| 遊戲 | 可借鏡的細節 |
|------|------------|
| **Slay the Spire** | 「篝火 = 整備節點」+ 「Path Choice 不分支劇情」 |
| **Darkest Dungeon** | Narrator 觀察聲音 + Expedition/Hamlet 雙場景 |
| **Hades** | 對話分層:戰鬥中(輕)/ 區域結尾(中)/ Hub(重)|
| **Banner Saga** | 旅程式線性敘事 + 戰鬥節點 |
| **Inscryption Act 1** | 卡牌節拍緊湊 + 緊湊敘事不打斷 |

### 反向參考(不要走的路)

| 遊戲 | 為什麼不要學 |
|------|-----------|
| **Slay the Princess** | 多結局分支會把 30 分鐘戰役稀釋到無感 |
| **Persona 5** | Social Sim 養成違反 OHK「無養成」原則 |
| **Hades 死亡循環機制本身** | P6 失敗是「有機後果」非「死了重來」 |
| **Wildermyth** | 故事生成器讓敘事失去設計掌控 |

### One Turn Kill 的特殊位置

OTK 採取「**敵人設計遞增 + 荒野氛圍 = 敘事**」的極簡裝飾路線,**完全不靠對話與 NPC**。

P6 不採用 OTK 路線的理由:
- P6 不是 roguelike 無限重玩,**敘事壓力小很多**
- P6 單場戰鬥認知負擔低(填達標 bar),**有空間容納敘事**
- P6 雙模式設計讓敘事可在騎兵模式自動跳過

但 OTK 的「**敵人本身講故事**」哲學對 P6 仍適用——敵人視覺類別(兔子/狼/豹/熊)是 P6 版的「敵人即敘事」。

- 加入「**雙模式節奏**」概念
- 結構選擇從「線性 4-6 連戰」改為「γ 三段升級 + 騎兵模式可滑動到 δ」
- 整備節點功能明確化(校準 + 卡牌回補 + 環境轉場)

### 遊戲核心系統機制.md 第七章(整備期)

- 整備節點的內容類型化(紮營對話 / 環境變化 / 敵方反應 / 戰利品整理)
- **明確「整備節點不需要互動選項」**

### 遊戲核心系統機制.md 第八章(戰後校準對話)

- **校準對話從「強制 modal」改為「異步動畫疊加」**
- 增加「失敗才強制停下」例外規則
- 加入「校準對話的雙模式呈現」段落

### 遊戲核心系統機制.md 第十一章(教學期 UI)

- 教學期 UI 改為**獵人模式的子集**——不是獨立系統,是雙模式中「獵人」端的明確化
- 對應的進階模式 UI = 騎兵模式

### 卡牌資料庫.md / 敵人資料庫.md

- 不需要重寫,但**驗算雙模式可行性**:每個敵人模板的「設計合約三問」對應獵人 / 騎兵兩種解法
