## 一、設計概念

### 核心想法

> **卡包 = 數張卡組成的「主題單位」**。
> 戰役牌庫 = 多個卡包構成。

### 為什麼有用

| 對象 | 卡包的價值 |
|------|---------|
| **設計者** | 用「卡包」做為平衡單位,降低 28+ 張個別卡的平衡複雜度 |
| **玩家(後期)** | 透過選不同卡包組合 → 不同 playstyle |
| **多角色系統** | 不同角色 = 不同預設卡包組合 = 不同個性 |
| **重玩價值** | 同戰役用不同卡包配置 = 不同體驗 |

### 跟同類遊戲的對應

| 遊戲 | 對應概念 |
|------|---------|
| **Library of Ruina** | 司書 niche(攻擊型 / 防禦型 / 異常型),玩家把對的人放對的地方 |
| **Slay the Spire** | 角色卡牌池差異(鐵甲、希子、瑞、看守者) |
| **Hearthstone / 對戰卡牌** | 職業卡池 + 中立卡 |
| **P6 既有「角色擅長之事」** | 卡組是角色的「能做到的事」(通用 + 專屬) |

## 二、兩種實現模式

### A.1 自由組卡包模式(玩家選擇)

```
玩家在戰役構築階段:
  從可用卡包池選 N 個卡包 
    ↓
  組成本場戰役的牌庫
    ↓
  進入戰役
```

**範例**:
```
玩家選擇:[補給卡包] + [弱點卡包(穿刺)] + [謹慎卡包]
組成牌庫:
  - 3 張基本攻擊(直擊)
  - 1 張偵查
  - 4 張穿刺攻擊(銳刺、貫穿)
  - 3 張偵查試探卡
  總計 11 張
```

**特性**:
- 玩家有**明確的構築選擇權**
- 適合**熟手玩家 / 重玩階段**
- 教學門檻較高

### A.2 角色預設卡包模式(系統預設)

```
玩家選角色 = 系統自動載入該角色的預設卡包組合
    ↓
  進入戰役(玩家看不到「卡包」字眼,只看到「我的卡組」)
```

**範例**(第一期序章主角):
```
序章主角 = 「商人護衛性格卡包」 + 「初階戰鬥卡包」 + ...
玩家看到:28 張卡的固定卡組
玩家不知道:這 28 張是由 N 個卡包組成
```

**特性**:
- 玩家**不需學「卡包」概念**
- 跟 P6 既有「**角色擅長之事**」設計完全相容
- 適合**新玩家 / 第一期序章**
- 卡包仍是**設計者的內部工具**

## 三、卡包類型範例

### 機制型卡包(對應 P6 卡牌兩軸)

| 卡包 | 內容範例 | 對應軸 |
|------|--------|------|
| **初階戰鬥卡包** | 3 張基本攻擊(直擊 / 銳刺 / 點火)| tool × combat,低階 |
| **偵查卡包** | 2-3 張情報卡(偵察、估量、警示)| tool × intel |
| **試探卡包** | 試探卡 + 1-2 張攻擊 | tool × compound |
| **爆發卡包** | 2-3 張爆發卡 | burst × combat |
| **角色卡包** | 角色卡組合(殺戮意志 + 致命凝視等)| character × * |

### 主題型卡包(跨軸混合)

| 卡包 | 主題 | 內容範例 |
|------|------|--------|
| **補給卡包** | 平民工具 | 3 張基本攻擊 + 1-2 張偵查 |
| **戰鬥卡包** | 火力集中 | 1 張基本 + 2 張收益更高的攻擊 |
| **弱點卡包(穿刺)** | 針對穿刺弱點 | 銳刺 ×3 + 貫穿 ×2 + 致命刺 |
| **謹慎卡包** | 偵察優先 | 偵察 + 估量 + 試探卡 |
| **奇襲卡包** | 高爆發 | 爆發卡 ×2 + 致命凝視 |

### 性格型卡包(對應角色性格)

| 性格 | 內容傾向 | 玩法風格 |
|------|--------|--------|
| **激進** | 戰鬥卡多 / 情報卡少 | 直接出手,不投資情報 |
| **保守** | 情報卡多 / 試探卡多 | 仔細偵察才出手 |
| **奇襲** | 基本卡少 / 強力單卡多 | 留資源等關鍵時刻爆發 |
| **均衡** | 各類比例平均 | 隨機應變 |

→ **A.2 模式下,玩家選的「角色」實際上 = 選了一個性格卡包**。

## 四、跟既有設計的對應與張力

### 跟 [卡牌設計原則.md §11 多角色設計] 對應

[卡牌設計原則.md](../B__Design_specifications/卡牌設計原則.md) 已提:

> 每個角色實際上是「**一套不同類型偏好的資源池 + 數張個體角色卡**」

**範例**:
> - 狩獵者:資源池偏穿刺與情報
> - 狂戰士:資源池偏衝擊與通用
> - 學者:資源池偏複合卡

→ **這正是 A.2 卡包模式的具體實現**——只是 P6 既有設計用「**資源池偏好 + 角色卡**」描述,卡包概念給了它一個**組合單位**的名字。

### 跟 [遊戲核心系統機制.md §7 卡組概念] 對應

[遊戲核心系統機制.md](../C__Implementation_benchmarks/遊戲核心系統機制.md) §7 提:

> 卡組由「**通用牌 + 角色專屬牌**」組成

→ **這正是「兩個卡包」的具體實現**:
- **通用牌卡包**:23 張(各角色共用)
- **角色專屬卡包**:5 張(角色獨有)

### 潛在張力

| 既有設計 | 卡包化的潛在張力 | 處理建議 |
|---------|---------------|--------|
| 「卡組 = 角色擅長之事,跨戰役不大幅變化」 | A.1 自由組讓戰役間卡組變化大 | **第一期不採 A.1**,跨戰役卡組仍固定 |
| 28 張固定卡組 | A.1 讓玩家可選不同組合 | **第一期維持 28 張**,卡包是後台組合工具(A.2) |
| 無養成原則 | 「解鎖更多卡包」可能像養成 | **解鎖卡包是「選項擴充」不是「能力強化」**,符合橫向擴充原則 |
| 序章主角 = 1 個角色 | 卡包概念若曝光給玩家可能困惑 | **第一期 A.2 模式,玩家看不到卡包字眼** |

## 五、平衡機制

### 卡包內容固定,不機率抽

每個卡包**內容固定**:打開「補給卡包」就是固定那幾張卡,**不採取機率抽選**。

→ 對應 [遊戲設計概念.md](../B__Design_specifications/遊戲設計概念.md) §6「**禁止抽牌 / 機率成敗**」原則。

### 戰役級 cost 上限(若採 A.1 自由組)

可能的平衡機制:
- 每個卡包有「**構築 cost**」數值
- 戰役有「**最大 cost 總和上限**」
- 玩家選卡包時 cost 不能超過上限

**重要釐清**:

| 概念 | 含義 | 是否符合 P6 |
|------|------|-----------|
| **戰役構築 cost** | 構築階段的平衡值 | ✓ 符合(構築階段) |
| **戰鬥內 cost** | 戰鬥中出卡消耗能量 | ✗ 違反 P6「無能量系統」原則 |

→ 卡包的 cost 是**構築階段平衡**,**不是戰鬥內 cost**,這個區分要明確。

### 一張卡可在多個卡包

提案:同一張「直擊」可能既在「初階戰鬥卡包」也在「補給卡包」。

**需釐清的實作意義**:

| 詮釋 | 含義 |
|------|------|
| **A:同 id 同實例** | 選兩個都含直擊的卡包,**總共仍只一張直擊**(去重) |
| **B:同 id 不同實例** | 選兩個都含直擊的卡包,**會有兩張直擊** |

→ **建議採詮釋 A**(去重),否則容易堆疊不平衡。

### 卡包不一定唯一

提案後期可用**複數同卡包**構成牌庫(例:選兩個「初階戰鬥卡包」)。

→ 這是 A.1 後期的擴充,**第一期不必處理**。

## 六、設計問題與處理方向

### 問題 1:卡包的「擁有 vs 選擇」邊界

**選項**:
- A. 玩家**擁有所有卡包**,每場戰役選帶哪幾個(無解鎖)
- B. 玩家**通關後解鎖新卡包**(有解鎖,但**只是選項擴充**)

**建議**:
- 第一期不開放選擇(A.2 模式)
- 後續期數開放 B(對應「橫向擴充」原則)

### 問題 2:第一期是否曝光「卡包」概念給玩家

**選項**:
- A. **隱藏**:玩家看到的是「我的卡組 28 張」,不知道有卡包
- B. **曝光**:玩家看到「序章主角 = 商人護衛卡包 + 初階戰鬥卡包」

**建議**:**A 隱藏**——降低教學門檻,卡包僅是設計者的組合工具。

### 問題 3:第二期之後如何引入「玩家選卡包」

**漸進路徑**:
1. **第一期**:A.2 隱藏(玩家不知有卡包)
2. **第二期+**:加新角色 = 加新預設卡包組合(仍隱藏)
3. **更後期**:解鎖「**自由組卡包模式**」作為重玩擴充

### 問題 4:卡包之間的相容性

如果一張卡在多個卡包,可能造成:
- 玩家選「補給包 + 戰鬥包」,兩個都含直擊 → 詮釋 A 去重?
- 但這會讓某些卡包搭配「**有效張數變少**」,有損玩家預期

**建議**:
- 卡包之間**盡量不重複卡** → 設計時注意
- 必要時採詮釋 A 去重,且 UI 明確告知

### 問題 5:「印象最深保留 2-3 張」概念整合

對應 [戰役章節式架構討論.md](../F__History/戰役章節式架構討論.md) 的「印象最深保留」概念:
- **玩家通關後,可保留 2-3 張當該戰役最印象深的卡**
- 這幾張卡可以成為「**自定卡包**」的種子(後續期數)

→ 第一期不必處理,但**架構預留**。

## 七、推薦採用策略

### 第一期(序章)

**採用 A.2 隱藏卡包**:
- 序章主角 = 預設卡包組合(玩家看到 28 張固定卡組)
- **玩家看不到「卡包」字眼**
- 但設計者**內部用卡包做組合與平衡**

**好處**:
- 教學門檻不變
- 不違反「卡組是角色擅長之事」原則
- 為後續多角色預留架構

### 第二期之後(多角色)

**繼續 A.2 隱藏卡包,但卡包池擴大**:
- 加新角色 = 加新性格卡包 + 新角色專屬卡包
- 通用卡包池逐步擴張(新角色可能引入新通用卡)
- 玩家**仍看不到「卡包」字眼**,只看到「角色 A 的卡組 / 角色 B 的卡組」

### 更後期(成熟階段)

**開放 A.1 自由組卡包模式**作為「重玩擴充」:
- 解鎖條件:完成主冒險
- 玩家可在 Hub 進入「**自由構築模式**」
- 用累積的所有卡包自由組合
- **不影響主線戰役**(主線仍用預設組合)

→ **這對應 [遊戲設計概念.md §6](../B__Design_specifications/遊戲設計概念.md) 的「橫向擴充」原則**。
