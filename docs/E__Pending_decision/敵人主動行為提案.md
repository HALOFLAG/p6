# 敵人主動行為提案

> **狀態**: 後續期數待決策
> **第一期是否實作**: 否。5 個原型敵人 traits 皆為空陣列;`traits: Array = []` 欄位已預留
> **待決策問題**: 第二期之後逐步加入(戰役 2 警示型、戰役 3 反擊型),具體哪期加哪型待設計
> **最後同步**: 2026-05-13

本文件記錄「**敵人主動行為**」機制的設計提案——讓敵人在戰鬥中不只是「被動需求表」,而是有特性、有反應、有威脅。

**第一期 Scope 嚴格規定**:
- **不實作任何特性行為邏輯**(反擊 / 警示 / 傳染 / 學習 / 順序重組 / 反情報 均不實作)
- **戰鬥介面不顯示「特性提示」UI 元素**(後續加入)
- **冒險手記不記錄特性資訊**(後續加入)
> **任何「第一期就加入特性」的提案先確認此 Scope 邊界**。第一期序章 5 個敵人**保持純被動**,維持教學基準的純粹。

## 一、設計動機

### P6 目前敵人在戰鬥中**完全被動**

| 既有元素 | 算「主動」嗎? |
|---------|-----------|
| 戰鬥狀態(先攻/標準/被襲) | 半 — 是戰前狀態 |
| 失敗代價(狼變精英) | 半 — 是失敗後的後續 |
| 跨場資訊持續 | ✗ — 那是玩家情報 |
| 戰中反應 | **✗ 完全沒有** |

### 引入主動行為的目的

| 目標 | 預期效果 |
|------|--------|
| **戰鬥緊張感** | 有「時間壓力 / 威脅倒數」 |
| **敵人個性化** | 每隻敵人有「會做什麼」,不只「需求數值」 |
| **策略張力** | 玩家面對「是否中斷敵人 X 動作」的決策 |
| **敘事豐富度** | 「準備施法」「召喚援軍」可帶入故事 |
| **跟雙列戰鬥互補** | 雙列 = 空間維度,主動行為 = **時間維度** |

→ 跟 [雙列戰鬥提案.md](雙列戰鬥提案.md) 互補,共同擴展 P6 的戰鬥變化空間。

## 二、跟 P6 既有設計的相容性原則

任何敵人主動行為**必須符合**以下原則,否則排除:

| 原則 | 含義 |
|------|------|
| **不違反 OHK** | 不能變成「多階段戰鬥」「逐步扣血」等違反「單一決定瞬間」 |
| **不違反單一檢查** | 結算瞬間仍是一次判定,不能拆成多次 |
| **不違反玩家是判斷者** | 行為條件**明確且事前可知**,不是隨機 |
| **不違反對稱即誠實** | 敵人行為有「明確規則」,玩家可學習對應 |
| **不違反失敗有有機後果** | 行為的「未阻止後果」要漸進有機,不是直接 GG |
| **不引入動作元素** | 不依賴反應速度 / 精確時機操作 |

## 三、採用的特性類型總覽

採用的特性分**三大類**:

| 類別 | 特性 | 觸發時機 | 主要設計 |
|------|------|--------|--------|
| **A 反擊型** | A1 消耗式 / A2 減效式 | 玩家本擊結算瞬間 | 對玩家本擊組成作出反應 |
| **B 威脅型** | 讀條倒數 / 操作倒數 / 條件升級 / 敵人學習 / 順序重組 / 反情報 | 戰中 / 戰後跨場 | 敵人有「會做的事」,玩家有預防/接受的選擇 |
| **C 傳染型** | C1 異常傳遞 / C2 環境惡化 / C3 援軍呼叫 | 擊殺後 / 連戰中 | 跨場景影響 |

→ **每個敵人模板最多 1-2 個特性**,避免認知超載。

## 四、反擊型(A)詳細設計

### A1 消耗式反擊

**機制**:標記特定卡類型,玩家使用該類型時**消耗額外資源**。

**限制**:
- 只能影響**低階基本卡**(資源池卡),**不能影響關鍵資源**(爆發 / 角色 / 情報卡)
- 額外消耗 = 該類型使用次數 +1 消耗(或本擊額外扣 1 張同類型)

**敘事**:玩家為突破該敵人「**多費了一點力氣或動作**」

**範例**:
```
敵人:精煉重甲衛兵
  特性 A1:「鎧甲反擊」
  
  規則:玩家使用衝擊類卡時,該卡使用次數額外 -1
       (例:打 1 張直擊,實際消耗 2 次使用次數)
  
  不影響:爆發卡、角色卡、情報卡、其他類型卡
  
  視覺暗示:鎧甲明顯加厚、磨損的衝擊痕跡
```

### A2 減效式反擊

**機制**:玩家打出的某類型卡**貢獻減少或無效**。

**程度梯度**:
- **輕度減效**:該類型卡貢獻 -1
- **中度減效**:該類型卡貢獻減半
- **完全無效**:該類型卡貢獻歸零(等同該敵人對該類型免疫)

**敘事**:敵人「**直接抵擋住動作**」

**範例**:
```
敵人:火耐性巨魔
  特性 A2:「火免疫」(完全無效)
  
  規則:玩家使用燃燒類卡時,該卡貢獻歸零
  
  視覺暗示:濕潤毛皮、火光下無焦痕
  
  玩家對應:不能用燃燒,必須用衝擊或穿刺
```

### A3 需求調整反擊(❌ **排除**)

> 違反 P6「結算瞬間單一」原則,本擊不應該逐個結算直到通過。

## 五、威脅型(B)詳細設計

### B1 操作倒數(非真實時間)

**機制**:**敵人在玩家本擊中操作次數累積到 N 時,進入升級狀態**。

**限制**:**不是真實時間倒數**,是**操作數倒數**。

**敘事**:敵人逐步察覺玩家動作,進入警戒/狂暴。

**範例**:
```
敵人:警戒型守衛
  特性 B1:「警覺累積」
  
  規則:玩家在本擊中 Place 第 5 張卡時,
        該敵人進入警戒狀態,需求 +2
  
  視覺暗示:敵人逐漸抬頭、武器舉起的動作
  
  玩家對應:精簡本擊(用少數高效卡)避免觸發
```

### B2 讀條倒數(限制版)

**這是修改過的版本**,跟原本的「真實時間壓力」不同。

#### 核心原則(嚴格遵守)

| 限制 | 為什麼 |
|------|------|
| **不能強制玩家什麼時候打出** | 違反「玩家自主節奏」原則 |
| **不能設定「讀條結束 = 遊戲結束」** | 違反「OHK 美學」儀式精神 |
| **必須在戰鬥開始時給予提示**(如「敵人在觀察你」) | 玩家事前可預期 |
| **後果只能是「需求微幅提高」或「精英化」** | 後果可控,不會直接 GG |

#### 機制

```
敵人開戰時:
  視覺暗示「敵人在觀察你...」
  讀條開始計時(僅作為視覺/敘事元素)
    ↓
  若玩家在合理時間內出擊:正常戰鬥
  若玩家拖延:
    讀條走完 → 敵人「反應過來」
      → 該敵人需求微幅提高(+1 或 +2)
      → 或進入精英化狀態(更致命但仍是同敵人)
    ↓
  讀條走完後 ≠ 強制結束本擊
  玩家仍可繼續思考、Place、Lock、結算
```

#### 敘事

「**敵人能看清你的動作了**」——不是時間壓力,是敵人能力的視覺化。

#### 範例

```
敵人:觀察型刺客
  特性 B2:「凝視」
  
  開戰提示:「他正在觀察你,等你出手前的破綻...」
  
  讀條規則:30 秒視覺讀條(僅暗示)
  
  讀條走完後:
    - 需求 +1(基本反應)
    - 或進入精英狀態(若多次玩家拖延)
  
  玩家對應:
    - 早點出手 → 避開反應
    - 投情報先揭露 → 確認後再出手
    - 仍可深思熟慮 → 接受小幅 +1 後果
```

#### 跟雙模式 UI 的關係

- **獵人模式**:讀條視覺顯示,但玩家可慢慢來
- **騎兵模式**:讀條視覺可能淡化(熟手不需要提示)

### B3 條件升級(精英化)

**機制**:敵人未在特定條件達成前被擊敗,則**升級為精英狀態**。

**敘事**:「**儀式完成、力量解放、變身**」

**範例**:
```
敵人:儀式法師
  特性 B3:「儀式」
  
  規則:該敵人若未在本場戰鬥被擊敗,
        進入儀式完成狀態(精英化)
  
  精英狀態:需求 +3,失敗即 GO(對應狼型精英邏輯)
  
  視覺暗示:法陣展開、咒語持續吟唱
  
  玩家對應:
    - 一次本擊解決(避免精英化)
    - 投資情報 + 高效卡集中
```

### B4 敵人學習機制 ⭐ **(新方向)**

**這是用戶提出的新變形**,值得重點開發。

#### 機制

```
連戰中:
  [前哨敵人 A] 玩家用某類型擊敗(例:衝擊)
    ↓
  [學習機制觸發]
    ↓
  [後續同類別敵人 B] 進入戰鬥時:
    「察覺玩家的衝擊偏好,衝擊需求 +2」
```

#### 詳細規則

| 規則 | 內容 |
|------|------|
| **觸發條件** | 「擊敗本場用最多次的類型」是該敵人偏好 |
| **學習程度** | 視敵人類別:狼型 +1 / 豹熊 +2 |
| **學習範圍** | **同連戰內後續同類別敵人**;連戰結束清除 |
| **可被反制** | 玩家**混合多類型擊敗前哨**(沒明顯偏好)→ 學習無效 |
| **可被揭露** | 試探卡 / 警示卡可揭露「該敵人是否會學習」 |

#### 設計優勢

- ✓ **強迫策略多樣化**(玩家不能整場用同一類型)
- ✓ **連戰真正成為策略單位**
- ✓ 對稱即誠實(玩家學敵人 / 敵人學玩家)
- ✓ 敘事意義強(訓練軍隊、進化生物)

#### 範例

```
敵人:訓練有素的軍隊
  特性 B4:「學習」
  
  規則:擊敗該敵人時,若玩家本擊有某類型 ≥ 3 張,
        本連戰後續同類別敵人該類型需求 +2
  
  視覺暗示:軍服整齊、姿態警戒
  
  玩家策略選擇:
    A. 用該類型快速擊敗 → 接受後續變難
    B. 用混合類型擊敗 → 學習無效,但本擊資源消耗多
    C. 投情報先揭露 → 精準規劃
```

### B5 順序重組 ⭐ **(新方向)**

**這也是用戶提出的新變形**。

#### 機制

連戰中,**前哨敵人的結果**會**重新調整後續敵人順序**(不改數值)。

```
連戰預設順序: [A] → [B] → [C] → [D] → [E (boss)]

情境 1:玩家完美擊敗 A
  → A「警報」傳出:[B] [C] [D] 中**強的提前**
  → 變為:[A] → [C] → [B] → [D] → [E]

情境 2:玩家失敗 A
  → A 逃跑:[B] [C] [D] 中**弱的提前**
  → 變為:[A] → [B] → [D] → [C] → [E]
```

#### 設計優勢

- ✓ 不改數值,符合 P6 平衡精神
- ✓ 強化「**連戰是策略單位**」概念
- ✓ 完美擊敗的「警報」概念(敵人警覺)
- ✓ 失敗的「資源消耗」概念(弱的提前)
- ✓ 對應 LoR 的「**順序決策**」概念

### B6 反情報 ⭐ **(新方向)**

**搭配 B5,作為「失敗有有機後果」的延伸**。

#### 機制

具有「**反情報特性**」的敵人,在玩家失敗時,**讓已揭露的情報失效**。

```
玩家投了「估量」揭露連戰中所有剩餘敵人需求
  ↓
玩家失敗前哨敵人 A(有反情報特性)
  ↓
A 逃跑,警告所有同夥
  ↓
**已揭露的情報失效**(後續敵人需求重新隱藏)
  ↓
玩家被迫重新投情報 OR 盲打
```

#### 限制

- **只有有反情報特性的敵人才會觸發**,不是所有敵人
- 失效**可逆**:玩家可再投情報重新揭露

#### 設計優勢

- ✓ 失敗的**真實代價**(情報失效)
- ✓ 對應「**敵人會反偵察**」的敘事
- ✓ 強化情報投資的真實重量

## 六、傳染型(C)詳細設計

### C1 異常傳遞(限制版)

**機制**:擊殺敵人後,玩家獲得負面狀態,**影響下場戰鬥**。

**限制版的兩種異常**:
1. **降低下一輪最大出卡數量**(例:下場戰鬥最多 4 張卡)
2. **限制某卡片類型出牌**(例:下場戰鬥不能用衝擊類)

**範例**:
```
敵人:詛咒術師
  特性 C1:「臨終詛咒」
  
  規則:擊殺後,玩家下場戰鬥不能 Lock 任何卡(only Place)
  
  視覺暗示:吟唱詛咒、施法手勢
  
  玩家對應:用情報投資緩衝、不要把重要 lock 留到下場
```

→ 異常**只影響下一場**,連戰結束/整備期清除。

### C2 環境惡化(待深入)

**方向確認,但具體環境效果需要進一步討論**。

可能的方向:
- **戰場著火**(燃燒效率變化)
- **下雨**(燃燒抑制 / 視覺受限)
- **黑暗**(情報效果減半 / 強制被襲)
- **崎嶇地形**(影響某些類型效率)

→ 需要單獨討論「**環境系統**」的具體實作。

### C3 援軍呼叫(限制版)

**機制**:不能多一個普通戰鬥(會跟失敗精英化衝突),改用兩種:

| 子類 | 機制 |
|------|------|
| **C3a 雙列觸發** | 該敵人未擊敗 = 下場變雙列戰鬥(援軍出現) |
| **C3b 強制精英化** | 該敵人未擊敗 = 該敵人變精英(類似狼型失敗,但是「條件性升級」)|

**範例**:
```
敵人:狼群首領
  特性 C3a:「召喚狼群」
  
  規則:未在 X 場內擊敗 = 下場變雙列戰鬥(本敵 + 援狼)
  
  視覺暗示:嚎叫、警戒族群

敵人:儀式法師
  特性 C3b:「儀式完成」
  
  規則:未在本場擊敗 = 升級為精英狀態
  
  跟 B3 條件升級重疊,可合併設計
```

→ **C3a 跟雙列戰鬥提案緊密結合**,**C3b 等同 B3 條件升級**。

## 七、排除的設計類型

| 類型 | 為什麼排除 |
|------|---------|

## 八、設計約束(防止失控)

### 約束 1:每個敵人模板最多 1-2 個特性

- 大部分敵人**無特性**(對應「視覺即敘事」基準)
- 特殊敵人**1-2 個特性**(差異化標誌)
- **絕不**讓敵人特性堆疊變試算表

### 約束 2:特性必須視覺呈現

| 層級 | 顯示內容 |
|------|--------|
| Layer 1(立即可見) | **特性存在的視覺暗示**(姿態、裝備、表情) |
| Layer 4(情報揭露) | **特性的精確規則** |
| 試探卡 binary | **「該敵人是否有特性 / 哪一類」** |

### 約束 3:特性必須有「對應方法」

- 玩家熟悉敵人 → 可學會避開 / 中斷 / 對應
- **不能設計「無法對抗的特性」**(否則違反元知識累積原則)
- 對抗方法用**既有卡組就能做到**,不需要特化卡

### 約束 4:特性的後果**有機漸進**

- 不能「未中斷 = 直接 GG」(除非該敵人本來就是豹/熊型)
- 後果應該是「**需求微幅提高**」「**情報失效**」「**精英化**」這類**可繼續玩**的狀態

### 約束 5:特性**獨立於失敗代價類別**

- 失敗代價依**敵人類別**(兔/狼/豹/熊)
- 特性是**額外的**戰術維度
- 兩者**不互相覆蓋**

---

## 九、跟既有 P6 設計的整合方式

### 跟「敵人四層資訊架構」整合

| 層 | 特性的對應 |
|---|--------|
| Layer 1(立即可見) | 特性的**存在**(視覺暗示) |
| Layer 2(視覺線索) | 特性類型的暗示(警戒姿態 = 警覺型;盔甲厚 = 鎧甲反擊型)|
| Layer 3(即時狀態) | 特性的**當前狀態**(已觸發 / 倒數中 / 已學習) |
| Layer 4(隱藏狀態) | 特性的**精確規則** |

### 跟「失敗代價系統」整合

特性 ≠ 失敗代價:
- 失敗代價依**類別**(兔/狼/豹/熊)
- 特性是**額外戰術維度**
- 例:**狼型 + 學習特性** = 狼型失敗代價(精英化) + 學習機制(後續敵人變難)

### 跟「Place/Lock 機制」整合

- 反擊型特性反應**卡牌組合 / 操作**
- 例:鎧甲反擊在玩家 Place 第 3 張衝擊卡時提示
- 玩家可**重新調整本擊組合**避開反擊

### 跟「情報系統」整合

- **警示卡**:可揭露「該敵人是否有特性 + 哪一類」
- **試探卡 binary**:「**該敵人是否會學習?**」「**該敵人是否會反情報?**」
- **估量**:在揭露需求時順帶提示「**該敵人有反情報特性,情報可能失效**」

### 跟「戰鬥紀錄(冒險手記)」整合

- 戰鬥紀錄頁面顯示「**敵人特性**」欄位
- 失敗的敵人留下「**特性未中斷的痕跡**」
- 玩家可在手記中看到「**這敵人會狂暴**」的標記

### 跟「雙列戰鬥」整合

- 雙列下,**各列敵人可獨立有特性**
- A 列敵人有「狂暴倒數」+ B 列敵人有「火耐性」
- 玩家分配卡時要同時考慮兩列特性

### 跟「連戰結構」整合

- B4 敵人學習 / B5 順序重組 / B6 反情報 都**作用於連戰層**
- 強化「**連戰是策略單位**」概念
- 對應 LoR 的「接待 / 司書狀態傳遞」精神

### 跟「雙模式 UI」整合

| UI 元素 | 獵人模式 | 騎兵模式 |
|--------|--------|--------|
| 特性視覺暗示 | 明確顯示 | 仍顯示(基本資訊)|
| 特性詳細規則 | 揭露後完整顯示 | 揭露後顯示 |
| 讀條倒數 | 完整視覺呈現 | 可能淡化 |
| 提示文字 | 詳細(「他在觀察你...」) | 精簡 |

---

## 十、第一期 / 後續期數規劃

### 第一期(序章)

- **5 個原型敵人都不引入特性**
- 保持簡潔教學基準
- **資料結構預留** `traits: []` 空陣列
- 雙模式 UI 設計時**考慮特性顯示位置**(未來填充)

### 第二期之後

- **逐步引入特性**:
  - **戰役 2**:加入 1-2 個 B 系列(警示型 / 條件升級)
  - **戰役 3**:加入反擊型(A 系列)+ 敵人學習(B4)
  - **戰役 4+**:傳染型(C 系列)+ 順序重組(B5)+ 反情報(B6)
  - **戰役 5+**:複雜組合(2 個特性的精英敵人)

### 跨期累積目標

- 每場戰役**約 1-2 隻有特性的敵人**(對應「特殊敵人」設計)
- 大部分敵人仍是**純被動模板**(對應「視覺即敘事」基準)
