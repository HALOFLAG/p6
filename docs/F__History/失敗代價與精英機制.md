# 失敗代價與精英機制

> **狀態**: 歷史 / 已被取代
> **被誰取代**: [遊戲核心系統機制.md](../C__Implementation_benchmarks/遊戲核心系統機制.md) §5-§6(失敗代價 + 精英機制)+ [敵人設計原則.md](../B__Design_specifications/敵人設計原則.md) §1, §7(四類分類 + 設計合約);2026-05-12 重構

本文件記錄 P6 的**敵人類別系統 + 失敗代價系統 + 精英敵人機制**的最終設計(2026-05-11 確立)。

本文件**取代**以下舊設計:
- ❌ 精英化敵人 10% 隨機出現
- ❌ 精英化敵人有特殊修飾池(重防護盾/韌性/致命要害)
- ❌ 失敗 = 敵人逃脫消失
- ❌ 三類敵人(普通/精英/Boss)的舊分類

> 本機制是「玩家失敗造成精英化」的有機後果系統,搭配「四類敵人的差異化失敗代價」,符合 [設計概念.md](設計概念.md) 第十章「失敗的結果是有機的——不是設計師強加的限制,而是玩家自己決策造成的後果」。

---

## 一、核心原則

兩個原則,所有機制為它們服務:

1. **玩家的失敗要有代價**(可感知、漸進、有後續)
2. **關鍵敵人不能被跳過**(預設精英 + 劇情敵人 + Boss 戰役節奏不被破壞)

### 一句話總結

> **並非所有敵人都會殺你——但會記恨的敵人會以精英形態回歸,而預設精英與 Boss 失敗即終戰。**

---

## 二、敵人四類分類

| 類別 | 範例 | 失敗代價 | 敘事邏輯 |
|------|------|---------|---------|
| **A. 兔子(無威脅普通)** | 哨兵 | 卡牌消耗;敵人逃走,**不再回來** | 「沒有殺意」的對手 |
| **B. 狼(威脅普通)** | 巡守、重甲衛兵 | 卡牌消耗 + 變精英回歸 + 精英失敗 = GO | 「會記恨並學習」的對手 |
| **C. 豹(預設精英)** | 伏擊者 | **失敗即 GAME OVER**(本來就是精英,沒有「曾經普通」階段) | 「本來就要殺你」的對手 |
| **D. 熊(Boss / 劇情敵人)** | 領主 | **失敗即 GAME OVER** | 戰役節奏的關鍵 |

### 為什麼是四類

從敘事邏輯反推:**並非所有敵人都對玩家構成致命威脅**。

- 一隻兔子的失敗不應該變強來找你算帳——它就只是逃走
- 一隻狼的失敗會學到經驗——它記住你,下次帶著殺意回來
- 一隻豹本來就是來殺你的——沒有第二次機會
- 一隻熊是無庸置疑的危險——必須集合一切才能生還

這個分類**讓敘事邏輯與機制完美一致**,且解決:
- 「為什麼這隻兔子失敗會變精英來殺我」的違和感
- 「為什麼這隻刺客可以給我第二次機會」的違和感
- 「為什麼最終 boss 失敗才會 GO」的不一致感

## 三、各類別的失敗代價詳細

### A. 兔子型失敗(無精英化機制)

```
觸發:玩家對兔子型敵人 commit 後 Underkill
處理:
  1. 投入的卡牌依然消耗
  2. 兔子逃走,從本連戰移除
  3. **不會精英化回歸**
  4. 不會替換下連戰敵人
```

**設計意圖**:給玩家「廉價失敗的學習場」。兔子是新手友善設計,失敗代價輕但不無視(卡牌仍消耗)。

**注意**:兔子型不是「廢敵人」——它仍提供戰鬥練習、資源消耗壓力,且擊敗它仍有節奏意義。

---

### B. 狼型失敗(三層代價)

#### 第一層:連戰內第 1 次失敗

```
觸發:玩家對狼型敵人 commit 後 Underkill
處理:
  1. 投入的卡牌依然消耗
  2. 該敵人標記為「精英狀態」
  3. 排到該連戰末段(Boss 之前)
  4. 連戰場次 +1
玩家會在該連戰結束前再次面對它(精英化)
```

#### 第二層:連戰內第 2+ 次失敗(狼型)

```
觸發:該連戰已有 1 次失敗精英排在末段,玩家再次對狼型失敗
處理:
  1. 投入的卡牌依然消耗
  2. 該敵人「移出該連戰」進入「未解決池」
  3. 連戰繼續進行
連戰結束後玩家會面對「追擊選擇」
```

#### 第三層:連戰結束 → 追擊機制(只針對狼型未解決池)

```
觸發:連戰結束時未解決池有狼型敵人
玩家選擇:
  追擊一隻:
    - 額外戰鬥(該精英敵人,被襲狀態)
    - 打贏 → 從未解決池移除 → 可選繼續追擊或結束
    - 打輸 → GAME OVER
  結束追擊:
    - 未追擊的敵人保留在未解決池
    - 進入整備期
    - 未解決池中的敵人會「替換下一連戰的普通敵人」(見第五章)
```

### 追擊期間規則

- 追擊發生在**連戰結束之後、整備期之前**
- 追擊敵人狀態 = **被襲**(它們「埋伏」等你)
- 卡牌使用照常(用該連戰殘餘資源)
- 整備期回補(若採用)在所有追擊結束後才觸發
- 一次只追擊一隻,打完可選擇繼續或停止

---

### C. 豹型失敗(預設精英)

```
觸發:玩家對豹型敵人 commit 後 Underkill
處理:**GAME OVER**

豹本來就是精英級對手,沒有「第二次機會」。
玩家必須一次擊敗(類似 OHK Boss 的決鬥精神)。
```

**設計意圖**:給玩家「**這次不能失敗**」的高張力時刻。豹的視覺暗示要極明顯(銳利、致命氣質),讓玩家從外觀就讀出威脅等級。

**何時遇到豹型**:連戰中段或末段(警示玩家「真正的考驗來了」)。

---

### D. 熊型失敗(Boss / 劇情敵人)

```
觸發:玩家對熊型敵人 commit 後 Underkill
處理:**GAME OVER**

熊是 Boss 級對手,代表戰役節奏的關鍵時刻。
失敗即終戰,符合 OHK「最終一擊」的儀式精神。
```

**設計意圖**:作為連戰節奏的高潮點(每連戰最後一隻 + 最終戰領主)。

---

## 四、精英敵人的兩種來源

精英化現在有兩種來源,機制上失敗都 = GO,但敘事上不同:

| 來源 | 範例 | 視覺 | 玩家心理 |
|------|------|------|---------|
| **狼變的精英**(失敗造成) | 失敗的巡守、失敗的重甲衛兵 | 跟原本一樣 + UI 標籤「精英化 ⚠」 | 「我記得它,我有戰鬥紀錄資料」 |
| **豹型預設精英**(本來就在) | 伏擊者 | 物種本身的精英級視覺 | 「我第一次遇到,但它一招斃命」 |

**狼精英**:給玩家「**回憶上次的清算機會**」(經驗+紀錄優勢)
**豹精英**:給玩家「**新挑戰的瞬間壓力**」(純粹判讀挑戰)

兩者豐富體驗的不同層面。

## 五、未追擊敵人的處理:替換下連戰(只針對狼型)

```
進入下連戰前:
  1. UI 明確告知:「⚠ N 隻精英來自上連戰」
  2. 未解決池中的狼型精英替換下連戰的普通敵人
  3. 替換從**連戰最前面**開始
  4. 關鍵敵人(豹/熊位置)保留,不被替換
```

### 替換規則細節

假設下連戰預設 5 隻:`[A] [B] [C] [D] [E(熊 boss)]`(其中 D 可能是豹,E 是熊)。

- 未解決池 1 隻狼精英 X → 下連戰變 `[X 精英] [B] [C] [D] [E]`
- 未解決池 2 隻狼精英 X、Y → 下連戰變 `[X 精英] [Y 精英] [C] [D] [E]`
- **豹型(D)與熊型(E)位置永遠保留**
- 替換永遠從連戰**前面**開始,只替換兔子型或狼型位置

### 為什麼替換從前面開始

- 玩家進入連戰立即面對失敗的代價(警示感強)
- 豹型/熊型保留位置,戰役節奏的「重要時刻」維持
- 「上連戰失敗 = 下連戰開頭就吃苦頭」邏輯直接

## 六、Game Over 條件

任何精英敵人 + 豹型 + 熊型失敗 = GAME OVER:

| 情境 | 失敗結果 |
|------|---------|
| 連戰中面對狼型精英(失敗回歸的)失敗 | GAME OVER |
| 追擊狼型時失敗 | GAME OVER |
| 下連戰面對替換進來的狼型精英失敗 | GAME OVER |
| 任何豹型(預設精英)失敗 | GAME OVER |
| 任何熊型(Boss / 劇情敵人)失敗 | GAME OVER |

**唯一不導致 GO 的失敗**:兔子型失敗、狼型第一次失敗

## 七、戰役流程範例

### 範例 A:順利通關(0 失敗)

```
連戰 1(假設 5 隻:1 兔 + 3 狼 + 1 熊):
  哨兵(兔) → 巡守(狼) → 重甲(狼) → 巡守(狼) → 領主(熊)
  全部 ✓ OHK
[整備期]
連戰 2(假設 5 隻:1 兔 + 2 狼 + 1 豹 + 1 熊):
  ...

總戰鬥:依連戰配置而定
```

### 範例 B:連戰 1 兔子失敗 + 1 狼失敗

```
連戰 1: 5 隻
  1. 哨兵(兔)→ ✗ Underkill
     → 哨兵逃走,**不變精英**,從連戰移除
     → 連戰場次變 4
  2. 巡守(狼)→ ✓
  3. 重甲(狼)→ ✗ Underkill
     → 重甲變精英排末
     → 連戰場次變 5
  4. 巡守(狼)→ ✓
  5. 領主(熊)→ ✓ OHK
  6. 重甲精英 → ✓ OHK

總戰鬥:6 場(原本 5 隻 - 1 兔子失敗 + 1 重甲精英 = 5,加上原本順序執行)

實際是:
  1. 哨兵(兔) - 失敗逃走
  2. 巡守 - 成功
  3. 重甲(狼) - 失敗 → 精英排末
  4. 巡守 - 成功
  5. 領主(熊) - 成功
  6. 重甲精英 - 成功
共 6 場戰鬥(其中第 1 場兔子失敗仍佔 1 場)
```

### 範例 C:遇到豹型,玩家失敗 = GAME OVER

```
連戰 2: 5 隻
  1. 巡守(狼)→ ✓
  2. 重甲(狼)→ ✓
  3. 伏擊者(豹)→ ✗ Underkill
     → GAME OVER
     ┃
     ┗━ 戰役終結
```

### 範例 D:狼型 2 次失敗 + 玩家追擊

```
連戰 1: 5 隻
  1. 哨兵(兔)→ ✓
  2. 巡守(狼)→ ✗ 失敗 1 → 變精英排末
  3. 重甲(狼)→ ✗ 失敗 2 → 移出進未解決池
  4. 巡守 → ✓
  5. 領主(熊)→ ✓
  6. 巡守精英 → ✓
  
[追擊階段]
  未解決池:{重甲}
  玩家選追擊
  重甲精英(被襲)→ ✓
  玩家選結束追擊

[整備期]
```

## 八、視覺與 UI 要求

### 1. 敵人類別的視覺辨識

詳見 [視覺線索詞彙表.md](視覺線索詞彙表.md) D 章節「敵人類別視覺暗示」。

簡述:
- **兔子型**:小體型、無威脅姿態、武器簡陋或無
- **狼型**:中體型、武器明顯、警戒姿態
- **豹型**:銳利外觀、致命氣質、襲擊預備姿態
- **熊型**:壓倒性體型、Boss 級裝備

**完全靠視覺辨識,不需 UI 標籤標示類別**(類似物種辨識)。

### 2. 精英敵人的狀態暗示(只針對狼變的精英)

玩家進入戰鬥時,UI 標示「**精英化 ⚠**」+「**這是你上次面對過的敵人**」:
- 不改變敵人本身外觀
- 透過 UI 標籤、戰鬥開場特效、音樂等暗示
- 戰鬥紀錄可調出上次的數據

### 3. 豹型敵人的「警告感」

豹型沒有額外 UI 標籤(因為本來就有精英級視覺),但:
- 戰鬥開場可能有特殊音效強化「警告」
- NPC 對話強調「這隻不能失敗」

### 4. 未解決池顯示

連戰結束時的追擊選擇介面:
```
┌─────────────────────────────────┐
│ 未解決池                        │
│ ┌─────────┐ ┌─────────┐       │
│ │ 重甲精英│ │ 巡守精英│       │
│ └─────────┘ └─────────┘       │
│                                 │
│ [追擊 重甲] [追擊 巡守] [結束]  │
└─────────────────────────────────┘
```

### 5. 下連戰開始的預警

進入下連戰前的轉場畫面:
```
┌─────────────────────────────────┐
│ 連戰 2                          │
│                                 │
│ ⚠ 1 隻精英來自上連戰將在開頭出現│
│                                 │
│ [開始連戰]                      │
└─────────────────────────────────┘
```

### 6. 戰鬥紀錄的「失敗回顧」

狼型精英敵人戰鬥開始前(可選):
- 顯示上次面對該敵人時的本擊組合
- 顯示上次的結果(差多少達標)
- 讓玩家可以回憶並規劃對策

## 九、與其他系統的關係

### 與情報卡(警示)— 功能重新設計

**舊版**:揭露下一場/剩餘敵人「是否精英化」(現已不適用)

**新版**:揭露**敵人類別**
- 不 lock:揭露此連戰所有剩餘敵人的類別(兔子/狼/豹/熊)
- Lock:揭露當前敵人的類別

這個資訊**對玩家極有用**——「下一場是豹 = 失敗即 GO,我要謹慎」是真實的戰略資訊。

### 與情報卡(全知之眼)— 揭露範圍更新

**舊版**:揭露剩餘所有敵人包含「精英化具體修飾」(現已不適用)

**新版**:揭露剩餘所有敵人完整資訊,包含:
- 弱點類型
- 完整需求表
- 戰鬥狀態 instance
- **敵人類別**(兔子/狼/豹/熊)

### 與情報卡(試探擊)— Binary 揭露重新定義

**舊版**:揭露當前敵人「是否精英化」yes/no

**新版**:揭露當前敵人「**失敗是否會 GAME OVER**」yes/no
- 對兔子/狼(第一次):NO
- 對豹/熊/狼(已是精英狀態):YES

這個 binary 對玩家極關鍵——「我可以失敗嗎?」是最直接的判斷依據。

### 與戰鬥紀錄系統

**戰鬥紀錄變成必要功能**(原本是「玩家方便用」)。理由:
- 玩家必須能查閱失敗的狼型敵人資訊
- 狼型精英敵人對戰前,UI 可從戰鬥紀錄調出上次的數據
- 玩家累積的元知識需要紀錄系統支持

### 與整備期

- 整備期回補(若採用)在追擊**之後**才觸發
- 玩家在追擊期間用「該連戰殘餘資源」
- 這強化了「立即追擊 = 用最少資源解決」vs「延後 = 用整備期回補但下連戰更難」的權衡

### 與戰役章節結構

- 戰役長度可控(每連戰最多 +1 狼精英 + 追擊場次)
- 失敗多會延長戰役,但有 GAME OVER 上限
- 30 分鐘戰役目標可能浮動(0-2 失敗的玩家會打 30-40 分鐘)

## 十、跟舊設計的對照

| 面向 | 舊設計 | 新機制 |
|------|-------|--------|
| 敵人分類 | 普通 / 精英 / Boss(3 類)| 兔子 / 狼 / 豹 / 熊(4 類) |
| 精英化來源 | 10% 隨機 | 狼失敗造成 + 豹本來就是 |
| 精英化外觀 | 暗紅光暈 + 徽章 + 環境扭曲 | 跟原本敵人完全相同(只有 UI 標籤,狼變的) |
| 精英化修飾 | 三選一(重防護盾/韌性/致命要害) | 無修飾(外觀數值不變) |
| 失敗代價 | 統一「卡牌消耗 + 敵人逃脫消失」 | **依類別分**:兔子(逃走)、狼(精英回歸)、豹/熊(GO) |
| 戰役失敗條件 | 隱性「資源耗盡」 | 明確「豹/熊/精英失敗 = GAME OVER」 |
| 警示卡功能 | 揭露隨機事件 | 揭露敵人類別(極有用) |
| 失敗的敘事邏輯 | 統一一致(全部都會精英化)| 依敵人個性差異化(兔子不會記恨,狼會) |

## 十一、待處理 / 後續確認

採用此機制後需要處理的:
