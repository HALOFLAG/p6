# 第一部分:核心願景

## 一、遊戲核心定位

本作是一款**單人電子卡牌戰術遊戲**,核心玩法以「一擊必殺(OHK, One-Hit Kill)」為設計主軸。玩家在連續戰鬥中,將有限的卡牌資源精準分配給每一個敵人,以達成連續 OHK 為目標。

遊戲的本質不是「卡牌構築」,也不是「位置戰術」,而是:**用卡牌作為資源語言的戰術規劃遊戲**。卡牌是表達資源類型與數量的介面,戰術規劃是核心決策結構。

### 玩家在做的事
- 蒐集情報、分析敵人
- 從有限資源中組合出本擊
- 在不完整資訊下做出不可逆的承諾
- 透過理解累積,逐步精通系統

### 玩家不在做的事
- 在牌組構築中追求運氣
- 在棋盤上移動單位
- 在裝備樹中升級角色
- 在商店中堆積資源

---

## 二、OHK 美學的五大原則

OHK 不是機制裝飾,而是一種設計哲學。以下五個原則互相鎖死,缺一不可:

### 原則一:減法即內容 (Subtraction is content)
不是「先做出完整遊戲再考慮拿掉什麼」,而是「從一無所有開始,每加一個元素都要證明它的必要性」。沒有經驗值、沒有商店、沒有裝備,是設計選擇而非預算限制。

### 原則二:對稱即誠實 (Symmetry is honesty)
玩家與敵人受同樣的規則限制——一擊定生死,沒有單方面的血量優勢。傳統 RPG 的不對稱(玩家承受幾十下、敵人扛不住幾下)創造的是耐久戰,OHK 的對稱創造的是純粹的一決勝負。

### 原則三:瞬間即儀式 (The moment is the ritual)
沒有累積成長作為獎勵,那一擊本身就是獎勵。整個遊戲的所有設計——音效、畫面、節奏——都是為了讓那個瞬間被放大。

### 原則四:理解即進度 (Understanding is progression)
沒有等級、沒有裝備、沒有解鎖技能,玩家「變強」的唯一方式是更深入理解系統與敵人。教學設計很重要,玩家需要能感受到「我變懂了」的回饋。

### 原則五:變化是隱性支柱 (Variation is the hidden pillar)
前四個原則都依賴一個前提:**內容必須真正多變**。沒有真正多變的內容,前面四個原則都會在 50 小時遊玩時間內崩塌:減法會無聊、對稱會被算盡、瞬間會變例行公事、理解會封頂。

這就是為什麼遊戲需要在三個層面持續創造變化:
- 敵人組合(input randomness 的核心)
- 卡牌池(每局不同的可用工具)
- 弱點配置(同類型敵人也有微差)

如果這三層的變化夠真實,玩家不會「打通關後」感到無聊——因為沒有「通關」這件事,只有「這局打贏了」與「這局沒打贏」。

### 五原則的相互支撐
- 減法讓瞬間有空間被放大
- 對稱讓理解成為唯一變數
- 變化讓理解永遠不會封頂
- 任何一個原則被妥協,整個美學就鬆動

## 四、與類似遊戲的本質差異

### 與 One Turn Kill 的差異
- One Turn Kill 的核心是「**牌組內部的牌序解謎**」——湊出 Combo、控制抽牌
- 本作的核心是「**敵人之間的資源分配**」——同樣的資源池,怎麼最佳配給多個目標

### 與 Slay the Spire 的差異
- Slay the Spire 用隨機抽牌創造張力,運氣是骨幹
- 本作完全沒有抽牌,所有卡牌資訊對玩家公開

### 與 Mewgenics 的差異
- Mewgenics 是「位置戰術 + 環境互動」的格網戰術遊戲,戰鬥有空間維度
- 本作沒有空間維度,戰鬥是抽象的資源分配判定
- Mewgenics 是戰鬥內部的複雜度(一場仗很多細節),本作是戰鬥之間的複雜度(每場仗很快結束,但要連續打、要分配跨場資源)

### 與 Titan Souls 的關係
- Titan Souls 證明 OHK 美學作為核心設計的可行性,但它是動作遊戲
- 本作借鏡其哲學(極簡、儀式化、拒絕安全網),但機制是規劃導向而非執行導向
- Titan Souls 的「死亡即情報」結構不適用,本作必須提供等價的發現感載體

### 與 Pirates Outlaws Arena 的關係
- Pirates Outlaws Arena 證明「**連續戰鬥 + 跨場資源管理**」這個結構本身有觀眾
- 該模式的精簡(去掉主線探索)驗證了「集中於戰鬥」的可行性
- 但該模式仍是 Roguelite Deckbuilder DNA(隨機抽牌、即時牌組構築、HP 系統)
- 本作在此基礎上**進一步淨化**:沒有 HP、沒有抽牌、沒有即時構築
- 可借鏡的具體細節:Reward Panel 的精簡決策結構(3 選 1)、Champion 機制的節奏感

### 設計光譜上的進化關係

```
Pirates Outlaws 主線
    ↓ 拿掉探索、地圖、市場
Pirates Outlaws Arena
    ↓ 再拿掉抽牌、HP、即時構築、RNG
你的遊戲
```

每一層的「拿掉」都讓核心體驗更聚焦。**減法不是設計的失敗,而是設計的成熟**。

# 第二部分:戰役架構

## 五、戰役整體架構

每局遊戲是一場「戰役」,由多次循環組成,最終以一場綜合性連戰結束:

```
[整備 1] → [連戰 1] → [整備 2] → [連戰 2] → ... → [最終連戰]
```

### 整備階段
- 蒐集情報(多種準確度的選項)
- 獲得少量資源(回補工具牌使用次數)
- 規劃下一場連戰
- 結構建議:採用 Pirates Outlaws Arena 的精簡決策模式(例如 3 選 1)

### 連戰階段
- 多場單次 commit 戰鬥
- 驗證情報、消耗資源
- 留下可查閱的戰鬥紀錄

### 最終連戰
**必須是質變而非量變**。應該是「集大成驗證」而非「比較難的連戰 N」。

設計選項:
- **集大成驗證**:所有前面遇過的敵人類型都出現一次
- **無情報啟動**:開戰時所有敵人情報都模糊,仰賴玩家累積的元知識
- **多階段轉化**:最終 Boss 在被 OHK 後變形成新形態,需求表與弱點全部重置

---

## 六、三條貫穿全局的「流」

### 情報流
整備期蒐集 → 戰鬥期驗證 → 紀錄累積成知識,每輪情報精度提升

### 資源流
整備期少量補給 → 戰鬥期消耗 → 永遠處於勉強夠用的狀態

### 紀錄流
- 每場戰鬥的情報、資源、卡牌使用自動歸檔
- 作為玩家的研究素材,**不作為遊戲機制**
- 不要做成成就機制(集滿 X 解鎖 Y),那會讓玩家為收集而戰鬥
- 紀錄是讓玩家「看見自己的成長」的鏡子

## 七、隨機性哲學

### 核心分野
- **輸入隨機**(玩家決策前的變化):擁抱
- **輸出隨機**(玩家決策後的變化):拒絕

### 具體應用
| 隨機面向 | 評斷 | 說明 |
|---------|------|------|
| 卡牌獲得 | ✓ 適合 | 每局卡池不同,但讓玩家看見並選擇,不做盲抽 |
| 敵人組合 | ✓ 必須 | 變化是替代「死亡學習」的關鍵變數,但要有設計約束 |
| 敵人屬性微差 | ✓ 推薦 | 同類敵人需求表的數字在已知範圍內浮動,讓元知識仍有用 |
| 卡牌使用結果 | ✗ 禁止 | 機率性成敗會直接摧毀美學 |

### 設計哲學
本作應該是「**完全確定但極度多樣**」的——所有結果由玩家輸入決定,但組合空間極大。Into the Breach 是這個哲學的典範。

### 重要分辨
「不確定」不等於「隨機」。
- 「需求表已固定,玩家不知道精確數字」→ Hidden state(好設計)
- 「需求表是 X,但玩家攻擊有 20% miss」→ Output randomness(壞設計)

外觀都是「不確定」,本質完全相反。

---

## 八、情報管道

### 主要管道
1. **戰鬥試探**:直接出手後從結果反推
2. **主動揭露**:花費洞察類資源換取確定資訊
3. **元記憶**:玩家跨局累積的敵人知識

### 輔助管道
4. **行為訊號**:敵人外觀、姿態、動畫透露狀態
5. **推論與比較**:從場景、隊形、已死敵人反推未知
6. **不可靠情報**:便宜但有誤差的情報

### 設計原則
情報管道應分**主軸與點綴**。主軸要簡單到玩家熟練操作,點綴要稀有到玩家會記得它出現的關卡。建議:核心 3 個管道(戰鬥、主動揭露、行為訊號) + 1–2 個情境管道。

---

## 九、玩家決策迴圈

```
情報蒐集 → 分析推算 → 投入分配 → 結算驗證 → 反饋學習
   ↑                                              ↓
   └──────────────────────────────────────────────┘
```

### 各階段的張力
| 階段 | 玩家在做的事 | 張力來源 |
|------|------------|----------|
| 情報 | 投入洞察揭露 vs 賭直接出手 | 資訊不完整 |
| 分析 | 推算各敵人的最低資源需求 | 隱藏修正讓估算永遠帶誤差 |
| 投入 | 切割卡牌池給多個敵人 | 給太少會失敗,給太多後續不夠 |
| 結算 | OHK 達成或承擔失敗代價 | 失敗代價要痛但不致命 |

### 核心
玩家不是在解「正確答案題」,而是在做「**不完整資訊下的最佳投資判斷**」。沒有不確定就退化成計算機,沒有承諾代價就退化成隨便填空。

# 第三部分:機制原則

## 十、血量、失敗與戰役結束的設計觀

### 沒有傳統 HP 系統
本作不存在傳統血量,只有兩種狀態載體:
- **敵人**:需求表(各類型需要多少次數)。不是會慢慢被削減的血條,只有「任一條件達標 / 都沒達標」兩種狀態
- **玩家**:卡牌池。會隨戰役消耗,但不是「HP 條」的形式

這已經是 OHK 美學最純粹的形式——**用「資源狀態」取代「血量狀態」**。再加任何 HP 系統都是退步:
- 加敵人血條 → 違反「對稱即誠實」(敵人慢慢死、玩家一擊死)
- 加玩家血條 → 創造額外的失敗向度,跟「卡牌消耗」競爭注意力
- 加雙方對稱的一擊死 HP(Titan Souls 模式)→ 在規劃類遊戲裡不適用

**乾淨的設計就是:敵人是需求表,玩家是卡池**。

### 沒有顯性的失敗上限
不需要「Lives 系統」或「Game Over 次數」,**戰役結構本身已經提供隱性上限**:

機制如下(2026-05-11 重設):
- **依敵人類別差異化失敗代價**(兔子 / 狼 / 豹 / 熊)
- 兔子型失敗 = 卡牌消耗,敵人逃走
- 狼型失敗 = 卡牌消耗 + 變精英回歸(累積會進未解決池)
- 豹型 / 熊型失敗 = GAME OVER
- 精英敵人(狼變的)失敗 = GAME OVER
- 玩家不是「死了 N 次後 Game Over」,而是「精英敵人擊不倒」

詳見 [失敗代價與精英機制.md](失敗代價與精英機制.md)。

### 為什麼這種隱性上限更優雅
- **失敗的代價是漸進的、可感知的**,依敵人類別差異化,不是「踩線就死」的二元判定
- **結果是有機的**——不是設計師強加的限制,而是玩家自己決策造成的後果
- **不會有「我還剩 1 命」的焦慮感**,取而代之的是「精英化會回來找我」的具體後續
- **敘事邏輯一致**——兔子不會記恨、狼會記仇、豹本來就要殺你、熊是必殺局

### 失敗的具體代價(依敵人類別)

| 類別 | 失敗代價 |
|------|---------|
| 兔子(無威脅普通) | 卡牌消耗,敵人逃走不再回來 |
| 狼(威脅普通) | 卡牌消耗 + 變精英回歸;累積會進未解決池;精英失敗 = GO |
| 豹(預設精英) | GAME OVER |
| 熊(Boss / 劇情敵人) | GAME OVER |

## 十一、單一檢查原則

> **每場戰鬥的勝敗判定應該收斂到一個結算瞬間,玩家在 commit 前面對的是「選哪條路」的單一決策,而非「同時滿足多個獨立條件」。**

### 為什麼這個原則重要
玩家在戰鬥中不應該需要同時驗證多個獨立成功條件。如果某個機制會引入「獨立的失敗條件」(玩家必須 A AND B 都成立才能贏),它就違反了這個原則,應該重新設計成「**多條件 OR 結構**」(任一達標即贏)的形式。

### 具體應用:類型計數結算模型

採用「**類型計數**」設計:

- 每個敵人有一張**需求表**:每個傷害類型需要多少「次數」才能擊殺
- 加上一個「**混合任意需求**」:不限類型只比總次數
- 玩家本擊只要在任一條件達標就 OHK
- 所有條件指向同一個結算瞬間

具體舉例:
- 敵人:衝擊需求 2 / 穿刺需求 5 / 燃燒需求 6 / 混合需求 7
- 玩家 commit 2 張衝擊卡 → 衝擊計數 2 ≥ 2 → ✓ OHK
- 玩家 commit 5 張穿刺卡 → 穿刺計數 5 ≥ 5 → ✓ OHK(較貴)
- 玩家 commit 7 張任意混合卡 → 混合計數 7 ≥ 7 → ✓ OHK(很貴,代表「不知弱點時的保險路徑」)
- 玩家 commit 1 衝擊 + 3 雜類 → 各條件都不達 → ✗ Underkill

### 為什麼這個解法最好
- 簡單(玩家不需算乘法,純加總比對)
- 「對應弱點」價值清晰可見(衝擊只要 2 vs 穿刺要 5 = 弱點顯而易見)
- **勇敢直打仍可行**(走混合任意路徑,代價高但能成功)
- **謹慎優化也可行**(揭露弱點 → 走最低需求路徑)
- 沒有「對不起,你必須有衝擊類卡才能繼續」的鎖鑰問題(混合任意永遠是備援)
- 結算瞬間直觀:玩家看到「填滿哪個需求 bar」就立即知道結果

### 平衡原則
**混合任意需求 ≈ 最強弱點需求 × 3-4**

這樣才能避免玩家靠「全押多種類型」(hedge)走捷徑,逼迫玩家盡量找出弱點。

### 規則應用範例
未來若想加入「敵人有護盾,先破護盾再攻擊」這種機制,要先檢查是否違反原則:
- ❌ 兩個獨立階段(先破盾才能攻擊)→ 違反
- ✓ 護盾把某類型需求 +N 次 → 符合(單一檢查,護盾是調整需求,不是新階段)

## 十二、呈現層原則

> **系統內部所有運算都是精確的、確定的,但呈現給玩家的資訊應該是定性的、層級化的。玩家做的是判斷,不是計算。**

### 核心轉變:誰是計算者
這個原則決定了一個根本性問題:「誰是計算者?」

- **傳統卡牌遊戲**:玩家是計算者,系統只負責驗證。玩家在腦中跑試算表:「3+2+1=6,需求 5, 過了」——遊戲變成數學題。
- **本作的選擇**:**系統是計算者,玩家是判斷者**。玩家用類型概念思考:「這敵人看起來怕衝擊,我有兩張衝擊就夠了——但要不要再加一張保險?」

### 具體呈現方式

**卡牌的呈現**(類型+次數):
- 顯示:類型(衝擊類)、次數貢獻(★★ = +2 衝擊)、定性效果描述
- 不顯示:精確修飾邏輯與系統內部機制細節

**敵人的呈現**:
- 顯示:強度層級(中等強度的目標)、外觀視覺線索、行為訊號
- 不顯示(預設):各類型的精確需求次數

**情報牌的功能轉變**:
情報牌的價值在於**揭露需求表**——「把弱點範圍縮窄為具體類型」「把模糊強度轉換為精確需求次數」。情報投資的價值在於從定性轉換為定量。

### 為什麼這個選擇重要
- **避免遊戲退化成試算表**——玩家用直覺與經驗判斷,不是用計算器
- **元知識的累積感更明顯**——老玩家對「這種敵人 + 這種卡組合 = 這種結果」的直覺更敏銳
- **失敗的學習效果更好**——戰後對話會傳達「方向對 / 對應錯 / 量不足」的份量感讓玩家校準直覺
- **「猜弱點」機制更有意義**——玩家在讀取遊戲世界,不是讀取試算表

### 風險與應對
- **新手學習門檻較高** → 戰後對話提供定性回饋(「太弱 / 剛好 / 過量」),教學期可顯性化次數
- **平衡負擔轉移到設計者** → 需要更多 playtesting,確保各層級的差距合理
- **某些玩家會反感** → 這是強形狀設計的代價,會篩選出對的玩家

### 與其他原則的呼應
- 跟「單一檢查原則」是同一血脈——**用呈現層的克制,強迫玩家進入正確的思考模式**
- 強化了「理解即進度」原則——理解的不是數字,是系統的脾性

# 第四部分:警惕事項

## 十三、應當避免的失敗模式

從 Pirates Outlaws Arena 的觀察整理出三個典型失敗模式,本作的設計天然防禦這些問題,但**錯誤的實作會讓你掉進同樣的坑**。

### 失敗模式一:遊戲被解出來
**症狀**:固定內容 + 玩家適應 = 玩家收斂到單一最佳循環,後續所有局都重複
**本作的防禦**:輸入隨機 + 隱藏狀態(敵人組合每局不同,屬性微差,情報需主動投資揭露)

**警惕的變形**:「假隨機」實際上是固定內容
- 如果「隨機生成」實際上只有 5–6 種模板在重複,玩家還是會找出每種模板的最佳解
- **生成的真正多樣性比表面多樣性更重要**——10 種敵人 × 真正自由組合 比 100 種預製組合更耐玩

### 失敗模式二:脆弱的鎖鑰設計
**症狀**:特定敵人必須用特定養成卡才能擊殺,沒有就卡關
**本作的防禦**:多條件 OR 結構(見「單一檢查原則」),沒有理想類型卡仍可走「混合任意」備援路徑

**警惕的變形**:「弱點類型」變成「弱點特定卡」
- 如果某種類型只有 1–2 張可用卡,玩家沒選到時就回到鎖鑰問題
- **每個弱點類型至少要有 3–5 張不同設計的卡牌可用**,維持「有彈性的匹配」

### 失敗模式三:強迫試錯
**症狀**:無事前情報,只能戰鬥才能得知——首次必死,死後才能應對
**本作的防禦**:多重情報管道(外觀線索、主動揭露、推論比較、不可靠情報)

**警惕的變形**:「情報需要付太貴的代價」
- 如果主要管道(主動揭露)的成本太高,玩家會被迫盲打,等於回到問題三
- **情報投資的回報必須讓「謹慎玩法」可行**——不一定每場都要揭露,但需要時要能負擔得起

### 為什麼這三個問題會「聚集」
這三個問題不是巧合,而是**同一個設計取向**的三個必然後果。當設計取向是「**固定內容 + 玩家適應**」時:
- 固定內容 → 玩家可以背題 → 收斂到單一最佳解(問題一)
- 某些難關 → 設計師必須創造強制門檻 → 鎖鑰機制(問題二)
- 要讓首次嘗試有意義 → 但內容固定無法重玩學習 → 變成試錯 tax(問題三)

本作選擇了相反的設計取向:「**變化內容 + 玩家判斷**」,從根本上避開這三個問題。但要持續守住這個方向,需要在每個具體實作上保持警惕。

## 十四、設計上必須拒絕的東西

接受 OHK 美學意味著主動拒絕某些「對玩家友好」的設計:

### 機制層面
- ❌ 失敗保險(OHK 失敗後可以補刀)
- ❌ 逐步強化(打贏前面後面變容易)
- ❌ 戰鬥之間的資源恢復(超過上一輪消耗的補給量)
- ❌ 永久性升級或經驗值
- ❌ 裝備系統
- ❌ 機率性卡牌成敗
- ❌ 卡牌的盲抽
- ❌ 顯性的 HP 系統(玩家或敵人)
- ❌ 顯性的失敗上限(Lives 系統)

### 經營層面
- ❌ 跨整備循環攜帶的貨幣或卡牌
- ❌ 道具收集
- ❌ Hub 升級系統

### 呈現層面
- ❌ 卡牌與敵人的精確數值顯示(預設情況)
- ❌ 完整透明的試算表式介面

### 判準
**問你自己「這個東西能不能跨整備循環攜帶?」如果答案是能,它就在侵蝕 OHK 美學。**

**問你自己「這個東西讓玩家用算術而非判斷?」如果答案是會,它就在侵蝕 OHK 美學。**

這些拒絕看起來都「對玩家不利」,但它們恰恰是讓玩家深度投入的原因。Titan Souls、Into the Breach、Hotline Miami 都不對玩家妥協,這就是它們的玩家會死忠的原因。

## 結語:OHK 美學的本質

> OHK 美學的本質,是相信「克制」本身就是一種給玩家的禮物——透過拒絕給予玩家所有能讓事情變簡單的工具,反過來給予他們唯一不能被廉價購買的東西:**真正的精通感**。

這就是為什麼這類遊戲注定小眾,但會被它的小眾愛得異常深。

### 設計的核心三句話
- **減法不是損失,是聚焦**
- **不確定不是隨機,是判斷的舞台**
- **隱藏不是欺瞞,是把計算交給系統,把判斷留給玩家**

掌握這三句話,任何設計決策都能被檢驗。
