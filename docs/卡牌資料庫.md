## 一、序章前半卡組總覽

序章前半卡組由 **3 種卡型**構成,起手 20 張,經 4 次整備補給後達 34 張(終戰時)。

| 卡型 | id | 弱點類型 | 起手張數 | 終戰前累積 |
|------|----|---------|--------|-----------|
| 射箭 | `tool_arrow_pierce` | 穿刺 | 10 | 17 |
| 投石 | `tool_stone_impact` | 衝擊 | 8 | 15 |
| 偵查 | `intel_weakness` | 情報 | 2 | 2 |
| **總計** | | | **20** | **34** |

> 注意:序章前半**不引入**爆發卡、角色卡、試探卡、flexible 卡。
> 這是教學階段的最小卡型集合,讓玩家專注於「類型對應 + 結算邏輯」基本概念。

## 二、結算機制快速摘要

> 完整機制見 [遊戲核心系統機制.md 第二章](遊戲核心系統機制.md)。

- 每張卡 = **一次使用**,用完從卡組消耗
- 每張卡貢獻一種類型的次數(`contribution` 欄位)
- 結算 = 加總本擊各類型次數,**任一條件達標即 OHK**
- 偵查卡為 `optional` lock(廣度 vs 深度的取捨)

### 3.3 偵查(情報)

| 欄位 | 值 |
|------|-----|
| `id` | `intel_weakness` |
| `name` | 偵查 |
| `resource_class` | tool |
| `function_class` | intel |
| `card_form` | individual |
| `weakness_type` | none |
| `contribution` | null |
| `strength_level` | 1(★) |
| `lock_class` | optional |
| `special_effect` | 雙模式(見下) |
| `description` | 情報卡。不 lock 換廣度;Lock 換深度 |

**雙模式**:

- **不 lock**:結束本擊時揭露此連戰**所有剩餘敵人的弱點類型**(廣度)
- **Lock**:即時揭露**當前敵人的弱點類型**(深度)

**敘事意象**:藉著踏查、聆聽、辨識足跡,理解獵物的弱點所在。父親引導之下的判讀練習。

**JSON 範例**:

```json
{
  "id": "intel_weakness",
  "name": "偵查",
  "resource_class": "tool",
  "function_class": "intel",
  "card_form": "individual",
  "weakness_type": "none",
  "contribution": null,
  "strength_level": 1,
  "lock_class": "optional",
  "special_effect": {
    "spec_unlocked": { "action": "reveal_remaining_weakness_types", "scope": "current_battle_chain" },
    "display_text_unlocked": "不 lock:結束本擊時揭露此連戰所有剩餘敵人的弱點類型",
    "spec_locked": { "action": "reveal_weakness_type", "target": "current" },
    "display_text_locked": "Lock:即時揭露當前敵人弱點類型"
  },
  "description": "情報卡。不 lock 換廣度(剩餘所有);Lock 換深度(當前敵人)"
}
```

## 四、整備階段補給規格

> 序章前半的整備補給是**劇情驅動、固定發放**,玩家無選擇。

### 4.1 起手卡組(序章開場)

| 卡型 | 張數 |
|------|------|
| 射箭 | 10 |
| 投石 | 8 |
| 偵查 | 2 |
| **總計** | **20** |

### 4.2 各整備階段的補給

| 整備階段 | 觸發時機 | 補給內容 | 卡組累計 | 劇情敘事 |
|---------|---------|--------|---------|---------|
| 整備 1 | 連戰 1(兔子×3)結束 | **完全補給**:消耗的卡片全部恢復 | 回到 20 | 完全休息,父親檢查兒子的裝備 |
| 整備 2 | 連戰 2(兔子+狐狸×2)結束 | +2 射箭、+2 投石 | 24(理論值) | 父親從袋中取出補給 |
| 整備 3 | 連戰 3(兔子+狐狸+狼)結束 | +2 射箭、+2 投石 | 28(理論值) | 狼嚎聲漸近,父親要兒子撿起更多石頭 |
| 整備 4 | 連戰 4(狼×3)結束 | +3 射箭、+3 投石 | 34(理論值) | 父親警覺異變,將自己備用的箭全部給兒子 |

### 4.3 整備階段的特殊規則

- **整備 1 = 完全補給**:第一連戰的「教學重來機制」+ 整備 1 共同確保玩家不會在開頭就因卡組耗盡卡關
- **整備 2-4 = 累積補給**:玩家實際持有的數量 = 起手 - 已消耗 + 補給。「卡組累計理論值」假設玩家無消耗
- **無選擇權**:玩家不能拒絕補給或自選張數。整備 = 劇情段落

## 五、戰前驗算

對序章前半 4 種敵人(詳見 [敵人資料庫.md](敵人資料庫.md))進行擊殺路徑驗算。

> **公式提醒**:混合需求 = max × 1.5 四捨五入。

### 5.1 兔子(衝擊 3 / 穿刺 1 / 混合 5)

| 路徑 | 計算 | 結果 |
|------|------|------|
| 走弱點穿刺 | 射箭 ×1 = 1 穿刺 | ✓ OHK(最省) |
| 走非弱點衝擊 | 投石 ×3 = 3 衝擊 | ✓ OHK |
| 走混合 | 任 5 張 | ✓ OHK(最貴) |

### 5.2 狐狸(衝擊 2 / 穿刺 2 / 混合 3)

| 路徑 | 計算 | 結果 |
|------|------|------|
| 走穿刺 | 射箭 ×2 = 2 穿刺 | ✓ OHK |
| 走衝擊 | 投石 ×2 = 2 衝擊 | ✓ OHK |
| 走混合 | 任 3 張 | ✓ OHK |

> 設計意圖:狐狸**無弱點偏好**,各路徑成本接近。教學「沒有弱點差異時要投注足夠資源」。

### 5.3 狼(衝擊 3 / 穿刺 2 / 混合 5)

| 路徑 | 計算 | 結果 |
|------|------|------|
| 走弱點穿刺 | 射箭 ×2 = 2 穿刺 | ✓ OHK |
| 走非弱點衝擊 | 投石 ×3 = 3 衝擊 | ✓ OHK |
| 走混合 | 任 5 張 | ✓ OHK |

### 5.4 異變的巨狼(衝擊 5 / 穿刺 8 / 混合 12)

| 路徑 | 計算 | 結果 |
|------|------|------|
| 走弱點衝擊 | 投石 ×5 = 5 衝擊 | ✓ OHK |
| 走非弱點穿刺 | 射箭 ×8 = 8 穿刺 | ✓ OHK |
| 走混合 | 任 12 張 | ✓ OHK |

> 設計意圖:**異變的巨狼是高需求 BOSS**,精算後通常需要整備 4 階段補給的最終裝備(終戰前 17 射箭 + 15 投石)才有餘裕擊殺。預期玩家**首戰會戰敗**,然後菁英化版本進入正式 BOSS 戰。

### 5.5 連戰總成本估算(理想路徑,全用弱點)

| 連戰 | 內容 | 理想路徑成本 |
|------|------|-----------|
| 1 | 兔子 ×3 | 3 射箭 |
| 2 | 兔子 + 狐狸 ×2 | 1 射箭 + 4 任意 |
| 3 | 兔子 + 狐狸 + 狼 | 3 射箭 + 2 任意 |
| 4 | 狼 ×3 | 6 射箭 |
| 5 | 兔子 ×2 + 巨狼 + 巨狼(菁英) | 2 射箭 + 5 投石 + 5 投石 |

> 玩家實際消耗會超過理想路徑(誤判、教學重來、保守加碼)。整備補給的設計就是吸收這個誤差。

## 六、後續期數參考:舊版 28 張卡規格

> **以下為 2026-05-13 版本的 28 張完整卡組規格**,作為**後續期數設計參考**保留。
> 此版本含爆發卡 / 角色卡 / 試探卡 / flexible 卡等進階機制,**第一期序章不實作**。

### 6.1 舊版卡池總覽

| lock_class | 數量 | 卡 |
|-----------|------|-----|
| `"none"` | 19 | 14 資源池 + 3 純戰鬥爆發 + 2 純戰鬥角色 |
| `"optional"` | 3 | 偵察、估量、警示 |
| `"required"` | 6 | 試探斬、探刺、試探擊、爆裂、洞察一擊、全知之眼 |
| **總計** | **28** | |

### 6.2 舊版 Layer A 資源池卡

| 卡 | id | ★ | base | 池總供給(舊 use_limit 設計) |
|----|----|---|------|--------|
| 直擊 | `tool_pool_impact_basic` | ★ | +1 衝擊 | 9 衝擊(3 張 × 3 use) |
| 重斬 | `tool_pool_impact_strong` | ★★ | +2 衝擊 | 8 衝擊(2 張 × 2 use) |
| 銳刺 | `tool_pool_pierce_basic` | ★ | +1 穿刺 | 9 穿刺 |
| 貫穿 | `tool_pool_pierce_strong` | ★★ | +2 穿刺 | 8 穿刺 |
| 點火 | `tool_pool_burn_basic` | ★ | +1 燃燒 | 6 燃燒 |
| 預判燒灼 | `tool_pool_burn_strong` | ★★ | +2 燃燒 | 8 燃燒 |

> **若移植到 2026-05-14 新模型**:同樣供給轉換成「卡組張數」=直擊池 9 張、重斬池 4 張、銳刺池 9 張、貫穿池 4 張、點火池 6 張、預判燒灼池 4 張(每張用一次)。

### 6.3 舊版 Layer B 進階卡(摘要)

詳細 JSON 規格參考 git 歷史的 `v3.1` 版本。要點摘要:

- **情報卡 3 張**(偵察、估量、警示):optional lock,廣度 vs 深度
- **試探卡 3 張**(試探斬、探刺、試探擊):required lock + binary 揭露
- **爆發卡 4 張**(重擊、致命刺、烈焰、爆裂):一次性高貢獻
- **角色卡 4 張**(殺戮意志、致命凝視、全知之眼、洞察一擊):戰役招牌

> 這些卡型**將於序章後半或後續戰役引入**,引入順序依教學節奏設計。

### 6.4 後續期數的卡池擴充原則

- **序章後半**:可能引入燃燒池(火把/油料)+ 試探卡(用於陌生森林敵人)+ 1 張角色卡(主角覺醒)
- **第二期+**:逐步引入爆發卡 / 多張角色卡 / 多角色支援
- 設計鐵律:**新卡引入必須有對應的敘事事件**(撿到 / 學會 / 鍛造 / 父親遺物),不憑空獲得
