## 一、敵人四類分類:兔子 / 狼 / 豹 / 熊

敵人類別決定**失敗代價**,並透過視覺類別本身傳達「失敗會怎樣」。

### 四類概覽

| 類別 | 敵人定位 | 失敗代價 | 敘事邏輯 | 視覺直覺 |
|------|---------|---------|---------|---------|
| **A. 兔子(無威脅普通)** | 教學基準、廉價失敗的學習場 | 卡牌消耗;敵人逃走,**不再回來** | 「沒有殺意」 | 「打不過也沒關係」 |
| **B. 狼(威脅普通)** | 主要遊戲場景 | 卡牌消耗 + 變精英回歸 + 精英失敗 = GO | 「會記恨並學習」 | 「會回來找我」 |
| **C. 豹(預設精英)** | 高張力時刻 | **失敗即 GAME OVER** | 「本來就要殺你」 | 「不能失敗」 |
| **D. 熊(Boss / 劇情敵人)** | 戰役節奏的高潮 | **失敗即 GAME OVER** | 戰役節奏的關鍵 | 「全力以赴」 |

### 設計合約(各類別的核心契約)

**兔子**:玩家看到應立即直覺「**這隻打不過也沒關係**」。兔子不是「廢敵人」——它仍提供戰鬥練習、資源消耗壓力,且擊敗它仍有節奏意義。

**狼**:玩家看到應直覺「**這隻會記恨,失敗會回來找我**」。狼是主要遊戲場景,失敗有有機代價但不致命。

**豹**:玩家看到應直覺「**這隻不能失敗,我得認真**」。豹的視覺暗示要極明顯(銳利、致命氣質),讓玩家從外觀就讀出威脅等級。

**熊**:玩家看到應直覺「**這是戰役的關鍵時刻,必須全力以赴**」。熊是 Boss 級對手,代表戰役節奏的關鍵時刻。

### 跨類別的設計原則

1. **物種視覺差異要明顯**:玩家不需要 UI 標籤就能從外觀辨識類別
2. **狼變的精英敵人外觀不變**:UI 標籤「精英化 ⚠」標示,但敵人本身視覺與原本一樣
3. **豹型外觀本身就傳達威脅**:不需要額外「精英標記」
4. **熊型外觀本身就傳達 Boss 氣質**:不需要額外標記

> 設計上**每個物種就是一個視覺類別**,類別辨識靠物種本身的視覺特徵,而非外加的精英標記。

## 二、模板格式定義

所有新敵人模板必須使用此標準格式填寫。

```
### 模板:[名稱]
**一句話定位**:[此模板在系統中的角色]
**敵人類別**:[兔子 / 狼 / 豹 / 熊](對應失敗代價)

#### 層 1 — 模板永久顯性
- 大類識別:
- 敵人類別:[兔子 / 狼 / 豹 / 熊]
- 弱點範圍(可單獨擊殺的類型,不在範圍內的類型只能透過混合任意路徑計入):
- 弱點需求層級範圍(弱點類型的次數需求):
- 戰鬥狀態傾向:

#### 層 2 — 次級視覺線索
所有條目引用視覺線索詞彙表(第四章):
- [視覺元素] → [含義]
- ...
- 敵人類別視覺暗示

#### 層 3 — 即時狀態(完全顯性)
- 戰鬥狀態標籤
- 上一場用情報牌揭露的相關資訊(若該情報牌指向本場目標)
- 若為狼型精英狀態:UI 標籤「精英化 ⚠」+ 戰鬥紀錄可調出上次資料
- 其他戰鬥開始時必定告知玩家的事項

#### 層 4 — 隱藏狀態(僅情報牌可揭)
- **完整需求表**:本實例各類型(在弱點範圍內)的精確需求次數
- **混合任意需求**
- 敵人類別(可透過警示揭露)

#### 失敗後處理(依敵人類別)
- **兔子型**:卡牌消耗,敵人逃走不再回來
- **狼型**:第一次失敗 → 變精英排末;第二次以上失敗 → 進未解決池
- **豹型**:失敗即 GAME OVER
- **熊型**:失敗即 GAME OVER

#### 實例變異規則
- 主要挑戰軸分布:
  - 80% — 主軸:需求表 instance(弱點類型 × 需求層級的具體組合)
  - 20% — 主軸:戰鬥狀態變異
- 非主軸維度:停留在模板基準
- **精英化不在此規則中**(精英化是玩家失敗造成,不是隨機 instance)

#### 設計合約
- 教玩家:
- 老手能做到:
- 新手失敗時能理解:

#### 登場時機建議
- 適合出現階段:(早期教學 / 中期挑戰 / 後期驗收)
- 最終連戰適合度:(是 / 否)
- 不應出現的情境:
```

## 三、9 條敵人設計原則

任何新模板提案前必須通過全部檢查。違反任何一條都要重新設計或明確說明為什麼例外。

### 1. 層 2 視覺線索必須可推論,不能靠背

- 黃金標準範例:**持矛 → 穿刺弱點 →**「長兵器使用者站位偏遠,近身穿刺難擋」(因果直接,玩家第一次見可自行推論)
- 反例:**紅腰帶 → 衝擊弱點**(純記憶,失敗)
- 判準:玩家第一次猜不到沒關係,但事後回想要覺得「對,這說得通」
- **Why:** 否則層 2 從「老手的回報」變成「老手的課本」

### 2. 每個實例只有一條「主要挑戰軸」

- 不是「只能變一個維度」,而是「同時間只有一個維度達到最大幅變異,其他軸保持模板基準/可預測值」
- 預設分布:80% 主軸=需求表 instance / 20% 主軸=戰鬥狀態
- **Why:** 避免「每隻都是新東西」摧毀元知識,同時保持「熟悉但不重複」

### 3. 連戰位置不是敵人屬性

- 屬於「連戰配置設計」範疇,跟敵人模板分開設計
- **Why:** 概念混淆會讓兩邊都做不好

### 4. 精英化敵人的視覺合約(2026-05-11 更新)

- 狼變的精英敵人**外觀數值與原本完全相同**,只有 UI 標籤「精英化 ⚠」標示
- 豹型本身就有精英級視覺,不需額外標記
- **Why:** 「外觀數值不變」讓玩家用「戰鬥紀錄 + 元知識」擊敗精英,符合「理解即進度」原則

### 5. 失敗代價必須符合敵人類別合約

- 兔子失敗 → 逃走,不會精英化
- 狼失敗 → 精英化回歸 / 進未解決池
- 豹/熊失敗 → GAME OVER
- 不允許「兔子失敗變精英來找我」這種違反敘事邏輯的設計
- **Why:** 「為什麼這隻兔子失敗會變精英來殺我」的違和感會破壞玩家對類別的信任

### 6. 視覺覆蓋必須完整,不允許用等機率隨機填補設計缺口

- 若某「弱點 × 需求層級」組合沒有對應的層 2 視覺線索,正確做法是回去補設計
- 不允許「沒視覺暗示時取兩種等機率」這種寫法——那是把設計失敗包裝成隨機性
- **Why:** 「不確定」與「隨機」不同。所有不確定必須是 hidden state(玩家可投資揭露),不是 output randomness

### 7. 每個模板必須有「設計合約」三問

- **教玩家什麼** / **老手能做到什麼** / **新手失敗時能理解什麼**
- **Why:** 答不出來的模板就只是「另一個血條」

### 8. 每個模板必須有「登場時機建議」

- 適合出現階段(早期教學 / 中期挑戰 / 後期驗收)
- 最終連戰適合度(是 / 否)
- 不應出現的情境
- **Why:** 沒明寫的話,教學型敵人可能被誤放在最終連戰當填充,削弱結構張力

### 9. 層 3 內建假設:跨場資訊持續性

- 上一場用情報牌揭露的資訊,下一場開始時玩家仍可看見(顯示在層 3)
- 這對應「情報是跨場投資」的設計,所有模板格式都假設此行為
- **Why:** 否則情報牌的「跨場效益」承諾就斷掉

## 四、視覺線索詞彙表

本章定義 P6 中所有層 2 視覺線索的標準對應關係,作為**角色設計**與**美術設計**的共同參考。

### 核心原則

#### 可推論性

每條線索都必須有「事後說得通」的推論依據。
- ✓ 持矛 → 穿刺弱點 → 「長兵器使用者站位偏遠,近身穿刺難擋」
- ✗ 紅腰帶 → 衝擊弱點(純記憶,無因果)

#### 跨模板一致

同一個視覺元素在所有模板中代表同樣的含義。

#### 互斥對應

每條視覺元素只對應單一含義。新增條目前需檢查是否與既有條目衝突。

#### 覆蓋完整,不允許隨機填補

若某「需求次數 × 弱點」組合在本表中找不到對應視覺,正確做法是回來補設計新條目,不允許用「等機率隨機」處理。

### A. 弱點類型線索

#### A.1 衝擊弱點(vulnerable to blunt/impact)

| 視覺元素 | 推論依據 |
|---------|---------|
| 雙手大型武器(大斧、戰錘、巨劍) | 重型武器使用者慣以攻代守,防守步伐較慢,難在衝擊前完成位移緩衝 |
| 揮擊鏈式武器(鏈鞭、流星錘) | 揮擊軌跡需要空間,無法緊密貼身防禦,反向衝擊能推倒重心 |
| 鏈甲、層疊環甲(無內襯) | 鏈環直接傳遞鈍器衝擊力至軀體,無緩衝吸收 |
| 硬殼鎧甲但無棉襯 | 力量傳遞不被吸收,骨骼受衝擊 |

**反證視覺**:多層緩衝甲、棉甲外覆 → 衝擊被吸收,不可能是衝擊弱點

#### A.2 穿刺弱點(vulnerable to thrust/pierce)

| 視覺元素 | 推論依據 |
|---------|---------|
| 長柄武器(長矛、長戟) | 站位偏遠,近身穿刺難以即時轉防 |
| 大盾近戰 | 持盾正面強,但盾邊與盾下方暴露,穿刺可繞入 |
| 大型整片硬甲但接縫明顯 | 細尖物可從關節縫隙穿入 |
| 輕甲、皮甲、布甲 | 缺乏結構強度,直接被穿透 |
| 鎧甲設計偏側面/肩部加厚 | 為擋揮擊設計,正面緩衝相對偏薄 |

**反證視覺**:鎖鏈緊密交織 + 內襯密實 → 穿刺難以分開鏈環

#### A.3 燃燒弱點(vulnerable to fire/burn)

| 視覺元素 | 推論依據 |
|---------|---------|
| 木質結構、樹皮甲 | 直接燃燒 |
| 毛皮、皮裘外覆 | 易燃材質 |
| 油布、織物外覆 | 易燃材質 |
| 蠟封結構、樹脂塗層 | 受熱融化失去防護 |
| 多毛/多油脂體型 | 引燃媒介充足 |
| 自然/植物/獸類外觀 | 有機體易燃 |

**反證視覺**:純金屬光面甲(無有機外覆) / 濕潤環境 / 自帶火屬性視覺(火紋、灰燼質感) → 不可能是燃燒弱點

> **已知覆蓋盲點**:目前燃燒弱點線索集中在「材質有機」,可能無法支援「金屬重甲但仍有燃燒弱點」的設計需求。若有此需求要補二級邏輯(例如「鎧甲關節大量油料潤滑 → 油料引燃 → 關節失能」)。

### B. 需求層級線索

| 層級 | 視覺元素 | 推論依據 |
|------|---------|---------|
| 低 | 體型瘦小、護具殘缺破損、明顯舊傷、磨損生鏽 | 防護結構不完整 |
| 中 | 一般成人比例、完整但無強化的標準裝備 | 標準防護 |
| 高 | 體型巨大厚實、多層護具補強、紮根姿態、額外肩胸甲強化 | 防護結構強化 |

**疊加原則**:需求層級線索與弱點線索互相獨立。「整片厚甲但接縫多」= 高需求 + 穿刺弱點,兩個特徵獨立呈現,玩家分別讀取,不應合併解讀。

### C. 戰鬥狀態傾向線索

| 傾向 | 視覺元素 | 推論依據 |
|------|---------|---------|
| 先攻 | 高處警戒姿勢、瞭望姿態、視覺優勢裝備(望遠工具、銳利視線) | 此類角色設計上「會先發現玩家」 |
| 被襲 | 隱匿斗篷、暗色低調配色、蹲伏潛行姿態、武器收納未持 | 此類角色「先發現你才動手」,玩家進入時已落下風 |
| 標準 | 直立、武器持手中但非攻擊姿勢 | 對等接戰 |

### D. 敵人類別視覺暗示

#### D.1 兔子型(無威脅普通)

| 視覺元素 | 推論依據 |
|---------|---------|
| 小體型、瘦弱 | 物理上不具威脅 |
| 無武器,或武器極簡(短刀、棍棒) | 戰鬥能力低 |
| 退縮、警戒、想逃姿態 | 沒有殺意,只想自保 |
| 表情/姿勢透露恐懼 | 不會主動傷害玩家 |
| 配色偏柔和(土色、灰色) | 不傳達威脅 |

**反證**:銳利武器、戰鬥姿態 → 不是兔子型

#### D.2 狼型(威脅普通)

| 視覺元素 | 推論依據 |
|---------|---------|
| 中體型,人形戰士比例 | 標準對手 |
| 武器明顯(劍、矛、斧、戰錘) | 有殺意但專業 |
| 警戒姿態,但有規範動作 | 訓練過的戰士 |
| 鎧甲完整或皮甲 | 有防護能力 |
| 配色偏中性(金屬色、皮革色) | 軍事化威脅 |

**反證**:無武器(→兔子)、致命氣質(→豹)、壓倒性體型(→熊)

#### D.3 豹型(預設精英)

| 視覺元素 | 推論依據 |
|---------|---------|
| 銳利、致命的氣質 | 「本來就要殺你」 |
| 武器精緻、致命(雙匕首、毒鞭、致命刺擊武器) | 用於一擊斃命 |
| 隱匿、潛伏、襲擊預備姿態 | 不是公平對峙 |
| 暗色配色(深色斗篷、暗色裝束) | 殺手氣質 |
| 體型可能瘦削但精煉 | 「速度+致命」而非「力量」 |

**反證**:無威脅姿態(→兔子/狼)、壓倒性體型(→熊)

#### D.4 熊型(Boss / 劇情敵人)

| 視覺元素 | 推論依據 |
|---------|---------|
| 壓倒性體型(明顯大於玩家) | 物理上不可忽視 |
| 重型武器、巨型裝備 | 對等的力量等級 |
| 王者姿態、領袖氣場 | 戰役節奏的關鍵 |
| 額外裝飾(紋章、披風、王冠等) | 劇情上的「重要性」 |
| 配色可能鮮明(深紅、金色等高識別色) | 「不可錯過的目標」 |

**反證**:中等體型(→狼)、銳利氣質(→豹)

### E. 衝突檢查表

絕對禁止的視覺-含義配對:
- 重型武器 ↔ 高靈活度暗示
- 純金屬甲 ↔ 燃燒弱點(除非有二級邏輯支持)
- 矮小體型 ↔ 高需求
- 警戒姿勢 ↔ 被襲傾向
- 兔子型外觀(無威脅、無武器)↔ 致命攻擊性
- 熊型外觀(壓倒性體型)↔ 兔子型失敗代價(無精英化)

### F. 詞彙表擴充規則

新增條目時必須:
1. 寫出「視覺元素 → 推論依據」二元組
2. 推論依據符合「事後說得通」標準
3. 檢查是否與現有條目衝突
4. 確認該含義是否已有更好的視覺表達(避免冗餘)
5. 若是橫跨多個模板會用到的元素,標註此元素的「跨模板含義鎖定」

## 五、需求表設計原則

### 單主軸原則

每隻敵人的需求表應該有清楚的「主軸」——某個類型明顯比其他低,作為「弱點」。

### 弱點範圍 vs 不在範圍內

- **弱點範圍**:可單獨擊殺的類型(走專項路徑即可 OHK)
- **不在範圍**:該類型無法單獨擊殺,只能透過「混合任意」路徑計入

範例:哨兵的弱點範圍是「衝擊 / 穿刺」,燃燒不在範圍——林地偵察兵不能用火惹眼。

### 強度層級對應

弱點需求次數的層級:
- **低(1)**:1 張 ★ 卡即過(教學基準)
- **中低(2)**:1 張 ★★ 卡或 2 張 ★ 卡
- **中(2-3)**:1-2 張 ★★ 卡
- **中高(3)**:2 張 ★★ 卡或 1 張 ★ + 1 張 ★★
- **高(4)**:1 張 ★★★ 爆發卡 或 1 張 ★★ + 1 張 ★★
- **極高(6+)**:必須投入爆發 / 角色卡組合

## 六、平衡規則(2026-05-13 定案)

### 6.1 混合任意需求公式

> **混合需求 = 該敵人最高類型需求 × 1.5,取整數**

#### 範例

| 敵人 instance | 各類型需求 | 最高 | 混合 |
|--------------|---------|-----|-----|
| 哨兵 弱衝擊低 | 衝擊 1 / 穿刺 4 | 4 | **6** |
| 哨兵 弱衝擊中低 | 衝擊 2 / 穿刺 5 | 5 | **8** |
| 巡守 弱衝擊中 | 衝擊 2 / 穿刺 5 / 燃燒 5 | 5 | **8** |
| 巡守 弱衝擊中高 | 衝擊 3 / 穿刺 6 / 燃燒 6 | 6 | **9** |
| 巡守 弱衝擊高 | 衝擊 4 / 穿刺 7 / 燃燒 7 | 7 | **11** |
| 重甲衛兵 弱衝擊中 | 衝擊 2 / 穿刺 5 | 5 | **8** |
| 重甲衛兵 弱衝擊中高 | 衝擊 3 / 穿刺 6 | 6 | **9** |
| 重甲衛兵 弱衝擊高 | 衝擊 4 / 穿刺 7 | 7 | **11** |
| 伏擊者 弱穿刺低 | 穿刺 1 / 燃燒 3 | 3 | **5** |
| 伏擊者 弱穿刺中 | 穿刺 2 / 燃燒 4 | 4 | **6** |
| 領主 弱衝擊高 | 衝擊 4 / 穿刺 7 | 7 | **11** |
| 領主 弱衝擊極高 | 衝擊 6 / 穿刺 9 | 9 | **14** |
| 領主 弱穿刺高 | 衝擊 7 / 穿刺 4 | 7 | **11** |
| 領主 弱穿刺極高 | 衝擊 9 / 穿刺 6 | 9 | **14** |

#### 設計意圖

確保「全押 hedge」不是有效策略,逼迫玩家盡量找出弱點對應路徑。

**三條路徑的成本關係**:
```
走弱點(最低需求)< 走非弱點(最高需求)< 走 hedge(最高 × 1.5)
```

→ hedge 約為走錯非弱點專項的 +50% 成本。**鼓勵玩家先揭露弱點**,但 hedge 仍可達成,不會卡關。

#### 歷史:被取代的兩個提案

| 提案 | 缺陷 |
|------|------|
| **×4 of min**(初版) | 對中低需求敵人 hedge 反而比走非弱點便宜 |
| **×1.7 of sum**(過渡) | 對巡守等多類型敵人 hedge 變過貴(11 變 31)|

### 6.2 本擊上限(暫定)

> **依敵人強度決定本擊上限**。環境影響(連戰位置 / 場景)後續加入。

| 敵人類型 | 上限計算 |
|---------|--------|
| 普通敵人(兔 / 一般狼)| 最低類型需求 × 2 |
| 精英敵人(精英狼)| 最低類型需求 × 2.5 |
| 預設精英(豹)| 最低類型需求 × 2.5 |
| BOSS(熊)| 最低類型需求 × 3 + 5 |

#### 範例

| 敵人 | 最低需求 | 計算 | 本擊上限 |
|-----|--------|-----|--------|
| 哨兵 弱衝擊低 | 1 | 1 × 2 | **2 張** |
| 巡守 弱衝擊中 | 2 | 2 × 2 | **4 張** |
| 重甲衛兵 弱衝擊高 | 4 | 4 × 2 | **8 張** |
| 伏擊者 弱穿刺低(豹) | 1 | 1 × 2.5 | **3 張**(取整) |
| 領主 弱穿刺極高(熊) | 6 | 6 × 3 + 5 | **23 張** |

#### 設計意圖

- **防止「dump 全資源 fini boss」**:不能整場戰役不出卡,最後 boss 用 25 張卡 OHK
- **強迫精算本擊**:玩家必須在上限內找最有效組合
- **保留情報投資價值**:走 hedge 通常接近上限,等於強制玩家先揭露

#### 未決細節

- 環境效果如何修正上限?
- 失敗後重新挑戰時上限是否變化?
- 雙模式 UI 下上限是否提示?

→ 詳細數值待紙上測試 + 實機調整。

## 七、失敗代價設計合約

### 各類別的設計合約

#### 兔子型失敗

```
觸發:玩家對兔子型敵人 commit 後 Underkill
處理:
  1. 投入的卡牌依然消耗
  2. 兔子逃走,從本連戰移除
  3. **不會精英化回歸**
  4. 不會替換下連戰敵人
```

**設計意圖**:給玩家「廉價失敗的學習場」。兔子是新手友善設計,失敗代價輕但不無視(卡牌仍消耗)。

#### 狼型失敗(三層代價)

**第一層 — 連戰內第 1 次失敗**

```
1. 投入的卡牌依然消耗
2. 該敵人標記為「精英狀態」
3. 排到該連戰末段(熊型 Boss 之前)
4. 連戰場次 +1
玩家會在該連戰結束前再次面對它(精英化)
```

**第二層 — 連戰內第 2+ 次失敗**

```
1. 投入的卡牌依然消耗
2. 該敵人「移出該連戰」進入「未解決池」
3. 連戰繼續進行
連戰結束後玩家會面對「追擊選擇」
```

**第三層 — 連戰結束 → 追擊機制** → 詳見 [遊戲核心系統機制.md](遊戲核心系統機制.md) 第六章

#### 豹型失敗

```
觸發:玩家對豹型敵人 commit 後 Underkill
處理:**GAME OVER**

豹本來就是精英級對手,沒有「第二次機會」。
玩家必須一次擊敗(類似 OHK Boss 的決鬥精神)。
```

**設計意圖**:給玩家「**這次不能失敗**」的高張力時刻。豹的視覺暗示要極明顯,讓玩家從外觀就讀出威脅等級。

#### 熊型失敗

```
觸發:玩家對熊型敵人 commit 後 Underkill
處理:**GAME OVER**

熊是 Boss 級對手,代表戰役節奏的關鍵時刻。
失敗即終戰,符合 OHK「最終一擊」的儀式精神。
```

**設計意圖**:作為連戰節奏的高潮點(每連戰最後一隻 + 最終戰領主)。

### 配套情報卡更新

- **警示卡**:揭露敵人類別(兔子/狼/豹/熊),取代舊版「揭露精英化」
- **試探擊**:揭露「失敗是否會 GAME OVER」yes/no
- **全知之眼**:揭露範圍包含 enemy_class

## 八、登場時機建議

每個模板都必須明寫「登場時機建議」,避免教學型敵人被誤放在最終連戰當填充。

### 三階段定位

| 階段 | 適合 | 不適合 |
|------|------|--------|
| **早期教學** | 兔子、低需求狼 | 豹、熊 |
| **中期挑戰** | 標準狼、首次出現豹 | 純教學基準的兔子 |
| **後期驗收** | 精英化狼、極高需求熊 | 過低需求模板 |

### 最終連戰適合度

- ✓ 適合:狼型(填充)、熊型(boss)、精英化變體
- ✗ 不適合:純教學模板(哨兵)
- 部分:豹型(作為驗收前的高張力中段)

### 不應出現的情境

每個模板都應明寫「不應出現的情境」。例:
- 哨兵不應出現在中後期(過低需求破壞節奏)
- 伏擊者不應出現在首輪連戰(被襲狀態對新手過於懲罰)
- 領主不應出現在首輪連戰(玩家還沒累積足夠爆發牌)

## 九、設計新敵人的檢查清單

新增一個敵人模板前,必須通過以下所有檢查。

### 9 條設計原則檢查(對應第三章)

1. □ 層 2 視覺線索都可推論(寫得出因果)
2. □ 每個實例只有一條主要挑戰軸
3. □ 連戰位置不混進敵人屬性
4. □ 精英化外觀合約符合(狼變的不改外觀)
5. □ 失敗代價符合敵人類別合約
6. □ 視覺覆蓋完整,無「等機率隨機填補」
7. □ 設計合約三問都答得出來
8. □ 登場時機建議明寫
9. □ 層 3 跨場資訊持續性保留

### 敵人類別檢查

10. □ 敵人類別明確(兔子 / 狼 / 豹 / 熊)
11. □ 視覺類別本身就傳達「失敗會怎樣」
12. □ 對應視覺線索詞彙表 D 章節

### 需求表檢查

13. □ 弱點範圍與不在範圍區分清楚
14. □ 弱點需求層級對應強度檔位
15. □ 混合任意需求依公式計算
16. □ 所有 instance 都有對應視覺暗示

### 與 OHK 美學的相容性檢查

17. □ 失敗結果是有機的(玩家造成,不是設計師強加)
18. □ 不引入「玩家第一次遭遇必懵」的未知機制
19. □ 視覺暗示讓「老手免情報擊殺」可行
20. □ 對應「設計合約三問」全部答得出來

通過所有檢查才能進入資料庫。
