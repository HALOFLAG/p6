## 七、未解設計缺口

以下是已記錄但尚未完成設計的項目,實作時可決定要不要先補:

| 項目 | 重要度 | 影響 | 建議 |
|------|--------|------|------|
| **NPC 性格與對話文本** | 中 | 戰後校準的實際呈現 | 用佔位文本先實作機制,文本後補 |
| **戰鬥紀錄系統** | 高(新機制下升級) | 玩家對精英狀態的「往前看」需求 | 至少實作基本紀錄 + 查閱介面 |
| **連戰配置設計** | 中 | 5 隻敵人在 2 連戰中如何排序 | 原型用既定順序,正式版再規劃 |
| **整備期回補機制** | 中 | 影響資源遞減節奏 | 原型可先不實作,測試後校準 |
| **教學期 UI 預設細節** | 中 | 第一場玩家體驗 | 至少做出 UI 灰階機制 |
| **本擊上限** | 中 | 避免 dump 全資源 | 詳見 [本擊上限討論.md](本擊上限討論.md);原型先不限制,後續決定 |
| **混合任意需求公式 ×1.7 of sum** | 中 | 影響 hedge 路徑成本 | 第三次紙上測試後決定是否採用 |
| **金屬甲燃燒弱點覆蓋** | 低 | 視覺線索詞彙表盲點 | 暫不需要,有需求時補 |
