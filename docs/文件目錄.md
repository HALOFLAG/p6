## 二、主線文件總覽(6 份,必讀)

### 1. [遊戲設計概念.md](遊戲設計概念.md)

**定位**:入門文件 + 設計哲學裁判依據

**包含**:P6 是什麼、10 大核心特色、OHK 美學 5 原則、核心機制概述、一場戰役的遊玩流程、設計上必須拒絕的東西、與既有作品的關係

**何時讀**:第一次接觸 P6 / 設計決策搖擺時(回到原則)

---

### 2. [遊戲核心系統機制.md](遊戲核心系統機制.md)

**定位**:完整系統運作說明

**包含**:Place/Lock 三子階段、類型計數結算、敵人四層資訊架構、戰鬥狀態系統、失敗代價與精英機制、連戰結構與追擊機制、整備期、戰後校準對話、猜弱點機制、呈現層原則、教學期 UI、戰鬥銜接、跳過機制(待設計)、視覺語言要求

**何時讀**:實作前必讀 / 機制疑問時查閱

---

### 3. [卡牌設計原則.md](卡牌設計原則.md)

**定位**:設計新卡的規範手冊

**包含**:無抽牌哲學、三層結構(工具/爆發/角色)、兩軸正交分類(資源 × 功能)、卡牌形態(資源池 vs 個體)、lock_class 三分類、contribution + flexible、完整資料欄位定義、special_effect + action 列表、玩家可見性對照、卡牌獲得隨機性、多角色設計、**設計新卡的檢查清單**

**何時讀**:新增/修改卡牌前必讀

---

### 4. [敵人設計原則.md](敵人設計原則.md)

**定位**:設計新敵人的規範手冊

**包含**:敵人四類分類(兔子/狼/豹/熊)、模板格式定義、9 條敵人設計原則、**視覺線索詞彙表**(完整 A-F 章節)、需求表設計原則、混合任意需求公式、失敗代價設計合約、登場時機建議、**設計新敵人的檢查清單**

**何時讀**:新增/修改敵人前必讀

---

### 5. [卡牌資料庫.md](卡牌資料庫.md)

**定位**:28 張原型卡的完整實例庫

**包含**:卡池總覽 + 結算機制摘要 + 14 張資源池卡 + 3 張情報卡 + 3 張試探卡 + 4 張爆發卡 + 4 張角色卡(各含表格 + JSON + 對應原則標註)+ 卡池總供給驗算 + 對 5 隻敵人的擊殺路徑驗算 + 已知盲點

**何時讀**:實作 / 改數值 / 測試 / 列印紙卡

**配套**:[卡牌規格.xlsx](卡牌規格.xlsx)(Excel 列印版)
