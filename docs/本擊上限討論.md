# 本擊上限討論

## 文件目的與狀態

本文件記錄 2026-05-11 對「動態本擊上限」設計的討論與分析。

**結論:暫不採用**,維持原型階段「無本擊上限」的設定。本提案保留作為日後實測後的決策參考。

**何時重新檢討**:
- 第三次紙上測試後,觀察「無上限」是否再度暴露 dump 問題
- 電子原型 MVP 完成後,實測「最終戰玩家 commit 規模」與「儀式感」的關係
- 玩家自述「commit 自由度」是否令人困擾

---

## 一、起源

第二次紙上測試暴露的問題:
- 玩家在最終戰(領主)commit 5 張(直擊 + 爆裂 + 洞察一擊 + 殺戮意志 + 致命凝視)
- 結果大幅 overkill 83%
- **「精準一擊」感受被稀釋為「全押全衝」**

無本擊上限的根本問題:
- 玩家在最終戰可以「dump 全部資源」,因為之後沒有戰鬥需要保留
- 「精準 vs 過量」的判斷被「保險就是正確」取代
- OHK 美學的「克制」精神依賴外部框架,不能全靠玩家自我控制

> 提案:**動態本擊上限**——依敵人難度或遊戲階段動態調整 commit 卡張數的上限。

---

## 二、三種策略對比

### A. 純依敵人(每隻獨立設定)

每隻敵人有自己的 `commit_size_limit`:

| 敵人 | 提案上限 | 設計意圖 |
|------|---------|---------|
| 哨兵 | 2 | 弱敵逼玩家用 1-2 張完成,強化「精準」 |
| 巡守 | 4 | 中等敵人允許組合 |
| 重甲衛兵 | 4 | 同上 |
| 伏擊者 | 3 | 被襲狀態下少卡更緊張 |
| 領主 | 6 或無限 | 最終戰允許招牌儀式 |

**優點**:每隻敵人「能用幾張」變成個性的一部分;弱敵不能濫用資源
**缺點**:玩家認知負擔(每場看上限);敵人設計多一維;同類敵人 instance 仍要區分上限會更複雜

### B. 純依階段(戰役推進)

按連戰位置設 baseline:

| 階段 | 本擊上限 |
|------|---------|
| 連戰 1 | 3 |
| 連戰 2 | 4 |
| 連戰 3 | 5 |
| 最終戰 | 無限 |

**優點**:玩家可預期,「成長感」(戰術空間成長,非角色升級)
**缺點**:同階段所有敵人一視同仁,失去敵人個性層次

### C. 混合(階段 baseline + 個別 override,推薦)

| 階段 | Baseline | Override 範例 |
|------|---------|--------------|
| 連戰 1 | 3 | 哨兵可降到 2(超弱敵的精準挑戰) |
| 連戰 2 | 4 | 某 mini-boss 可降到 2(機制挑戰) |
| 連戰 3 | 5 | 最終 boss 設無限或 7(儀式) |

**優點**:結構清楚 + 個別敵人可有特色 override
**缺點**:還是有複雜度但好管理

## 三、若採用,推薦的具體設定

走 C 混合,但對原型保持簡單:

### 修正後的最終建議

| 階段 | Baseline | 原型 5 敵人對應 |
|------|---------|--------------|
| 連戰 1 baseline | **4** | 哨兵 4 / 巡守 4 / 重甲 4 |
| 連戰 2 baseline | **4** | 伏擊者 4(因被襲狀態本來就不利) |
| 最終戰 | **6** | 領主 6(或無限,看儀式化程度) |

**只在必要時 override**(原型不需要)。

### 為什麼不更緊(例如 連戰 1=3)

- 玩家可能想「偵察 + 試探斬 + 重斬 + 直擊 = 4 張」(資訊+戰鬥組合)
- 上限 3 → 必須砍一張,可能砍掉資訊投資 → 玩家學不到「投資資訊的價值」
- **新手期應允許組合**,讓玩家學會「資訊 + 戰鬥」的協同

### 為什麼最終戰不無限

- 玩家「可能」會選 5 張,但他知道有 6 上限,會考慮「這隻值得用滿嗎?」
- **有上限的存在本身就讓玩家思考「我要不要用滿」**——這是判斷
- 無上限只變成「我帶夠了嗎」,decision 弱化

## 四、設計收益分析

### 收益 1:解決 dump 問題

上次最終戰測試:玩家 commit 5 張 = 大幅 overkill 83%

有上限的存在本身就讓玩家思考「**我要不要用滿上限**」——這是判斷,無上限就只是「我帶夠了嗎」。

### 收益 2:強化敵人個性

「這隻只給你 3 張機會」本身是設計表達:
- 哨兵 上限 3 = 「弱敵就該精準解決」
- 領主 上限 6 = 「終極挑戰允許全力」

**敵人不只是「需求多寡」,還有「能用幾張」的對抗節奏**。

### 收益 3:強化「精準擊殺」的價值

上限 3 對需求 2 的弱敵 = 完美擊殺最多容差 1 張
這讓「commit 1-2 張剛好過」成為精煉的玩法,而不是「commit 3-4 張保險」的浪費。

## 五、設計風險

### 風險 1:上限太緊壓縮策略

玩家可能想:**「偵察(揭弱點) + 試探斬(驗證) + 重斬(攻擊) + 直擊(再加 1)」= 4 張**(資訊+戰鬥組合)。

對哨兵 上限 3 → 必須砍一張,可能砍掉資訊投資 → 玩家學不到「投資資訊的價值」

**緩解**:新手期(連戰 1)的上限應該至少 = 4,讓玩家學會「組合」概念。

### 風險 2:玩家認知負擔

每場戰鬥多一個變數要記憶。

**緩解**:
- UI 上明顯顯示「本擊上限: X 張」
- 進入戰鬥前 NPC 提示(教學期):「這隻只給你 3 張機會,選好」

### 風險 3:對 c 類強制 lock 卡的衝突

某些 c 類卡(如試探斬)強制 lock,佔用本擊張數。
- 上限 3 + 1 試探斬 lock = 剩 2 張可 place
- 如果想加爆裂(c)+ 洞察(c) = 已經 3 張全部 c 類

**緩解**:c 類 lock 卡和 a/b 類等位佔本擊張數(沒有特權)。設計上接受這個張力——它是動態上限對 c 類的天然限制。

### 風險 4:「全戰役共享資源」+「本擊上限」可能讓資源遞減失去意義

如果你只能 3-4 張 commit,卡池有 59 use,壓力太低。

**緩解**:配合「無回補」或極低回補的設計,深消耗仍會逼玩家節省。或上限可以提高(每階段 +1)。

### 風險 5:OHK 美學「拒絕養成」與「上限隨階段提升」的張力

- 階段性提升上限 ≈ 「玩家成長」?
- 但這不是角色變強,是「戰術空間變大」
- 應該勉強符合「理解即進度」原則(玩家信心增加,敢用更多卡)
- 但要小心避免變成「成長系統」的感覺

## 七、無上限要不要保留作為選項?

延伸討論:某些角色卡能否解鎖「本場無上限」?

**範例設計**:某張角色卡 special_effect 為「本場本擊無上限」——讓玩家偶爾體驗「全力傾注」的儀式感。

但這個方向需要慎重:
- 解鎖無上限可能成為「終極招牌」的代名詞
- 戰役單次的「招牌動作」加上「無上限」= 玩家最強烈的儀式時刻
- 但若濫用,又回到 dump 問題

**保留為日後設計空間**,原型不需要。

採用 C 混合(連戰 1-2 上限 4、最終戰上限 6)是最有可能的方向,但實際數字需測試後微調。
