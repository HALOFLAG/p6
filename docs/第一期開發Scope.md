## 一-A、實作基準確定的平衡公式(2026-05-13,2026-05-14 補充取整規則)

### 混合任意需求公式

> **混合需求 = 該敵人最高類型需求 × 1.5,四捨五入取整數**

範例(序章前半敵人):
- 兔子(衝擊 3 / 穿刺 1)→ 4.5 → **5**
- 狐狸(衝擊 2 / 穿刺 2)→ 3 → **3**
- 狼(衝擊 3 / 穿刺 2)→ 4.5 → **5**
- 異變的巨狼(衝擊 5 / 穿刺 8)→ 12 → **12**

**設計含義**:hedge(走混合任意)= 走錯類型的成本約 +50%。**鼓勵玩家先揭露弱點再投資**;但 hedge 仍可達成,不會卡關。

→ 取代舊版「混合 = 最低需求 × 4」與「×1.7 of sum」提案。**所有敵人 instance 的混合值需重新計算**。

### 本擊上限

> **依敵人強度決定本擊上限**。環境影響(連戰位置 / 場景)**後續詳細定義時加入**。

第一期暫定計算方式:
- 普通敵人(兔 / 狐 / 一般狼):**上限 = 該敵人最低類型需求 × 2**
- 精英敵人(菁英化狼 / 菁英化巨狼):**上限 = 該敵人最低類型需求 × 2.5**
- BOSS(序章前半 = 異變的巨狼):**上限 = 該敵人最低類型需求 × 3 + 5**

範例(序章前半敵人):
- 兔子(最低=穿刺 1) → 1 × 2 = **2 張**
- 狐狸(最低=2)→ 2 × 2 = **4 張**
- 狼(最低=穿刺 2)→ 2 × 2 = **4 張**
- 異變的巨狼(最低=衝擊 5)→ 5 × 3 + 5 = **20 張**
- 菁英化異變的巨狼 → 5 × 2.5 = **12.5 → 13 張**(四捨五入)

**設計含義**:防止「dump 全資源 fini boss」的策略;玩家被迫**精算本擊**。

→ 詳細數值待紙上測試 + 實機調整。

## 二、序章戰役具體規格

### 2-A. 序章主角身份 ✅ 已定案

**獵人之子**(從 [開頭劇情提案.md](開頭劇情提案.md) A1 獵人父子方向)。NPC = 父親。
- **主題**:森林、獵人父子、傳承
- **後半轉折**:第五連戰後父親獨自前往調查異變,主角獨自尋父

### 2-B. 序章前半結構(實作首要目標)

```
[戰役開場劇情(獵人父子入林)] 
  → 連戰 1:兔子 ×3(教學基準) + 教學重來機制
  → [整備 1:完全休息,卡片全部恢復]
  → 連戰 2:兔子 + 狐狸 ×2(教學「無弱點偏好時要投資源」)
  → [整備 2:+2 射箭 / +2 投石 + 狼嚎敘事]
  → 連戰 3:兔子 + 狐狸 + 狼(教學狼弱點 + 卡牌會消耗)
  → [整備 3:+2 射箭 / +2 投石 + 狼嚎漸近,狼身上爪痕敘事]
  → 連戰 4:狼 ×3(驗收狼運作)
  → [整備 4:+3 射箭 / +3 投石 + 父親將備用裝備全給兒子]
  → 連戰 5:兔子 ×2 + 異變的巨狼(預期菁英化,二次擊敗)
  → [序章前半結局劇情:父親拿走斷臂,留下主角獨自前行]
```

- **總場戰鬥**:11 場(序章前半單獨)
- **連戰數**:5 連戰
- **整備節點**:4 次(全部劇情驅動補給)
- **預期時長**:**獵人模式 30-45 分** / **騎兵模式 15-20 分**

### 2-C. 序章前半敵人配置

| 連戰 | 敵人 | 教學意圖 | 失敗代價 |
|------|------|--------|--------|
| 1 | 兔子 ×3 | 選卡出卡結算邏輯 + 弱點對應練習 | 連鎖逃跑 → 教學重來 |
| 2 | 兔子 + 狐狸 ×2 | 辨識弱點(狐狸無偏好) | 兔逃跑連鎖、狐逃跑無連鎖 |
| 3 | 兔子 + 狐狸 + 狼 | 卡牌消耗 + 狼弱點 | 狼失敗 → 菁英化排末 |
| 4 | 狼 ×3 | 驗收狼運作 | 狼失敗 → 菁英化排末 |
| 5(BOSS)| 兔子 ×2 + 異變的巨狼 | 菁英化運作 + BOSS 投入 | 巨狼失敗 → 菁英化(預期);菁英化失敗 → GAME OVER |

詳細敵人模板規格 → [敵人資料庫.md](敵人資料庫.md)
詳細卡組與補給規格 → [卡牌資料庫.md](卡牌資料庫.md)

### 2-D. 序章後半結構(待設計,實作次要目標)

- **核心方向**:主角獨自尋找父親 + 調查森林異變
- **卡組構成**:卡牌可能變化(主角的家傳裝備有限,可能引入新類型)
- **敵人**:可能引入舊版 5 模板的部分(哨兵 / 巡守)
- **長度**:約序章前半的 70-100%(待設計)
- **實作優先順序**:序章前半完成 + 紙上測試通過後才設計後半

### 2-E. 自包含完整短篇要求

依 [遊戲設計概念.md](遊戲設計概念.md) §2 #10 的原則:**序章本身必須是一個自包含的完整短篇**——有起承轉合、有明確結局、有「主角達成了什麼」的閉合感,**同時留下通往後續戰役的鉤子**,但不能依賴後續戰役才能稱為「完整」。

序章前半的閉合感:**「兒子完成了第一次真正的狩獵,但父親離他而去」** — 完成 + 鉤子並存。

## 三、預留架構(資料結構支援但不做內容)

第一期實作時**資料結構與 UI 框架要支援**以下功能,但**內容只放序章**:

| 預留項 | 預留方式 |
|--------|---------|
| **多角色系統** | 卡牌資料結構支援「通用牌 vs 角色專屬牌」標記;Hub 預留「角色選擇」介面雛形(只一個選項) |
| **多戰役串接** | 戰役清單資料結構可加新戰役;Hub「戰役選擇」介面雛形 |
| **章節回顧** | 完成戰役後資料標記為「已完成」;回顧介面雛形 |
| **紀念品擴充** | Hub 紀念品展示位可容納多個;資料結構可加新紀念品 |
| **多存檔位** | 存檔系統設計時假設 per-角色 / per-戰役,雖然第一期只有一組 |
| **跳過序章功能** | 戰役有「可跳過」flag(第一期序章為 false) |

> 原則:**框架做好,後續加內容容易**。第一期不要把「只有 1 個」寫死成「只能 1 個」。

## 五、開發優先順序

   - 類型計數結算
   - 28 張卡的資料層 + 行為

   - 連戰結構
   - 失敗代價依類別差異化
   - 資源持續縮減失敗模型

### Phase 1B:敘事層(核心 + 內容)

3. **NPC 對話系統**
   - 戰後校準雙聲道
   - 6 種對話狀態判定
   - 異步呈現機制
   
4. **重要劇情系統**
   - 戰役開場 / 中段 / 末段劇情播放
   - 道中對話穿插

5. **Hub 雛形**
   - 基本場景 + NPC
   - 戰役選擇介面(只有序章)
   - 紀念品展示位

### Phase 1C:教學與輔助系統

6. **教學期 UI**
   - 強引導(灰階卡、提示文字)
   - 教學工具逐場解鎖

7. **戰鬥紀錄系統**
   - 戰鬥紀錄基本查閱
   - 失敗痕跡保留

8. **紀念品系統**
   - 完成序章 = 自動觸發紀念品
   - Hub 展示(無未解鎖位、無進度條)

### Phase 1D:打磨與驗證

9. **音效 / 動畫**
   - OHK 動畫 + 結算演出
   - 對話音效
   - 必要的環境音

10. **完整驗證**
    - 獵人模式跑完驗證(25-30 分)
    - 騎兵模式跑完驗證(8-15 分)
    - 失敗重啟流程驗證
    - 戰役失敗 → 重打整場驗證
    - 雙模式 UI 切換驗證

## 七、驗證標準

第一期完成後,**必須通過以下驗證才能上市**:

### 機制驗證

- [ ] 5 隻敵人都有至少 2 條擊殺路徑(弱點 / 混合)
- [ ] 28 張卡都有合理使用情境
- [ ] Place/Lock 機制玩家可直觀理解
- [ ] 失敗代價依類別差異化能在實際遊玩中感受到
- [ ] 資源持續縮減模型在 2-3 次失敗後仍可遊玩

### 節奏驗證

- [ ] 獵人模式完整跑完 25-30 分鐘,節奏感舒適
- [ ] 騎兵模式完整跑完 8-15 分鐘,流暢無打斷
- [ ] 道中對話不打斷戰鬥節奏
- [ ] 戰後校準對話異步播放正確

### 體驗驗證

- [ ] 新玩家能在無外部教學下完成序章(獵人模式)
- [ ] 至少 3 位玩家測試後通關率 > 60%
- [ ] 玩家失敗時能直覺理解原因(校準對話有效)
- [ ] 雙模式 UI 切換流暢,新玩家不會誤觸

### 敘事驗證

- [ ] 序章作為自包含短篇有完整起承轉合
- [ ] NPC 對話量級適中(40-50 句不會嫌煩)
- [ ] 紀念品事後呈現,玩家不會「為收集而玩」
