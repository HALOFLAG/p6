## 一、第一期 Scope 表

### 必做內容(第一期上市需具備)

| 範疇 | 規格 |
|------|------|
| **戰役** | 1 場(序章,亦為教學戰役) |
| **角色** | 1 個(序章主角,身份待定) |
| **卡組** | 28 張(全部為序章主角卡組) |
| **敵人** | 5 隻原型模板(哨兵 / 巡守 / 重甲衛兵 / 伏擊者 / 領主) |
| **戰鬥系統** | Place/Lock + 類型計數結算 + 失敗代價依類別差異化 |
| **戰役失敗機制** | 資源持續縮減模型(玩家自選重新挑戰) |
| **整備節點** | 存檔(暫停退出用)+ 校準對話 + 小劇情段落 |
| **戰後校準對話** | 雙聲道 + 異步呈現 + 6 種對話狀態 + 約 40-50 句 |

| **教學 UI** | 強引導路線(灰階 + 提示 + 逐步解鎖工具) |
| **雙模式 UI** | 雛形:獵人模式 + 完成序章後解鎖騎兵模式設置 |
| **戰鬥紀錄系統** | 基本版(查閱本擊組合 + 結果差距 + 失敗痕跡保留) |
| **紀念品系統** | 序章對應的紀念品(事後出現,不顯示未解鎖位) |
| **NPC 對話** | 序章戰役 + Hub 對話總量約 50-60 句 |

### 不在第一期 Scope

| 範疇 | 為什麼不做 |
|------|----------|
| **多角色** | 第一期專注完成序章主角的完整體驗 |
| **戰役 2+** | 第一期僅序章 |
| **競技場模式** | 後續期數作為延伸 |
| **挑戰 / 支線模式** | 後續期數作為延伸 |
| **跳過序章功能** | 第一期只有序章可玩,跳過無意義 |
| **完美通關特殊紀念品** | 第一期紀念品只有「完成戰役」的基本版 |
| **難度系統** | 後續視玩家反饋決定 |
| **多存檔位** | 第一期單一存檔(per 角色) |

## 一-A、實作基準確定的平衡公式(2026-05-13)

### 混合任意需求公式

> **混合需求 = 該敵人最高類型需求 × 1.5,取整數**

範例:
- 哨兵 弱衝擊低(衝擊 1 / 穿刺 4)→ 混合 = 4 × 1.5 = **6**
- 巡守 弱衝擊高(衝擊 4 / 穿刺 7 / 燃燒 7)→ 混合 = 7 × 1.5 = 10.5 → **11**
- 領主 弱穿刺極高(衝擊 9 / 穿刺 6)→ 混合 = 9 × 1.5 = 13.5 → **14**

**設計含義**:hedge(走混合任意)= 走錯類型的成本約 +50%。**鼓勵玩家先揭露弱點再投資**;但 hedge 仍可達成,不會卡關。

→ 取代舊版「混合 = 最低需求 × 4」與「×1.7 of sum」提案。**所有敵人 instance 的混合值需重新計算**。

### 本擊上限

> **依敵人強度決定本擊上限**。環境影響(連戰位置 / 場景)**後續詳細定義時加入**。

第一期暫定計算方式:
- 普通敵人(兔 / 一般狼):**上限 = 該敵人最低類型需求 × 2**
- 精英敵人(精英狼):**上限 = 該敵人最低類型需求 × 2.5**
- BOSS(熊):**上限 = 該敵人最低類型需求 × 3 + 5**
- 預設精英(豹):**上限 = 該敵人最低類型需求 × 2.5**

範例:
- 哨兵 弱衝擊低 → 上限 = 1 × 2 = **2 張**
- 重甲衛兵 弱衝擊高 → 上限 = 4 × 2 = **8 張**
- 領主 弱穿刺極高 → 上限 = 6 × 3 + 5 = **23 張**

**設計含義**:防止「dump 全資源 fini boss」的策略;玩家被迫**精算本擊**。

→ 詳細數值待紙上測試 + 實機調整。

## 二、序章戰役具體規格

### 結構配比

採用 **γ 三段升級**(對應教學需求 + 漸進負擔):

```
[戰役開場劇情(2 分)] 
  → 短連戰 1:2 場戰鬥 + 道中對話 ×2
  → [整備節點 1(1.5 分)] 
  → 中連戰 2:3 場戰鬥 + 道中對話 ×3
  → [整備節點 2 + 重要劇情(2.5 分)] 
  → 長連戰 3:3 場戰鬥(含 boss) + 道中對話 ×3
  → [戰役末劇情(2.5 分)]
```

- 總場戰鬥:6 場(2 + 3 + 3)
- 預期時長:**獵人模式 25-30 分** / **騎兵模式 8-15 分**
- 整備節點:2 次
- 重要劇情段:3 段(開場 / 中段 / 末段)

### 敵人配置(初步建議)

| 場次 | 敵人 | 類別 | 教學意圖 |
|------|------|------|--------|
| 1 | 哨兵 | 兔子 | 弱點對應的次數差距感(低需求,免費學習場) |
| 2 | 巡守(先攻變體) | 狼 | 戰鬥狀態的影響(免費情報的價值) |
| 整備 1 | — | — | 校準對話 + 場景轉場 + 卡牌回補 |
| 3 | 重甲衛兵 | 狼 | 弱點對應的投資價值(對應 vs 不對應的成本差) |
| 4 | 巡守(被襲變體) | 狼 | 戰鬥狀態的影響(資訊封鎖的代價) |
| 5 | 重甲衛兵 | 狼 | 可選失敗示範場(玩家若失誤,精英化排末) |
| 整備 2 + 劇情 | — | — | 重要劇情段:暗示更強敵人 + 卡牌回補 |
| 6 | 伏擊者 | 豹 | 「不能失敗」的高張力時刻 |
| 7 | 巡守 | 狼 | 過渡場(視覺暗示中需求標準戰) |
| 8 (BOSS) | 領主 | 熊 | 爆發牌與角色牌的投入時機(boss 戰) |

> 註:5 模板 6 場戰鬥 = 部分模板重複使用(視覺變體),例如重甲衛兵在場 3、場 5 出現,巡守在場 2、4、7 出現(不同戰鬥狀態變體)。

### 序章主角身份

**待定**——從 [開頭劇情提案.md](開頭劇情提案.md) 的 17 個候選方向選定一個。

待選方向會影響:
- 28 張卡的命名與視覺(目前是泛用名,需特化)
- 6 場戰鬥的場景與敘事
- NPC 角色身份
- Hub 視覺風格

### 自包含完整短篇要求

依 [遊戲設計概念.md](遊戲設計概念.md) §2 #10 的原則:**序章本身必須是一個自包含的完整短篇**——有起承轉合、有明確結局、有「主角達成了什麼」的閉合感,**同時留下通往後續戰役的鉤子**,但不能依賴後續戰役才能稱為「完整」。

## 三、預留架構(資料結構支援但不做內容)

第一期實作時**資料結構與 UI 框架要支援**以下功能,但**內容只放序章**:

| 預留項 | 預留方式 |
|--------|---------|
| **多角色系統** | 卡牌資料結構支援「通用牌 vs 角色專屬牌」標記;Hub 預留「角色選擇」介面雛形(只一個選項) |
| **多戰役串接** | 戰役清單資料結構可加新戰役;Hub「戰役選擇」介面雛形 |
| **章節回顧** | 完成戰役後資料標記為「已完成」;回顧介面雛形 |
| **紀念品擴充** | Hub 紀念品展示位可容納多個;資料結構可加新紀念品 |
| **多存檔位** | 存檔系統設計時假設 per-角色 / per-戰役,雖然第一期只有一組 |
| **跳過序章功能** | 戰役有「可跳過」flag(第一期序章為 false) |

> 原則:**框架做好,後續加內容容易**。第一期不要把「只有 1 個」寫死成「只能 1 個」。

## 四、美術 / 內容製作需求

### 視覺資產

| 類別 | 數量 | 備註 |
|------|------|------|
| 卡牌正面 | 28 張 | 含 23 通用 + 5 角色專屬 |
| 卡牌背面 | 1 種 | 統一卡背 |
| 敵人視覺 | 5 隻 × 若干 instance 變體 | 哨兵 4、巡守 9、重甲 6、伏擊者 4、領主 4 個 instance |
| 戰鬥場景 | 3-4 個 | 對應序章不同連戰段 |
| Hub 場景 | 1 個基礎 + 完成後變體 | 演化用 |
| NPC 立繪 | 1 個 + 表情 / 姿勢變體 | Hub 顯示用 |
| 紀念品視覺 | 1-3 個 | 序章對應的紀念品 |
| UI 元件 | 一套 | 卡牌槽、本擊區、結算動畫、需求 bar 等 |

### 文字內容

| 類別 | 估算量 |
|------|------|
| 重要劇情段(開場 / 中段 / 末段) | 3 段 × 300-500 字 ≈ 1000-1500 字 |
| 道中對話 | 5-7 次 × 50-100 字 ≈ 300-700 字 |
| 戰後校準對話 | 約 40-50 句(核心 6 + 模板 1-2 × 5 模板)|
| Hub NPC 對話 | 約 10-15 句(初次見 / 完成後 / 等待中等場景)|
| 卡牌描述文字 | 28 張 × 短描述 |
| 敵人描述文字 | 5 模板 × 短描述 |
| UI 文字 | 設置 / 提示 / 教學引導文字 |
| **總計** | **約 3000-4000 字**(中等量,可手寫,不需動態生成) |

### 音效 / 動畫

| 必要項 | 備註 |
|--------|------|
| OHK 擊殺動畫 | 1-2 秒,**動畫速度可調** |
| 結算 bar 填充動畫 | 視覺強化「逼近達標線」 |
| 卡牌拖放動效 | Place / Lock 視覺反饋 |
| 道中對話進場音 | 簡短提示音 |
| 戰後校準音效 | 與對話同步 |
| Hub 環境音 | 低密度氛圍音 |
| 失敗音效 | 不同類別(兔逃走 / 狼精英化 / 豹熊 GO) |

## 五、開發優先順序

### Phase 1A:核心遊玩(必須先做)

1. **戰鬥核心系統**
   - Place/Lock 機制
   - 類型計數結算
   - 28 張卡的資料層 + 行為
   - 5 隻敵人模板的資料層 + 行為
   
2. **戰役框架**
   - 連戰結構
   - 失敗代價依類別差異化
   - 資源持續縮減失敗模型
   - 整備節點機制(存檔 + 回補)

### Phase 1B:敘事層(核心 + 內容)

3. **NPC 對話系統**
   - 戰後校準雙聲道
   - 6 種對話狀態判定
   - 異步呈現機制
   
4. **重要劇情系統**
   - 戰役開場 / 中段 / 末段劇情播放
   - 道中對話穿插

5. **Hub 雛形**
   - 基本場景 + NPC
   - 戰役選擇介面(只有序章)
   - 紀念品展示位

### Phase 1C:教學與輔助系統

6. **教學期 UI**
   - 強引導(灰階卡、提示文字)
   - 教學工具逐場解鎖
   - 完成序章後解鎖精簡 UI 設置

7. **戰鬥紀錄系統**
   - 戰鬥紀錄基本查閱
   - 失敗痕跡保留
   - 精英敵人對戰前可調出上次資料

8. **紀念品系統**
   - 完成序章 = 自動觸發紀念品
   - Hub 展示(無未解鎖位、無進度條)

### Phase 1D:打磨與驗證

9. **音效 / 動畫**
   - OHK 動畫 + 結算演出
   - 對話音效
   - 必要的環境音

10. **完整驗證**
    - 獵人模式跑完驗證(25-30 分)
    - 騎兵模式跑完驗證(8-15 分)
    - 失敗重啟流程驗證
    - 戰役失敗 → 重打整場驗證
    - 雙模式 UI 切換驗證

## 七、驗證標準

第一期完成後,**必須通過以下驗證才能上市**:

### 機制驗證

- [ ] 5 隻敵人都有至少 2 條擊殺路徑(弱點 / 混合)
- [ ] 28 張卡都有合理使用情境
- [ ] Place/Lock 機制玩家可直觀理解
- [ ] 失敗代價依類別差異化能在實際遊玩中感受到
- [ ] 資源持續縮減模型在 2-3 次失敗後仍可遊玩

### 節奏驗證

- [ ] 獵人模式完整跑完 25-30 分鐘,節奏感舒適
- [ ] 騎兵模式完整跑完 8-15 分鐘,流暢無打斷
- [ ] 道中對話不打斷戰鬥節奏
- [ ] 戰後校準對話異步播放正確

### 體驗驗證

- [ ] 新玩家能在無外部教學下完成序章(獵人模式)
- [ ] 至少 3 位玩家測試後通關率 > 60%
- [ ] 玩家失敗時能直覺理解原因(校準對話有效)
- [ ] 雙模式 UI 切換流暢,新玩家不會誤觸

### 敘事驗證

- [ ] 序章作為自包含短篇有完整起承轉合
- [ ] NPC 對話量級適中(40-50 句不會嫌煩)
- [ ] 紀念品事後呈現,玩家不會「為收集而玩」
