# 結算機制演化討論

## 文件目的與狀態

本文件記錄 P6 結算機制從「**倍率傷害模型**」演化到「**類型計數模型**」的過程,作為設計決策的歷史依據。

**結論:採用類型計數模型**(2026-05-10 確立)。原倍率模型保留作為日後檢討時的對照素材。

---

## 一、原始倍率模型(已棄用)

### 機制定義

- **閾值** = 單一數字(如 35)
- **卡牌** = `base_strength`(數字)+ 類型 + 詞條
- **類型修飾** = 對應 ×1.5 / 不對應 ×0.5
- **詞條修飾** = 條件性乘數(如「獨擊×1.5」「已揭露×1.3」「本擊含情報×1.3」)
- **結算** = `Σ(每張卡的 base × 類型修飾 × 詞條修飾)`,結果 ≥ 閾值即 OHK

### 設計用意

- 「單一檢查」原則的具體化:閾值是核心,類型與詞條是修飾
- 多軸組合提供策略深度(資源類別 × 功能類別 × 弱點類型 × 強度層級 × 詞條)
- 詞條鼓勵特定戰術(獨擊、情報投資、跨場揭露)

### 第一次紙上測試的暴露問題(2026-05-10)

**測試結果**:5 場戰役,3/5 OHK 成功,但每場都暴露機制問題。

#### 問題 1:結算負擔過重
- 每張卡需算 `base × 類型修飾 × 詞條修飾`,4 張卡本擊 = 12 次計算
- 紙上原型每次 commit 後計算耗時 30 秒,**摧毀了「瞬間即儀式」的 OHK 美感**
- 即使數位實作能秒算,動畫要把多卡個別效果視覺凝聚成「一擊」的設計負擔依然極重

#### 問題 2:詞條觸發率極低
- 「已揭露」詞條:5 場戰鬥 4 張不同的已揭露卡進入本擊,只觸發 2 次。直接靠情報卡先揭露的次數 = **0**
- 「獨擊」詞條:玩家在最重要的最終戰本能傾巢而出(4 張),獨擊條件(commit_size==1)永遠不觸發
- **詞條的觸發鏈條對玩家認知負擔太重**,實戰中沒人這樣玩

#### 問題 3:卡牌效果不直覺
- 詞條與特殊效果的命名(獨擊、已揭露、觀敵之刃、焚識...)**沒有 communicate 行為**
- 玩家無法從卡名推測效果,必須先記憶系統規則
- 違反「玩家是判斷者」的核心精神——玩家被迫先當「規則記憶者」

#### 問題 4:玩家心智模型與設計意圖偏差
- 設計意圖:玩家用視覺+元知識做精準判斷,選對類型 = 翻倍效率
- 實際行為:玩家「不確定就堆量」「不確定就 hedge 多種類型」
- 結果:5 場戰鬥落點 = 大幅 Overkill ×2 + 勉強過 ×1 + 差一點 ×2,**精準擊殺 0 次**
- 玩家無法在「對應 vs 數量」之間建立直覺

#### 問題 5:閾值的視覺暗示太粗
- 視覺只能傳達「中閾值」這個層級(35-65),**玩家無法從視覺區分 35 vs 50 vs 65**
- 結果:玩家從前場經驗推估,而非從視覺判讀

#### 問題 6:卡池過剩
- 30 張卡對 5 場戰鬥的原型來說數量太多(只用 9/30 = 30%)
- 玩家無法快速建立「卡庫熟悉度」
- 認知負擔 + 篩選缺失 → 決策困難

## 二、過渡考量:純資源池模型

玩家在測試後提出:**把卡牌類型化(只分 3 種傷害)、玩家決策變成「投幾張」而非「選哪張」**。

這個方向直接解決了問題 1、3、4、6,但會喪失:
- 詞條系統(無處安放)
- 角色卡的「招牌動作」感
- 卡牌池的多變性
- 兩軸分類的設計亮點

---

## 三、最終方案:類型計數模型(採用)

### 機制定義

- **閾值** = 各類型所需次數的字典(如:衝擊 2 / 穿刺 5 / 燃燒 6 / 混合任意 7)
- **卡牌** = `(類型, 次數貢獻)` + 個體特殊效果
- **結算** = 比對玩家本擊在各類型的次數總和,**任一條件達標即 OHK**
- **無倍率、無詞條**

### 範例

**敵人**:衝擊 2 / 穿刺 5 / 燃燒 6 / 混合 7

- 玩家 commit「2 衝擊卡」→ 衝擊計數 = 2 ✓ OHK
- 玩家 commit「5 穿刺卡」→ 穿刺計數 = 5 ✓ OHK(較貴)
- 玩家 commit「混合 7 張」→ 混合 = 7 ✓ OHK(很貴,代表「不知弱點時的保險路徑」)
- 玩家 commit「1 衝擊 + 3 雜」→ 各類型都不達標 ✗ Underkill

### 為什麼這個方案優於前兩者

| 解決的問題 | 倍率模型 | 純資源池 | 類型計數 |
|----------|---------|---------|---------|
| 結算負擔 | ✗ 重 | ✓ 輕 | ✓ 輕(純加總比對) |
| 詞條問題 | ✗ 觸發率低 | ✓ 不存在 | ✓ 不存在(暫時擱置) |
| 卡牌效果直覺性 | ✗ 不直覺 | ✓ 直覺 | ✓ 直覺(類型+次數) |
| 玩家心智模型 | ✗ 判斷者 vs 保險者偏差 | ✓ 比例分配易懂 | ✓ 「需要幾張」更直覺 |
| 「對應弱點」價值 | ✓ 倍率傳達 | ✗ 弱化 | ✓ 用次數差距傳達 |
| 卡牌個體性 | ✓ 完整 | ✗ 大幅喪失 | ✓ 角色/爆發/情報保留 |
| 招牌動作儀式化 | △ 與獨擊詞條矛盾 | ✗ 弱化 | ✓ 個體卡保留 |
| 試探/情報玩法 | △ 詞條依賴 | ✗ 無處安放 | ✓ 個體卡複合貢獻 |

## 四、新模型對舊設計的繼承與切割

### 繼承(精神不變)

- OHK 美學的 5 原則(減法、對稱、瞬間、理解、變化)
- 視覺線索詞彙表(弱點類型概念不變)
- 敵人模板的層 1-3 視覺資訊架構
- 戰後校準對話系統
- 兩軸分類的「資源類別」(工具/爆發/角色,但結構簡化)
- 整備期回補機制(只是回補對象從「使用次數」變「池內張數」)

### 切割(機制重做)

- **倍率系統** → 完全棄用
- **詞條系統** → 暫時棄用(可作為日後擴充考慮)
- **base_strength 數字** → 改為 `次數貢獻`(整數 1-3)
- **「閾值=單一數字」** → 改為 `各類型需求字典`
- **結算分層** 從三層(情報/修飾/戰鬥)→ 兩層(情報/戰鬥)
- **兩軸分類**:資源類別保留,功能類別「複合」轉化為「個體卡複合貢獻」

## 五、給未來重新檢討時的提醒

如果未來決定回頭檢討倍率模型或加入詞條,要先回答:

1. **詞條的觸發機制如何避免「前場準備、本場觸發」的兩段投資鏈?**
2. **多卡乘法計算如何在動畫上凝聚成「一擊」的瞬間感?**
3. **卡牌效果的命名與視覺如何 communicate 行為而非概念?**
4. **如何避免玩家從「保險者」心智模型對抗設計意圖?**

如果這些問題沒有答案,**不要回到倍率模型**——它們是這次測試明確驗證的失敗點。
