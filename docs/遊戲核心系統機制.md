## 一、戰鬥三子階段:Place / Lock / 結束本擊

每場戰鬥是「**一個 commit moment**」:玩家從手上挑選若干張卡組成「本擊」,一次提交,所有效果同時結算。

### 「本擊」的定義

本擊 = 玩家在單場戰鬥中,從手上挑出並一次性提交的卡牌組合。本擊是設計上的最小單位,不是個別卡牌:

- 揭露效果依「本擊」內的卡綜合判定
- 結算結果依「本擊」整體類型次數決定
- 玩家的決策對象是「本擊」而非「個別卡」

### 卡牌的 lock_class 三分類

每張卡的 `lock_class` 決定它如何與本機制互動:

| lock_class | 行為 | 卡面標示 | 適用卡類 |
|-----------|------|---------|---------|
| **`"none"`** | Lock 與否完全不影響效果 | 無 lock 相關標示 | 純戰鬥卡(資源池、純爆發、純角色)= 19 張 |
| **`"optional"`** | Lock 與否**改變揭露對象**:不 lock = 廣度;Lock = 深度 | 卡面寫兩種模式效果 | 情報卡(偵察 / 估量 / 警示)= 3 張 |
| **`"required"`** | **必須先 lock 才能生效** | 卡面標 🔒 強制 lock | 試探卡 / flexible 卡 / 全知之眼 / 洞察一擊 = 6 張 |

### 三個子階段

1. **Place 階段**:玩家拖卡到本擊區,可隨時編輯。系統不結算任何效果,不消耗資源。flexible 卡顯示「未分配」狀態。

2. **動態 Lock 階段**:玩家對單張卡點「鎖定」:
   - **`"none"` 卡**:可 lock 也可不 lock,效果不變
   - **`"optional"` 卡**:lock 觸發 locked 模式;不 lock 則於「結束本擊」觸發 unlocked 模式
   - **`"required"` 卡**:**必須在此階段 lock**(否則「結束本擊」按鈕禁用)。lock 時 special_effect 立即觸發,flexible 必須完成分配
   
   Lock 不可撤回。Lock 過程中玩家可繼續新增 placed 卡或 lock 其他卡。

3. **結束本擊**:玩家點「結束本擊」按鈕(若有 required 卡未 lock 則禁用),系統鎖定所有未 lock 的 a/b 類 placed 卡,然後在戰鬥層整體計算各類型次數總和,逐項比對敵人需求表;**任一條件達標即 OHK**。
   - 期間若 b 類卡未 lock,觸發其 unlocked 模式 special_effect

### 結算範例

敵人需求表:衝擊 2 / 穿刺 5 / 燃燒 6 / 混合任意 7

玩家本擊流程:

```
1. [Place] 玩家置入:洞察一擊、重斬、直擊
   本擊狀態:衝擊 0(尚未分配 + 未結算)

2. [Lock 洞察一擊]
   → 觸發 special_effect:揭露當前敵人弱點 = 衝擊
   → 洞察一擊資源消耗(角色卡用掉)
   → 玩家現在知道弱點

3. [玩家分配洞察一擊的 flexible]
   洞察一擊 +2 → 分配 +2 衝擊

4. [結束本擊]
   → 鎖定剩餘:重斬、直擊
   → 戰鬥層整體計算:重斬 +2、直擊 +1、洞察 +2 = 衝擊 5
   → 衝擊計數 = 5(需求 2)→ ✓ 達標
   → 結果:OHK
```

### Lock 順序的策略意義

Lock 順序本身是判斷的一部分:
- 先 lock 揭露卡 → 獲取資訊優勢 → 後續決策更精準
- 先 lock 戰鬥卡 → 喪失資訊優勢,但表達「我已決定」的清晰承諾

設計上不限制順序,讓玩家自己經營「資訊→決策」的節奏。

### 設計含義

Place/Lock 機制讓「揭露」與「決策」之間有自然空隙——玩家做出**多個子動作**,但這些子動作仍構成「**一個本擊**」。這跟「分次戰鬥」是兩回事——玩家是在同一 commit 階段內讓資訊流動,不是發起多次 commit。對應 OHK 美學的「物理動作鏈」精神:一旦動作開始(lock)就不可收回,但其他動作可以基於資訊調整。

## 二、類型計數結算模型

### 為什麼這個模型

P6 採用「**單一檢查原則**」——每場戰鬥的勝敗判定收斂到一個結算瞬間,玩家在 commit 前面對的是「選哪條路」的單一決策,而非「同時滿足多個獨立條件」。

### 具體機制

- 每個敵人有一張**需求表**:每個傷害類型需要多少「次數」才能擊殺
- 加上一個「**混合任意需求**」:不限類型只比總次數
- 玩家本擊只要在任一條件達標就 OHK
- 所有條件指向同一個結算瞬間

### 結算範例

敵人:衝擊需求 2 / 穿刺需求 5 / 燃燒需求 6 / 混合需求 7

- 玩家 commit 2 張衝擊卡 → 衝擊計數 2 ≥ 2 → ✓ OHK(最省)
- 玩家 commit 5 張穿刺卡 → 穿刺計數 5 ≥ 5 → ✓ OHK(較貴)
- 玩家 commit 7 張任意混合卡 → 混合計數 7 ≥ 7 → ✓ OHK(很貴,「不知弱點時的保險路徑」)
- 玩家 commit 1 衝擊 + 3 雜類 → 各條件都不達 → ✗ Underkill

### 為什麼這個解法最好

- 簡單(玩家不需算乘法,純加總比對)
- 「對應弱點」價值清晰可見(衝擊只要 2 vs 穿刺要 5 = 弱點顯而易見)
- **勇敢直打仍可行**(走混合任意路徑,代價高但能成功)
- **謹慎優化也可行**(揭露弱點 → 走最低需求路徑)
- 沒有「對不起,你必須有衝擊類卡才能繼續」的鎖鑰問題
- 結算瞬間直觀:玩家看到「填滿哪個需求 bar」就立即知道結果

### 平衡原則:混合任意需求(2026-05-13 定案)

> **混合需求 = 該敵人最高類型需求 × 1.5,取整數**

範例:
- 巡守 弱衝擊高(衝擊 4 / 穿刺 7 / 燃燒 7)→ 混合 = 7 × 1.5 = 10.5 → **11**
- 領主 弱穿刺極高(衝擊 9 / 穿刺 6)→ 混合 = 9 × 1.5 = 13.5 → **14**

**設計含義**:hedge(走混合任意)約為走錯類型的 +50% 成本。**鼓勵玩家先揭露弱點再投資**;但 hedge 仍可達成,不會卡關。

→ 取代舊版「混合 = 最低需求 × 4」與「×1.7 of sum」提案。詳見 [敵人設計原則.md](敵人設計原則.md) 第六章。

### 本擊上限(2026-05-13 定案,暫定)

> **依敵人強度決定本擊上限**。環境影響(連戰位置 / 場景)後續加入。

| 敵人類型 | 上限計算 |
|---------|--------|
| 普通敵人(兔 / 一般狼)| 最低類型需求 × 2 |
| 精英敵人(精英狼)| 最低類型需求 × 2.5 |
| 預設精英(豹)| 最低類型需求 × 2.5 |
| BOSS(熊)| 最低類型需求 × 3 + 5 |

**設計含義**:防止「dump 全資源 fini boss」策略;玩家被迫**精算本擊**。具體數值待紙上測試 + 實機調整。

### 規則應用範例

未來若想加入「敵人有護盾,先破護盾再攻擊」這種機制,要先檢查是否違反原則:
- ❌ 兩個獨立階段(先破盾才能攻擊)→ 違反
- ✓ 護盾把某類型需求 +N 次 → 符合(單一檢查,護盾是調整需求,不是新階段)

## 三、敵人四層資訊架構

每個敵人的資訊以四層結構組織。玩家對敵人的「了解程度」由各層的揭露狀態決定。

### 層 1 — 模板永久顯性

**立即可見**,定義敵人的「大類」:

- 大類識別(輕甲偵察 / 重甲近戰 / 等)
- 敵人類別(兔子 / 狼 / 豹 / 熊)
- 弱點範圍(可單獨擊殺的類型)
- 弱點需求層級範圍(低/中/高)
- 戰鬥狀態傾向

### 層 2 — 次級視覺線索

**老手用眼讀**,需要玩家學習視覺語言:

- 持械暗示弱點類型(持矛 → 穿刺)
- 鎧甲狀態暗示需求層級(殘破皮甲 → 低)
- 姿態暗示戰鬥狀態(蹲伏潛行 → 被襲)
- 物種特徵暗示敵人類別

> 完整視覺線索對應表見 [敵人設計原則.md](敵人設計原則.md) 第四章。

### 層 3 — 即時狀態

**進入戰鬥可見**:

- 戰鬥狀態標籤(本場是標準/先攻/被襲)
- 上一場用情報牌揭露的相關資訊(跨場持續)
- 若為精英化的狼:UI 標籤「精英化 ⚠」+ 戰鬥紀錄可調出上次資料

### 層 4 — 隱藏狀態

**僅情報牌可揭**:

- 完整需求表(各類型精確次數)
- 混合任意需求
- 敵人類別(可透過警示卡揭露)

### 「看得懂 vs 看不懂」的張力

老手讀層 2 → 不投資情報 → 仍能對應弱點擊殺
新手讀不出層 2 → 必須投資情報 → 或盲打走混合任意路徑

這是元知識的核心舞台——P6 的「進度」就是讀層 2 能力的累積。

## 四、戰鬥狀態系統:先攻 / 標準 / 被襲

每場戰鬥開始時的處境本身是個戰術變數。

### 三種戰鬥狀態

| 狀態 | 效果 | 適用 | 設計感 |
|------|------|------|-------|
| **先攻** | 玩家獲得免費的前置揭露(一張情報牌等價,不消耗手牌) | 連戰第一個敵人、易偵察的敵人 | 玩家有準備、佔上風 |
| **標準** | 直接 commit | 大部分敵人 | 基本對峙 |
| **被襲** | 部分情報效果無效,或必須投入更多卡牌 | 埋伏者、刺客類敵人 | 措手不及 |

### 為什麼這個分類有用

讓敵人類型不只有「需求表多少」這個維度,還有「如何進入戰鬥」這個維度,敵人個性立刻豐富。

### 待決定的設計問題

- 戰鬥狀態是依敵人類型固定?
- 還是依玩家某些選擇影響?
- 或是隨機(input randomness 意義上)?

目前原型階段:由敵人模板的「戰鬥狀態傾向」決定 instance 機率分布。

## 五、失敗代價與精英機制(四類敵人差異化)

> 完整機制取代了舊版「10% 隨機精英化」設計(已棄用)。

### 核心邏輯

> **並非所有敵人都會殺你——但會記恨的敵人會以精英形態回歸,而預設精英與 Boss 失敗即終戰。**

失敗代價**依敵人類別**(兔子 / 狼 / 豹 / 熊)差異化。

### 敵人四類分類

| 類別 | 範例 | 失敗代價 | 敘事邏輯 |
|------|------|---------|---------|
| **A. 兔子(無威脅普通)** | 哨兵 | 卡牌消耗;敵人逃走,**不再回來** | 「沒有殺意」的對手 |
| **B. 狼(威脅普通)** | 巡守、重甲衛兵 | 卡牌消耗 + 變精英回歸 + 精英失敗 = GO | 「會記恨並學習」的對手 |
| **C. 豹(預設精英)** | 伏擊者 | **失敗即 GAME OVER** | 「本來就要殺你」的對手 |
| **D. 熊(Boss / 劇情敵人)** | 領主 | **失敗即 GAME OVER** | 戰役節奏的關鍵 |

### 狼型三層失敗代價

**第一層 — 連戰內第 1 次失敗**

```
1. 投入的卡牌依然消耗
2. 該敵人標記為「精英狀態」
3. 排到該連戰末段(熊型 Boss 之前)
4. 連戰場次 +1
玩家會在該連戰結束前再次面對它(精英化)
```

**第二層 — 連戰內第 2+ 次失敗**

```
1. 投入的卡牌依然消耗
2. 該敵人「移出該連戰」進入「未解決池」
3. 連戰繼續進行
連戰結束後玩家會面對「追擊選擇」
```

**第三層 — 連戰結束 → 追擊機制**(詳見第六章)

### 精英敵人的特性

- **外觀數值與原本完全相同**,只有 UI 標籤「精英化 ⚠」標示
- 失敗 = GAME OVER
- 戰鬥紀錄可調出上次面對的資料

### 精英敵人的兩種來源

| 來源 | 範例 | 視覺 | 玩家心理 |
|------|------|------|---------|
| **狼變的精英**(失敗造成) | 失敗的巡守、失敗的重甲衛兵 | 跟原本一樣 + UI 標籤 | 「我記得它,我有戰鬥紀錄資料」 |
| **豹型預設精英**(本來就在) | 伏擊者 | 物種本身的精英級視覺 | 「我第一次遇到,但它一招斃命」 |

### Game Over 條件

任何精英敵人 + 豹型 + 熊型失敗 = GAME OVER:

| 情境 | 失敗結果 |
|------|---------|
| 連戰中面對狼型精英失敗 | GAME OVER |
| 追擊狼型時失敗 | GAME OVER |
| 下連戰面對替換進來的狼型精英失敗 | GAME OVER |
| 任何豹型(預設精英)失敗 | GAME OVER |
| 任何熊型(Boss / 劇情敵人)失敗 | GAME OVER |

**唯一不導致 GO 的失敗**:兔子型失敗、狼型第一次失敗

### 為什麼這種隱性上限更優雅

- 不需要 Lives 系統或「Game Over 次數」
- **失敗代價漸進可感知**:從輕(兔子)→ 中(狼第一次)→ 重(狼累積)→ 終(豹/熊)
- **結果是有機的**——不是設計師強加的限制,而是玩家自己決策造成的後果
- **敘事邏輯一致**——兔子不會記恨、狼會記仇、豹本來就要殺你、熊是必殺局

## 六、連戰結構與追擊機制

### 戰役結構

```
[整備 1] → [連戰 1] → [追擊 1] → [整備 2] → [連戰 2] → [追擊 2] → ... → [最終連戰]
```

### 追擊階段

```
觸發:連戰結束時未解決池有狼型敵人
玩家選擇:
  追擊一隻:
    - 額外戰鬥(該精英敵人,被襲狀態)
    - 打贏 → 從未解決池移除 → 可選繼續追擊或結束
    - 打輸 → GAME OVER
  結束追擊:
    - 未追擊的敵人保留在未解決池
    - 進入整備期
    - 未解決池中的敵人會「替換下一連戰的普通敵人」
```

### 追擊期間規則

- 追擊發生在**連戰結束之後、整備期之前**
- 追擊敵人狀態 = **被襲**(它們「埋伏」等你)
- 卡牌使用照常(用該連戰殘餘資源)
- 整備期回補在所有追擊結束後才觸發
- 一次只追擊一隻,打完可選擇繼續或停止

### 未追擊敵人 → 替換下連戰

```
進入下連戰前:
  1. UI 明確告知:「⚠ N 隻精英來自上連戰」
  2. 未解決池中的狼型精英替換下連戰的普通敵人
  3. 替換從**連戰最前面**開始
  4. 關鍵敵人(豹/熊位置)保留,不被替換
```

範例:下連戰預設 `[A] [B] [C] [D] [E(熊 boss)]`

- 未解決池 1 隻狼精英 X → 下連戰變 `[X 精英] [B] [C] [D] [E]`
- 未解決池 2 隻狼精英 X、Y → 下連戰變 `[X 精英] [Y 精英] [C] [D] [E]`
- **豹型(D)與熊型(E)位置永遠保留**
- 替換永遠從連戰**前面**開始,只替換兔子型或狼型位置

### 為什麼替換從前面開始

- 玩家進入連戰立即面對失敗的代價(警示感強)
- 豹型/熊型保留位置,戰役節奏的「重要時刻」維持
- 「上連戰失敗 = 下連戰開頭就吃苦頭」邏輯直接

## 七、主冒險結構 + 戰役串接 + Hub

### 從「單戰役」到「主冒險」

P6 不是單一 30 分鐘遊戲,而是「**多戰役組成的主冒險**」。

```
[Hub 開場] → [序章戰役] → [Hub] → [戰役 2] → [Hub] → ... → [最終戰役 / 結局]
```

| 元素 | 規格 |
|------|------|
| 主冒險戰役數 | 3-6 場(體量可有多有少) |
| 主冒險總時長 | 1-2 小時 |
| 單戰役時長 | 30 分鐘(獵人模式)/ 8-15 分鐘(騎兵模式) |
| 解鎖機制 | **線性推進**(必須按順序解鎖) + **章節回顧**(已完成戰役可重打) |
| 序章 | 第一場戰役 = 教學戰役;後續重玩可主動跳過 |

### Hub 中樞場景

Hub 是戰役間的視覺化中樞,**承擔三個功能**:

| 功能 | 內容 |
|------|------|
| 敘事連結 | 視覺化戰役進度,**隨主冒險推進演化視覺與內容** |
| NPC 對話 | 少量 NPC 互動。**這個 NPC 同時是戰鬥內雙聲道的觀察聲音** |
| 紀念品展示 | 完成戰役後**事後出現**的紀念品(詳見下方紀念品系統規則) |

### Hub 演化 ≠ Hub 升級

[遊戲設計概念.md](遊戲設計概念.md) 第六章拒絕「Hub 升級系統」。本系統的「Hub 演化」**不是 Hub 升級**,差別在於:

| 機制 | 觸發方式 | 玩家投資 | 遊戲性影響 | 是否符合 OHK 美學 |
|------|---------|--------|----------|---------------|
| ✗ Hub 升級系統 | 玩家投資資源(時間 / 貨幣 / 道具) | 有 | 強化戰鬥力 / 解鎖能力 | 違反「無養成」 |
| ✓ **Hub 演化** | **完成戰役自動觸發** | **無** | **僅視覺 / 敘事變化** | 符合 |

→ Hub 演化是**單向觸發**(完成戰役 = 自動觸發),不是**雙向投資**(沒有「玩家花資源強化 Hub」)。

### 紀念品系統

紀念品是**遊戲風格的體現**,不是玩家可主動追求的目標。**核心原則**:

| 規則 | 內容 |
|------|------|
| **不是收集機制** | Hub **不顯示**未解鎖紀念品位、不顯示「???」、不顯示「N/M 已解鎖」進度條 |
| **不顯示解鎖條件** | 玩家事前**完全不知道**紀念品的存在或解鎖方式 |
| **事後呈現** | 玩家完成戰役 / 符合特定遊玩結果後,紀念品**自動出現**在 Hub |
| **純對應結果** | 紀念品只對應「**玩家剛剛實際體驗過**的事」,不引導未來行為 |
| **不影響遊戲性** | 不解鎖能力 / 不影響後續戰役 / 不改變敵人或卡牌 |

**為什麼這樣設計**:對應 [遊戲設計概念.md](遊戲設計概念.md) 第六章「紀錄不作為機制」——若 Hub 顯示「未解鎖 ???」或解鎖條件,玩家會**為收集而遊玩**,扭曲核心體驗。紀念品作為**事後的儀式性回饋**,讓玩家「啊,我達成了這個」,但不應該變成「我要去拿這個」。

### 卡組與戰役的關係

- 卡組 = **角色擅長能做到的事**,跨戰役不大幅變化
- 卡組由「**通用牌 + 角色專屬牌**」組成:
  - 通用牌:資源池卡 + 情報卡 + 試探卡 + 通用爆發 = 23 張左右
  - 角色專屬牌:角色卡 + 專屬爆發 = 5 張左右
- 每場戰役**自動重置至完整卡組**(上戰役消耗不繼承)
- 重玩可自選攜帶卡組(第一期只有預組;多角色為後續期數內容)

### 失敗與重啟邏輯(資源持續縮減模型)

P6 採用**自然代價**而非「存檔回滾」處理失敗:

| 情境 | 處理 |
|------|------|
| 戰鬥內 OHK 失敗 | 依敵人類別(兔/狼/豹/熊)差異化處理(見第五章) |
| 戰役 GAME OVER(精英 / 豹 / 熊失敗) | 玩家面對**兩選一**:重新挑戰 or 接受戰役失敗 |
| 重新挑戰 | **回到失敗連戰前的整備節點**,但卡組狀態**保持失敗時的消耗紀錄**(已用的牌不會回來) |
| 戰役失敗(玩家不重新挑戰 / 卡組耗盡) | 回到 Hub。可選擇重新挑戰整個戰役(此時**卡組完全重置**) |
| 主冒險不會「整體 GAME OVER」 | 戰役失敗只影響當前戰役,主冒險進度保留 |

### 重新挑戰的具體規則

1. **回到位置**:失敗連戰前的**整備節點**(不是戰役頭、不是失敗那場戰鬥前)
2. **卡組狀態**:**保持失敗時的消耗狀態**——已用的工具卡使用次數、用掉的爆發卡、用掉的角色卡**都不回來**
3. **整備期回補**:**不再次觸發**——回補只在玩家「**正常通過**」連戰後才會發生,失敗回退不重複給回補
4. **連戰內狀態**:**重置**——失敗時造成的狼型精英化等狀態清除,該連戰從第 1 場重新開始
5. **多次重新挑戰**:卡組會持續縮減,玩家自行判斷何時不再可行

### 為什麼這個設計符合 OHK 美學

| 原則 | 對應 |
|------|------|
| **失敗有真實代價** | ✓ 資源永久消耗,沒有「死了重來」的廉價感 |
| **對稱即誠實** | ✓ 玩家與敵人受同樣的規則限制——失敗的後果是物理性的、不可回退的 |
| **隱性失敗上限** | ✓ 卡組逐漸耗盡 → 玩家自然會在「無法繼續」前停下,不需要 Lives 計數 |
| **玩家自主性** | ✓ 玩家自己決定何時接受戰役失敗,沒有強制重來或強制放棄 |

### 邊界:何時算「無法繼續」?

「無法繼續」**完全由玩家自己判斷**。但機制上有兩個自然停損點:

- **資源不足以擊敗下一場敵人** → 玩家會自願選擇放棄該戰役
- **進入戰鬥但無卡可 commit** → 強制視為主動放棄

### 多角色架構(預留,後續期數實作)

- 不同角色 = 同一池中的不同卡組配置 + 不同角色專屬牌
- 解鎖時機:全破主冒險後(可能調整為戰役完成後)
- 內容形態:**同事件不同視角**(主冒險故事相同,但角色對話 / 觀察角度不同)
- **第一期不實作**,但資料結構應預留支援

### 支線模式(後續期數)

競技場 / 挑戰模式等支線內容**不在第一期 Scope 中**。後續期數加入,設計判準見 [戰役結構與節奏設計.md](戰役結構與節奏設計.md) 第六章。

## 八、雙模式節奏:獵人 vs 騎兵

P6 同一場戰役支持**兩種節奏並存**,由玩家當下熟練度與資源狀態決定模式。

### 兩種模式

| 模式 | 玩家狀態 | 節奏 | 戰鬥行為 | UI/敘事呈現 |
|------|---------|------|---------|-----------|
| **獵人模式** | 新手 / 資源不足 / 不熟敵人 | 慢、謹慎 | 投資情報、試探驗證、仔細推理、Place 重排多次 | 戰後校準完整、NPC 觀察豐富、道中對話可見 |
| **騎兵模式** | 熟手 / 資源充足 / 熟悉敵人 | 快、致命 | 不投情報、直接 OHK、Place 即 Lock | 戰後校準最小化、NPC 沉默、道中對話跳過 |

**這不是兩種遊戲,是同一場戰鬥的兩種展開方式**。決定權在玩家身上;**元知識的累積就是從獵人到騎兵的滑動軸**。

### P6 既有機制如何支持雙模式

| 機制 | 雙模式支持方式 |
|------|------------|
| 「猜弱點」是可選的 | 騎兵直接猜,獵人投資情報 |
| 情報卡 optional lock | 騎兵不用,獵人精算 |
| 試探卡作為「驗證猜測」 | 獵人用、騎兵不用 |
| 老手 vs 新手的設計合約 | 每個敵人模板就為兩種模式設計 |
| NPC 自適應教學 | 早期多話、後期沉默,自然從獵人 → 騎兵的過渡 |
| 視覺線索層 2 | 騎兵眼睛讀 = 不投情報 |

→ 雙模式不是新機制,**是 P6 設計的正確命名**——「理解即進度」原則的形象化。

### 雙模式 UI/呈現要求

| 必須做的 | 描述 |
|---------|------|
| 動畫速度可調 | 設置中 normal / fast,或點擊跳過動畫 |
| 校準對話可跳過 | hold 鍵跳過,或點「下一場」直接前進 |
| 道中對話可隱藏 | 第二次玩同戰役預設關閉 |
| 「結束本擊」即播放 | Place/Lock 不限時(獵人慢慢來),按下執行後全部結算一次跑完 |
| 重要劇情可跳過 | 首次必看,重玩可跳 |

| 不要做的 | 為什麼 |
|---------|------|
| 強制 modal 對話 | 任何「按確認才能繼續」的對話都會把騎兵拉下馬 |
| 強制動畫時長 | 看了 100 次的擊殺動畫不該每次都 2 秒 |
| 把教學寫進機制本身 | 教學是 UI 預設 + NPC 對話(可關),不是強制流程 |

### 連戰結構配比與雙模式

不同結構配比對應不同模式的契合度:

| 結構 | 獵人模式 | 騎兵模式 | 適用 |
|------|--------|--------|------|
| γ 三段升級(短→中→長,2 次整備) | ✓✓ 完美 | ✗ 太多停頓 | **教學戰役、首次玩** |
| δ 持續推進(無整備,持續斬殺) | ✗ 沒喘息 | ✓✓ 完美 | **重玩、騎兵模式** |
| α 直行式 / β 兩幕式 | ✓ 平衡 | △ 中段斷點打斷 | 標準戰役 / 旅程敘事 |

**關鍵**:同一場戰役、同一套機制,**結構自動隨玩家熟練度滑動**。詳見 [戰役結構與節奏設計.md](戰役結構與節奏設計.md)。

## 九、整備期與資源回補(含存檔點)

### 整備期內容

- 蒐集情報(多種準確度的選項)
- 獲得少量資源(回補工具牌使用次數)
- 規劃下一場連戰
- 戰後校準對話(見第十章)
- 小劇情段落(下節)
- **存檔點觸發**(下節)

### 整備節點的小劇情形態

整備節點 **不需要互動選項**,小劇情可採以下任一形式:

| 類型 | 內容 |
|------|------|
| 紮營對話 | NPC 與玩家的簡短對話 |
| 環境變化 | 場景轉換的視覺敘事(從森林 → 河岸 → 山徑) |
| 敵方反應延續 | 上輪敵人在「世界」中的回聲(如「遠方狼嚎」暗示精英化敵人歸來) |
| 戰利品/紀錄整理 | 玩家「翻看」戰鬥紀錄的視覺呈現 |

### 存檔點機制

| 規則 | 內容 |
|------|------|
| 存檔位置 | **僅在整備節點**(連戰內不可存檔) |
| 自動存檔 | 每次到達整備節點時自動存檔(用於暫停退出) |
| 手動暫停退出 | 整備節點時允許「存檔退出」,下次回來從此點繼續 |
| GAME OVER 處理 | **不是回滾**——詳見第七章「**失敗與重啟邏輯(資源持續縮減模型)**」 |
| 多存檔位 | 後續討論 — 目前預設單一存檔(per 戰役 / per 角色) |

**為什麼僅在整備節點存檔**:整備節點是天然的「玩家已下馬」時刻,提供暫停退出的合理切點。但**失敗的重來機制不是「回滾到存檔」**——失敗的資源消耗永久保留,符合 OHK 美學(詳見第七章)。

### 整備期回補比率

**起點建議:整備期回補的使用次數 ≈ 上一輪連戰消耗的 60-70%**

讓玩家永遠在「資源緩慢縮水」的趨勢中,但不至於崩盤。透過特定關卡或事件給予「補充包」作為奇點,創造「這輪我喘到氣了」的鬆緊節奏。

這個比例不是定論,是測試起點。實機調整時會感受到正確的數字。

### 關鍵原則

整備期能回補多少使用次數,**這個單一數字決定遊戲的全部感覺**:

- **太多** → 工具牌變「無限資源」,連戰的稀缺感消失,爆發牌變累贅
- **太少** → 玩家被迫過度依賴爆發牌,戰役後期資源枯竭,變計算機題
- **剛剛好** → 工具牌「每場可以用 2-3 張,但永遠不夠完全處理一場戰鬥」

## 十、戰後校準對話系統(含異步呈現)

### 系統目的(三個層次)

**第一層:校準直覺**——讓玩家知道這次決策大幅超出、剛好夠、還是差在哪裡。

**第二層:指向原因**——不直接說「你哪裡錯了」,而是給方向性的提示讓玩家自己連結。校準的不是答案,是讀題能力。

**第三層:補充視覺辨識的教學負擔**——特別是在玩家還沒建立起讀層 2 線索的能力時,校準對話可以當補充管道。

### 呈現方式:異步對話而非 modal

校準系統採用**雙聲道對話 + 敵方反應**的形式,不直接顯示數字。

**重要:對話採異步呈現,不暫停遊戲**:

| 場合 | 呈現方式 |
|------|--------|
| OHK 成功 | **字幕疊加擊殺動畫**,玩家邊看下一個敵人邊聽 / 讀。**不暫停**。 |
| OHK 失敗 | **強制停下**(失敗是學習的關鍵時刻,需要玩家注意) |
| 連戰結束 | 過場小劇情(可跳過) |
| 整備節點 | 完整呈現(玩家已下馬,有時間聽) |

**為什麼異步**:讓騎兵模式的玩家**不被打斷節奏**(對話飄過就好),獵人模式的玩家**自然會停下來看**——同一機制,玩家自己決定聽不聽。

→ 此機制對應 Hades 的「對話異步播放」設計,證實可行。

精確資訊(各類型次數貢獻明細、精確需求表、達標路徑、跨場統計)放在「**戰鬥紀錄**」中(玩家可主動查閱),不出現在即時校準回饋中。

### 雙聲道架構

**玩家聲音 — 決策視角**:代表玩家在反省自己的估算
- 範例:「我低估了那層護甲」「比想像的脆」

**NPC 聲音 — 觀察視角**:代表旁觀分析者,提供玩家可能沒察覺的角度
- 範例:「那個持矛的動作其實有透露,你看出來了」「護盾沒偵測到,下次要注意」

兩個聲音天然形成「你怎麼看 / 我怎麼看」的張力。

### 第三條反饋通道:敵方反應

敵方反應在結算動畫中即時呈現,獨立於對話之外:

| 結果 | 反應 | 設計用意 |
|------|------|---------|
| 弱點被讀對的 OHK | 短暫驚呼或憤怒——「怎麼知道那裡」 | 讓「精準擊殺」與「蠻力擊殺」感受不同 |
| Underkill,差一點 | 緊繃、沉默 | 玩家感覺「就差一點」 |
| Underkill,差很多 | 反而輕鬆,甚至嘲諷 | 敵人越輕鬆,玩家越知道自己低估了多少 |
| 大幅 Overkill | 來不及反應,直接崩解 | 「沒有反應」本身就是壓倒性的回饋 |

### 6 種對話狀態

| 結果區間 | 對話基調 | 指向的學習 |
|---------|---------|-----------|
| 大幅 Overkill | 輕鬆、甚至有點浪費 | 「這隻可以省資源」 |
| 精準擊殺 | 滿足、穩定 | 「這個判斷記住」 |
| 勉強過 | 緊繃、鬆一口氣 | 「這次很險,剛好夠」 |
| 差一步(類型錯) | 有點困惑、重新分析 | 「弱點沒匹配到」 |
| 差一步(量不足) | 直接的不夠感 | 「資源再多一點就過了」 |
| 大幅不足 | 明確失策 | 「方向根本錯了」 |

### 發言順序

採用**統一規則**:
- **敵方反應**:永遠是結算動畫的一部分
- **玩家 + NPC 對話**:永遠在動畫結束後

### 對話量策略

採用**「核心儀式化短語 + 模板識別性長對話」混合策略**:
- **核心 6 句**:每種對話狀態 1 句經典固定台詞,跨敵人共用(類似 Dark Souls 的「YOU DIED」)
- **每模板 1-2 句**:每個敵人模板的專屬識別性長對話,只在「精準擊殺」與「大幅不足」這兩個感受最強的狀態觸發

目標總量約 40-50 句(可手寫,不需動態生成)。

### 自適應教學

NPC 教學深度依**連戰位置**調整:
- 早期(首輪連戰):直接點出視覺線索
- 中期:給方向性提示
- 最終連戰:幾乎沉默,讓壓迫感自己說話

**框架詮釋**:不是難度調整,是「角色觀察玩家成長」。

## 十一、「猜弱點」機制

讓情報變成「可選工具而非必要工具」的關鍵設計。

### 核心邏輯

弱點是**客觀存在的事實**,揭露與否只影響玩家的「知曉」,不影響弱點本身。

### 玩家的選擇

- **A. 用情報換確定性**:花情報牌揭露弱點,下一場確定打中
- **B. 用元知識+直覺賭**:根據敵人外觀、行為線索,直接猜測攻擊類型

### 結算邏輯

- 出卡正確匹配弱點 → OHK 成立(無關玩家是否知道)
- 出卡未匹配弱點 → 失敗

### 情報的時間性

情報牌揭露的效果**延伸到下一場(甚至跨數場)的目標**,而不是為當前敵人鋪陳。這意味著:
- 情報是「投資」,不是「鋪陳」
- 玩家在 commit 時的決策變成「我這擊要不要塞情報牌?我這場可能因此打不贏,但下一場會輕鬆很多」
- 這是用當下的成功率買未來的確定性

### 為什麼這個機制好

- 情報變成可選工具而非必要工具
- 「勇敢的玩家」可以一直賭,「謹慎的玩家」可以一直查
- 兩種玩法都成立,定義玩家性格
- 完美對應 OHK 美學:工具不是必需品

### 設計責任

這個機制成立的前提是**敵人外觀必須提供足夠視覺線索**。盔甲類型暗示防禦特性、姿態暗示弱點位置、配色暗示屬性傾向。美術設計與遊戲機制要緊密配合。

---

## 十二、呈現層原則

> **系統內部所有運算都是精確的、確定的,但呈現給玩家的資訊應該是定性的、層級化的。玩家做的是判斷,不是計算。**

### 核心轉變:誰是計算者

- **傳統卡牌遊戲**:玩家是計算者,系統只負責驗證。玩家在腦中跑試算表(「3+2+1=6,需求 5, 過了」)——遊戲變成數學題。
- **P6 的選擇**:**系統是計算者,玩家是判斷者**。玩家用類型概念思考(「這敵人看起來怕衝擊,我有兩張衝擊就夠了——但要不要再加一張保險?」)。

### 具體呈現方式

**卡牌的呈現**(類型+次數):
- 顯示:類型(衝擊類)、次數貢獻(★★ = +2 衝擊)、定性效果描述
- 不顯示:精確修飾邏輯與系統內部機制細節

**敵人的呈現**:
- 顯示:強度層級(中等強度的目標)、外觀視覺線索、行為訊號
- 不顯示(預設):各類型的精確需求次數

**情報牌的功能轉變**:
情報牌的價值在於**揭露需求表**——「把弱點範圍縮窄為具體類型」「把模糊強度轉換為精確需求次數」。情報投資的價值在於從定性轉換為定量。

### 為什麼這個選擇重要

- **避免遊戲退化成試算表**
- **元知識的累積感更明顯**
- **失敗的學習效果更好**
- **「猜弱點」機制更有意義**

### 風險與應對

- **新手學習門檻較高** → 戰後對話提供定性回饋,教學期可顯性化次數
- **平衡負擔轉移到設計者** → 需要更多 playtesting
- **某些玩家會反感** → 這是強形狀設計的代價,會篩選出對的玩家

## 十三、教學期 UI 引導策略

P6 採用「UI 解法」處理「開局太多卡」問題,不採用「教學戰役卡池與主戰役卡池分離」的內容方案。

### 與雙模式節奏的關係

教學期 UI = **獵人模式的 UI 子集**——不是獨立系統,是雙模式中「獵人」端的明確化。對應的進階模式 UI(無灰階、無提示、純機制呈現) = 騎兵模式的呈現。玩家從教學期過渡到主戰役,實際上是從獵人模式 UI 滑動到騎兵模式 UI。

### 雙模式 UI 切換的解鎖時機

**騎兵模式 UI 不是「跳過教學」**,是「**教學已內化為玩家直覺**」。為了符合「教學設計很重要」原則,**新玩家無法直接跳到騎兵模式 UI**——此設置只在玩家**體驗完整教學週期**後才開放:

| 條件 | 解鎖內容 |
|------|--------|
| 玩家**完成第一場戰役**(序章) | 解鎖**精簡 UI 設置**選項 |
| 此後玩家可在設置中**手動切換** | 教學提示 / 灰階卡 / 道中對話顯示 / 動畫速度 |
| 第二場戰役起 | 系統**預設關閉部分教學提示**(序章已經完整提供過教學) |
| 從未通關序章的玩家 | **沒有切換選項**——強制獵人模式 UI |

→ **這個解鎖機制本身對應「理解即進度」原則**:玩家只在「**完成過完整教學**」後才獲得「**選擇精簡介面**」的資格。教學沒被跳過,只是內化了。

### A 類 vs B 類 UI 區分

| 路線 | 描述 | 是否符合 P6 原則 |
|------|------|---------------|
| **A 類:被動工具** | UI 提供整理工具,玩家自己決定怎麼用 | ✓ 系統幫玩家整理已知資訊 |
| **B 類:主動推薦** | UI 分析當下情境,系統替玩家建議 | ✗ 系統替玩家做判斷(必須拒絕) |

### 區分原則

> **系統可以整理資訊,但不能做判斷。**

### A 類 UI 工具集

| 工具 | 功能 |
|------|------|
| 類型篩選按鈕 | 衝擊 / 穿刺 / 燃燒 / 通用 / 情報 切換顯示 |
| 玩家自定義釘選 | 把常用卡釘到前列 |
| 本擊建構區與庫存區分離 | 拖拉到「本擊」槽 |
| Place vs Lock 兩區設計 | 對應 Place/Lock 機制 |
| flexible 分配滑桿 | flexible 卡 lock 時跳出分配介面 |
| 緊湊/展開切換 | 降低初始視覺密度 |
| 本擊範本儲存 | 老手提速 |

### 準 B 類中間方案(採用)

**基於玩家已揭露資訊的排序輔助**:如果玩家花了情報牌揭露「弱點是衝擊」,UI 把衝擊類卡推到前列。

**為什麼合規**:這個資訊已經是玩家知道的(他剛揭露的),UI 只是「幫他不要忘記」,不是「替他發現」。

### 教學期 UI 採用「強引導」路線

教學期應該強烈引導,但這個強引導的存在感本身要明顯到玩家知道「啊我在被引導,等我熟了就關掉」。

| 戰鬥 | UI 預設 | 配套 |
|------|--------|------|
| 教學戰 1(哨兵) | 灰階所有不相關卡,只剩 6 張(衝擊+穿刺各 3 張) | NPC 對話「先看這幾張,想想哪一張對應它的弱點」 |
| 教學戰 2-3 | 逐步解鎖 UI 工具(篩選按鈕、釘選、範本) | 玩家學會「自己整理資訊」的技能 |
| 主戰役起 | UI 完全開放,所有工具可用 | 玩家自由運用 |

### 跟 B 類的差異

教學期 UI 的「相關判斷」不是執行時根據敵人推測,而是**設計者預先指定**——就像教科書第一頁刻意只放 3 個例子。設計者在做課程設計,不是執行時判斷。

## 十四、戰鬥之間的銜接

### 為什麼需要

單次 commit 結構下,每場戰鬥獨立。但連戰必須在戰鬥之間有銜接,否則只是「五場獨立戰鬥」而非「真正的戰役」。

### 可選銜接機制

- **資源延續**:前一場剩下的某些效果可延續到下一場開頭
- **連殺累積**:連續 OHK 累積某種狀態,影響下一場
- **敵人連動**:某些敵人是前一場的「同伴」,前一場的擊殺方式影響它們的初始狀態
- **資訊共享**:前一場揭露的某些情報屬於該類型,下一場遇到同類型可套用

### 設計建議

不一定全部要做,**但至少選一個**,否則連戰只是表面的,骨子裡還是獨立戰鬥的拼接。

## 十五、跳過機制(原型未實作)

### 實作方式

做成有限次數的卡片(例如「迴避」、「轉進」),不是隨時可用的選項。

### 為什麼用卡片形式

- 跳過有機會成本(這張卡用了就沒)
- 避免養成「遇到不確定就跳」的逃避習慣
- 跳過本身可以是有趣的設計題

### 跳過卡的設計變化

- 一張只能跳當下
- 一張可以跳隊列中任何一個
- 一張跳過會讓敵人變強
- 一張跳過後該敵人在最終戰會出現

### 重要設計原則

**最終連戰中無法跳過,前期連戰可以跳過但有累積代價**——把跳過變成「拖到後面再算總帳」的策略選擇,而不是「逃避困難」的取巧。

## 十六、視覺語言要求

由於玩家可以「直接猜攻擊類型」,敵人的外觀必須提供足夠提示。

### 必須要做

- 同類敵人有共同視覺特徵
- 弱點類型可從外觀推論
- 戰鬥狀態(先攻 / 被襲)有視覺暗示

### 美術與機制的綁定

這不只是美學問題——遊戲機制本身依賴視覺溝通。**美術設計與機制設計必須同步進行**,不能分開做。

### 擊殺瞬間的儀式化

OHK 沒有經驗值或裝備獎勵,**「這一擊本身」就是獎勵**。如果擊殺感平淡,整個遊戲的核心體驗就垮了一半。

擊殺瞬間應該包含:
- 視覺強化(慢動作、特殊效果)
- 音效設計(明確的擊殺音)
- 節奏停頓(讓玩家「看見」勝利的瞬間)
- 不能讓玩家投牌後立刻看到結果——要讓他停頓一秒,看著各類型需求 bar 被填充逼近達標線,然後才崩解
