## 一、戰鬥三子階段:Place / Lock / 結束本擊

每場戰鬥是「**一個 commit moment**」:玩家從手上挑選若干張卡組成「本擊」,一次提交,所有效果同時結算。

### 「本擊」的定義

本擊 = 玩家在單場戰鬥中,從手上挑出並一次性提交的卡牌組合。本擊是設計上的最小單位,不是個別卡牌:

- 揭露效果依「本擊」內的卡綜合判定
- 結算結果依「本擊」整體類型次數決定
- 玩家的決策對象是「本擊」而非「個別卡」

### 卡牌的 lock_class 三分類

每張卡的 `lock_class` 決定它如何與本機制互動:

| lock_class | 行為 | 卡面標示 | 適用卡類 |
|-----------|------|---------|---------|
| **`"none"`** | Lock 與否完全不影響效果 | 無 lock 相關標示 | 純戰鬥卡(資源池、純爆發、純角色)= 19 張 |
| **`"optional"`** | Lock 與否**改變揭露對象**:不 lock = 廣度;Lock = 深度 | 卡面寫兩種模式效果 | 情報卡(偵察 / 估量 / 警示)= 3 張 |
| **`"required"`** | **必須先 lock 才能生效** | 卡面標 🔒 強制 lock | 試探卡 / flexible 卡 / 全知之眼 / 洞察一擊 = 6 張 |

### 三個子階段

1. **Place 階段**:玩家拖卡到本擊區,可隨時編輯。系統不結算任何效果,不消耗資源。flexible 卡顯示「未分配」狀態。

2. **動態 Lock 階段**:玩家對單張卡點「鎖定」:
   - **`"none"` 卡**:可 lock 也可不 lock,效果不變
   - **`"optional"` 卡**:lock 觸發 locked 模式;不 lock 則於「結束本擊」觸發 unlocked 模式
   - **`"required"` 卡**:**必須在此階段 lock**(否則「結束本擊」按鈕禁用)。lock 時 special_effect 立即觸發,flexible 必須完成分配
   
   Lock 不可撤回。Lock 過程中玩家可繼續新增 placed 卡或 lock 其他卡。

3. **結束本擊**:玩家點「結束本擊」按鈕(若有 required 卡未 lock 則禁用),系統鎖定所有未 lock 的 a/b 類 placed 卡,然後在戰鬥層整體計算各類型次數總和,逐項比對敵人需求表;**任一條件達標即 OHK**。
   - 期間若 b 類卡未 lock,觸發其 unlocked 模式 special_effect

### 結算範例

敵人需求表:衝擊 2 / 穿刺 5 / 燃燒 6 / 混合任意 7

玩家本擊流程:

```
1. [Place] 玩家置入:洞察一擊、重斬、直擊
   本擊狀態:衝擊 0(尚未分配 + 未結算)

2. [Lock 洞察一擊]
   → 觸發 special_effect:揭露當前敵人弱點 = 衝擊
   → 洞察一擊資源消耗(角色卡用掉)
   → 玩家現在知道弱點

3. [玩家分配洞察一擊的 flexible]
   洞察一擊 +2 → 分配 +2 衝擊

4. [結束本擊]
   → 鎖定剩餘:重斬、直擊
   → 戰鬥層整體計算:重斬 +2、直擊 +1、洞察 +2 = 衝擊 5
   → 衝擊計數 = 5(需求 2)→ ✓ 達標
   → 結果:OHK
```

### Lock 順序的策略意義

Lock 順序本身是判斷的一部分:
- 先 lock 揭露卡 → 獲取資訊優勢 → 後續決策更精準
- 先 lock 戰鬥卡 → 喪失資訊優勢,但表達「我已決定」的清晰承諾

設計上不限制順序,讓玩家自己經營「資訊→決策」的節奏。

### 設計含義

Place/Lock 機制讓「揭露」與「決策」之間有自然空隙——玩家做出**多個子動作**,但這些子動作仍構成「**一個本擊**」。這跟「分次戰鬥」是兩回事——玩家是在同一 commit 階段內讓資訊流動,不是發起多次 commit。對應 OHK 美學的「物理動作鏈」精神:一旦動作開始(lock)就不可收回,但其他動作可以基於資訊調整。

## 二、類型計數結算模型

### 為什麼這個模型

P6 採用「**單一檢查原則**」——每場戰鬥的勝敗判定收斂到一個結算瞬間,玩家在 commit 前面對的是「選哪條路」的單一決策,而非「同時滿足多個獨立條件」。

### 具體機制

- 每個敵人有一張**需求表**:每個傷害類型需要多少「次數」才能擊殺
- 加上一個「**混合任意需求**」:不限類型只比總次數
- 玩家本擊只要在任一條件達標就 OHK
- 所有條件指向同一個結算瞬間

### 結算範例

敵人:衝擊需求 2 / 穿刺需求 5 / 燃燒需求 6 / 混合需求 7

- 玩家 commit 2 張衝擊卡 → 衝擊計數 2 ≥ 2 → ✓ OHK(最省)
- 玩家 commit 5 張穿刺卡 → 穿刺計數 5 ≥ 5 → ✓ OHK(較貴)
- 玩家 commit 7 張任意混合卡 → 混合計數 7 ≥ 7 → ✓ OHK(很貴,「不知弱點時的保險路徑」)
- 玩家 commit 1 衝擊 + 3 雜類 → 各條件都不達 → ✗ Underkill

### 為什麼這個解法最好

- 簡單(玩家不需算乘法,純加總比對)
- 「對應弱點」價值清晰可見(衝擊只要 2 vs 穿刺要 5 = 弱點顯而易見)
- **勇敢直打仍可行**(走混合任意路徑,代價高但能成功)
- **謹慎優化也可行**(揭露弱點 → 走最低需求路徑)
- 沒有「對不起,你必須有衝擊類卡才能繼續」的鎖鑰問題
- 結算瞬間直觀:玩家看到「填滿哪個需求 bar」就立即知道結果

### 平衡原則:混合任意需求 ≈ 最強弱點需求 × 3-4

目前原型採用「**混合 = 最低需求 × 4**」,但對中低需求敵人此倍率可能過低(待第三次紙上測試後驗證);提案改為「混合 = 各類型專項需求總和 × 1.7」。詳見 [敵人設計原則.md](敵人設計原則.md) 第六章。

### 規則應用範例

未來若想加入「敵人有護盾,先破護盾再攻擊」這種機制,要先檢查是否違反原則:
- ❌ 兩個獨立階段(先破盾才能攻擊)→ 違反
- ✓ 護盾把某類型需求 +N 次 → 符合(單一檢查,護盾是調整需求,不是新階段)

## 三、敵人四層資訊架構

每個敵人的資訊以四層結構組織。玩家對敵人的「了解程度」由各層的揭露狀態決定。

### 層 1 — 模板永久顯性

**立即可見**,定義敵人的「大類」:

- 大類識別(輕甲偵察 / 重甲近戰 / 等)
- 敵人類別(兔子 / 狼 / 豹 / 熊)
- 弱點範圍(可單獨擊殺的類型)
- 弱點需求層級範圍(低/中/高)
- 戰鬥狀態傾向

### 層 2 — 次級視覺線索

**老手用眼讀**,需要玩家學習視覺語言:

- 持械暗示弱點類型(持矛 → 穿刺)
- 鎧甲狀態暗示需求層級(殘破皮甲 → 低)
- 姿態暗示戰鬥狀態(蹲伏潛行 → 被襲)
- 物種特徵暗示敵人類別

> 完整視覺線索對應表見 [敵人設計原則.md](敵人設計原則.md) 第四章。

### 層 3 — 即時狀態

**進入戰鬥可見**:

- 戰鬥狀態標籤(本場是標準/先攻/被襲)
- 上一場用情報牌揭露的相關資訊(跨場持續)
- 若為精英化的狼:UI 標籤「精英化 ⚠」+ 戰鬥紀錄可調出上次資料

### 層 4 — 隱藏狀態

**僅情報牌可揭**:

- 完整需求表(各類型精確次數)
- 混合任意需求
- 敵人類別(可透過警示卡揭露)

### 「看得懂 vs 看不懂」的張力

老手讀層 2 → 不投資情報 → 仍能對應弱點擊殺
新手讀不出層 2 → 必須投資情報 → 或盲打走混合任意路徑

這是元知識的核心舞台——P6 的「進度」就是讀層 2 能力的累積。

## 四、戰鬥狀態系統:先攻 / 標準 / 被襲

每場戰鬥開始時的處境本身是個戰術變數。

### 三種戰鬥狀態

| 狀態 | 效果 | 適用 | 設計感 |
|------|------|------|-------|
| **先攻** | 玩家獲得免費的前置揭露(一張情報牌等價,不消耗手牌) | 連戰第一個敵人、易偵察的敵人 | 玩家有準備、佔上風 |
| **標準** | 直接 commit | 大部分敵人 | 基本對峙 |
| **被襲** | 部分情報效果無效,或必須投入更多卡牌 | 埋伏者、刺客類敵人 | 措手不及 |

### 為什麼這個分類有用

讓敵人類型不只有「需求表多少」這個維度,還有「如何進入戰鬥」這個維度,敵人個性立刻豐富。

### 待決定的設計問題

- 戰鬥狀態是依敵人類型固定?
- 還是依玩家某些選擇影響?
- 或是隨機(input randomness 意義上)?

目前原型階段:由敵人模板的「戰鬥狀態傾向」決定 instance 機率分布。

## 五、失敗代價與精英機制(四類敵人差異化)

> 完整機制取代了舊版「10% 隨機精英化」設計(已棄用)。

### 核心邏輯

> **並非所有敵人都會殺你——但會記恨的敵人會以精英形態回歸,而預設精英與 Boss 失敗即終戰。**

失敗代價**依敵人類別**(兔子 / 狼 / 豹 / 熊)差異化。

### 敵人四類分類

| 類別 | 範例 | 失敗代價 | 敘事邏輯 |
|------|------|---------|---------|
| **A. 兔子(無威脅普通)** | 哨兵 | 卡牌消耗;敵人逃走,**不再回來** | 「沒有殺意」的對手 |
| **B. 狼(威脅普通)** | 巡守、重甲衛兵 | 卡牌消耗 + 變精英回歸 + 精英失敗 = GO | 「會記恨並學習」的對手 |
| **C. 豹(預設精英)** | 伏擊者 | **失敗即 GAME OVER** | 「本來就要殺你」的對手 |
| **D. 熊(Boss / 劇情敵人)** | 領主 | **失敗即 GAME OVER** | 戰役節奏的關鍵 |

### 狼型三層失敗代價

**第一層 — 連戰內第 1 次失敗**

```
1. 投入的卡牌依然消耗
2. 該敵人標記為「精英狀態」
3. 排到該連戰末段(熊型 Boss 之前)
4. 連戰場次 +1
玩家會在該連戰結束前再次面對它(精英化)
```

**第二層 — 連戰內第 2+ 次失敗**

```
1. 投入的卡牌依然消耗
2. 該敵人「移出該連戰」進入「未解決池」
3. 連戰繼續進行
連戰結束後玩家會面對「追擊選擇」
```

**第三層 — 連戰結束 → 追擊機制**(詳見第六章)

### 精英敵人的特性

- **外觀數值與原本完全相同**,只有 UI 標籤「精英化 ⚠」標示
- 失敗 = GAME OVER
- 戰鬥紀錄可調出上次面對的資料

### 精英敵人的兩種來源

| 來源 | 範例 | 視覺 | 玩家心理 |
|------|------|------|---------|
| **狼變的精英**(失敗造成) | 失敗的巡守、失敗的重甲衛兵 | 跟原本一樣 + UI 標籤 | 「我記得它,我有戰鬥紀錄資料」 |
| **豹型預設精英**(本來就在) | 伏擊者 | 物種本身的精英級視覺 | 「我第一次遇到,但它一招斃命」 |

### Game Over 條件

任何精英敵人 + 豹型 + 熊型失敗 = GAME OVER:

| 情境 | 失敗結果 |
|------|---------|
| 連戰中面對狼型精英失敗 | GAME OVER |
| 追擊狼型時失敗 | GAME OVER |
| 下連戰面對替換進來的狼型精英失敗 | GAME OVER |
| 任何豹型(預設精英)失敗 | GAME OVER |
| 任何熊型(Boss / 劇情敵人)失敗 | GAME OVER |

**唯一不導致 GO 的失敗**:兔子型失敗、狼型第一次失敗

### 為什麼這種隱性上限更優雅

- 不需要 Lives 系統或「Game Over 次數」
- **失敗代價漸進可感知**:從輕(兔子)→ 中(狼第一次)→ 重(狼累積)→ 終(豹/熊)
- **結果是有機的**——不是設計師強加的限制,而是玩家自己決策造成的後果
- **敘事邏輯一致**——兔子不會記恨、狼會記仇、豹本來就要殺你、熊是必殺局

## 六、連戰結構與追擊機制

### 戰役結構

```
[整備 1] → [連戰 1] → [追擊 1] → [整備 2] → [連戰 2] → [追擊 2] → ... → [最終連戰]
```

### 追擊階段

```
觸發:連戰結束時未解決池有狼型敵人
玩家選擇:
  追擊一隻:
    - 額外戰鬥(該精英敵人,被襲狀態)
    - 打贏 → 從未解決池移除 → 可選繼續追擊或結束
    - 打輸 → GAME OVER
  結束追擊:
    - 未追擊的敵人保留在未解決池
    - 進入整備期
    - 未解決池中的敵人會「替換下一連戰的普通敵人」
```

### 追擊期間規則

- 追擊發生在**連戰結束之後、整備期之前**
- 追擊敵人狀態 = **被襲**(它們「埋伏」等你)
- 卡牌使用照常(用該連戰殘餘資源)
- 整備期回補在所有追擊結束後才觸發
- 一次只追擊一隻,打完可選擇繼續或停止

### 未追擊敵人 → 替換下連戰

```
進入下連戰前:
  1. UI 明確告知:「⚠ N 隻精英來自上連戰」
  2. 未解決池中的狼型精英替換下連戰的普通敵人
  3. 替換從**連戰最前面**開始
  4. 關鍵敵人(豹/熊位置)保留,不被替換
```

範例:下連戰預設 `[A] [B] [C] [D] [E(熊 boss)]`

- 未解決池 1 隻狼精英 X → 下連戰變 `[X 精英] [B] [C] [D] [E]`
- 未解決池 2 隻狼精英 X、Y → 下連戰變 `[X 精英] [Y 精英] [C] [D] [E]`
- **豹型(D)與熊型(E)位置永遠保留**
- 替換永遠從連戰**前面**開始,只替換兔子型或狼型位置

### 為什麼替換從前面開始

- 玩家進入連戰立即面對失敗的代價(警示感強)
- 豹型/熊型保留位置,戰役節奏的「重要時刻」維持
- 「上連戰失敗 = 下連戰開頭就吃苦頭」邏輯直接

## 七、整備期與資源回補

### 整備期內容

- 蒐集情報(多種準確度的選項)
- 獲得少量資源(回補工具牌使用次數)
- 規劃下一場連戰
- 戰後校準對話(見第八章)

### 整備期回補比率

**起點建議:整備期回補的使用次數 ≈ 上一輪連戰消耗的 60-70%**

讓玩家永遠在「資源緩慢縮水」的趨勢中,但不至於崩盤。透過特定關卡或事件給予「補充包」作為奇點,創造「這輪我喘到氣了」的鬆緊節奏。

這個比例不是定論,是測試起點。實機調整時會感受到正確的數字。

### 關鍵原則

整備期能回補多少使用次數,**這個單一數字決定遊戲的全部感覺**:

- **太多** → 工具牌變「無限資源」,連戰的稀缺感消失,爆發牌變累贅
- **太少** → 玩家被迫過度依賴爆發牌,戰役後期資源枯竭,變計算機題
- **剛剛好** → 工具牌「每場可以用 2-3 張,但永遠不夠完全處理一場戰鬥」

## 八、戰後校準對話系統

### 系統目的(三個層次)

**第一層:校準直覺**——讓玩家知道這次決策大幅超出、剛好夠、還是差在哪裡。

**第二層:指向原因**——不直接說「你哪裡錯了」,而是給方向性的提示讓玩家自己連結。校準的不是答案,是讀題能力。

**第三層:補充視覺辨識的教學負擔**——特別是在玩家還沒建立起讀層 2 線索的能力時,校準對話可以當補充管道。

### 呈現方式:對話而非數值

校準系統採用**雙聲道對話 + 敵方反應**的形式,不直接顯示數字。

精確資訊(各類型次數貢獻明細、精確需求表、達標路徑、跨場統計)放在「**戰鬥紀錄**」中(玩家可主動查閱),不出現在即時校準回饋中。

### 雙聲道架構

**玩家聲音 — 決策視角**:代表玩家在反省自己的估算
- 範例:「我低估了那層護甲」「比想像的脆」

**NPC 聲音 — 觀察視角**:代表旁觀分析者,提供玩家可能沒察覺的角度
- 範例:「那個持矛的動作其實有透露,你看出來了」「護盾沒偵測到,下次要注意」

兩個聲音天然形成「你怎麼看 / 我怎麼看」的張力。

### 第三條反饋通道:敵方反應

敵方反應在結算動畫中即時呈現,獨立於對話之外:

| 結果 | 反應 | 設計用意 |
|------|------|---------|
| 弱點被讀對的 OHK | 短暫驚呼或憤怒——「怎麼知道那裡」 | 讓「精準擊殺」與「蠻力擊殺」感受不同 |
| Underkill,差一點 | 緊繃、沉默 | 玩家感覺「就差一點」 |
| Underkill,差很多 | 反而輕鬆,甚至嘲諷 | 敵人越輕鬆,玩家越知道自己低估了多少 |
| 大幅 Overkill | 來不及反應,直接崩解 | 「沒有反應」本身就是壓倒性的回饋 |

### 6 種對話狀態

| 結果區間 | 對話基調 | 指向的學習 |
|---------|---------|-----------|
| 大幅 Overkill | 輕鬆、甚至有點浪費 | 「這隻可以省資源」 |
| 精準擊殺 | 滿足、穩定 | 「這個判斷記住」 |
| 勉強過 | 緊繃、鬆一口氣 | 「這次很險,剛好夠」 |
| 差一步(類型錯) | 有點困惑、重新分析 | 「弱點沒匹配到」 |
| 差一步(量不足) | 直接的不夠感 | 「資源再多一點就過了」 |
| 大幅不足 | 明確失策 | 「方向根本錯了」 |

### 發言順序

採用**統一規則**:
- **敵方反應**:永遠是結算動畫的一部分
- **玩家 + NPC 對話**:永遠在動畫結束後

### 對話量策略

採用**「核心儀式化短語 + 模板識別性長對話」混合策略**:
- **核心 6 句**:每種對話狀態 1 句經典固定台詞,跨敵人共用(類似 Dark Souls 的「YOU DIED」)
- **每模板 1-2 句**:每個敵人模板的專屬識別性長對話,只在「精準擊殺」與「大幅不足」這兩個感受最強的狀態觸發

目標總量約 40-50 句(可手寫,不需動態生成)。

### 自適應教學

NPC 教學深度依**連戰位置**調整:
- 早期(首輪連戰):直接點出視覺線索
- 中期:給方向性提示
- 最終連戰:幾乎沉默,讓壓迫感自己說話

**框架詮釋**:不是難度調整,是「角色觀察玩家成長」。

## 九、「猜弱點」機制

讓情報變成「可選工具而非必要工具」的關鍵設計。

### 核心邏輯

弱點是**客觀存在的事實**,揭露與否只影響玩家的「知曉」,不影響弱點本身。

### 玩家的選擇

- **A. 用情報換確定性**:花情報牌揭露弱點,下一場確定打中
- **B. 用元知識+直覺賭**:根據敵人外觀、行為線索,直接猜測攻擊類型

### 結算邏輯

- 出卡正確匹配弱點 → OHK 成立(無關玩家是否知道)
- 出卡未匹配弱點 → 失敗

### 情報的時間性

情報牌揭露的效果**延伸到下一場(甚至跨數場)的目標**,而不是為當前敵人鋪陳。這意味著:
- 情報是「投資」,不是「鋪陳」
- 玩家在 commit 時的決策變成「我這擊要不要塞情報牌?我這場可能因此打不贏,但下一場會輕鬆很多」
- 這是用當下的成功率買未來的確定性

### 為什麼這個機制好

- 情報變成可選工具而非必要工具
- 「勇敢的玩家」可以一直賭,「謹慎的玩家」可以一直查
- 兩種玩法都成立,定義玩家性格
- 完美對應 OHK 美學:工具不是必需品

### 設計責任

這個機制成立的前提是**敵人外觀必須提供足夠視覺線索**。盔甲類型暗示防禦特性、姿態暗示弱點位置、配色暗示屬性傾向。美術設計與遊戲機制要緊密配合。

---

## 十、呈現層原則

> **系統內部所有運算都是精確的、確定的,但呈現給玩家的資訊應該是定性的、層級化的。玩家做的是判斷,不是計算。**

### 核心轉變:誰是計算者

- **傳統卡牌遊戲**:玩家是計算者,系統只負責驗證。玩家在腦中跑試算表(「3+2+1=6,需求 5, 過了」)——遊戲變成數學題。
- **P6 的選擇**:**系統是計算者,玩家是判斷者**。玩家用類型概念思考(「這敵人看起來怕衝擊,我有兩張衝擊就夠了——但要不要再加一張保險?」)。

### 具體呈現方式

**卡牌的呈現**(類型+次數):
- 顯示:類型(衝擊類)、次數貢獻(★★ = +2 衝擊)、定性效果描述
- 不顯示:精確修飾邏輯與系統內部機制細節

**敵人的呈現**:
- 顯示:強度層級(中等強度的目標)、外觀視覺線索、行為訊號
- 不顯示(預設):各類型的精確需求次數

**情報牌的功能轉變**:
情報牌的價值在於**揭露需求表**——「把弱點範圍縮窄為具體類型」「把模糊強度轉換為精確需求次數」。情報投資的價值在於從定性轉換為定量。

### 為什麼這個選擇重要

- **避免遊戲退化成試算表**
- **元知識的累積感更明顯**
- **失敗的學習效果更好**
- **「猜弱點」機制更有意義**

### 風險與應對

- **新手學習門檻較高** → 戰後對話提供定性回饋,教學期可顯性化次數
- **平衡負擔轉移到設計者** → 需要更多 playtesting
- **某些玩家會反感** → 這是強形狀設計的代價,會篩選出對的玩家

## 十一、教學期 UI 引導策略

P6 採用「UI 解法」處理「開局太多卡」問題,不採用「教學戰役卡池與主戰役卡池分離」的內容方案。

### A 類 vs B 類 UI 區分

| 路線 | 描述 | 是否符合 P6 原則 |
|------|------|---------------|
| **A 類:被動工具** | UI 提供整理工具,玩家自己決定怎麼用 | ✓ 系統幫玩家整理已知資訊 |
| **B 類:主動推薦** | UI 分析當下情境,系統替玩家建議 | ✗ 系統替玩家做判斷(必須拒絕) |

### 區分原則

> **系統可以整理資訊,但不能做判斷。**

### A 類 UI 工具集

| 工具 | 功能 |
|------|------|
| 類型篩選按鈕 | 衝擊 / 穿刺 / 燃燒 / 通用 / 情報 切換顯示 |
| 玩家自定義釘選 | 把常用卡釘到前列 |
| 本擊建構區與庫存區分離 | 拖拉到「本擊」槽 |
| Place vs Lock 兩區設計 | 對應 Place/Lock 機制 |
| flexible 分配滑桿 | flexible 卡 lock 時跳出分配介面 |
| 緊湊/展開切換 | 降低初始視覺密度 |
| 本擊範本儲存 | 老手提速 |

### 準 B 類中間方案(採用)

**基於玩家已揭露資訊的排序輔助**:如果玩家花了情報牌揭露「弱點是衝擊」,UI 把衝擊類卡推到前列。

**為什麼合規**:這個資訊已經是玩家知道的(他剛揭露的),UI 只是「幫他不要忘記」,不是「替他發現」。

### 教學期 UI 採用「強引導」路線

教學期應該強烈引導,但這個強引導的存在感本身要明顯到玩家知道「啊我在被引導,等我熟了就關掉」。

| 戰鬥 | UI 預設 | 配套 |
|------|--------|------|
| 教學戰 1(哨兵) | 灰階所有不相關卡,只剩 6 張(衝擊+穿刺各 3 張) | NPC 對話「先看這幾張,想想哪一張對應它的弱點」 |
| 教學戰 2-3 | 逐步解鎖 UI 工具(篩選按鈕、釘選、範本) | 玩家學會「自己整理資訊」的技能 |
| 主戰役起 | UI 完全開放,所有工具可用 | 玩家自由運用 |

### 跟 B 類的差異

教學期 UI 的「相關判斷」不是執行時根據敵人推測,而是**設計者預先指定**——就像教科書第一頁刻意只放 3 個例子。設計者在做課程設計,不是執行時判斷。

## 十二、戰鬥之間的銜接

### 為什麼需要

單次 commit 結構下,每場戰鬥獨立。但連戰必須在戰鬥之間有銜接,否則只是「五場獨立戰鬥」而非「真正的戰役」。

### 可選銜接機制

- **資源延續**:前一場剩下的某些效果可延續到下一場開頭
- **連殺累積**:連續 OHK 累積某種狀態,影響下一場
- **敵人連動**:某些敵人是前一場的「同伴」,前一場的擊殺方式影響它們的初始狀態
- **資訊共享**:前一場揭露的某些情報屬於該類型,下一場遇到同類型可套用

### 設計建議

不一定全部要做,**但至少選一個**,否則連戰只是表面的,骨子裡還是獨立戰鬥的拼接。

## 十三、跳過機制(原型未實作)

### 實作方式

做成有限次數的卡片(例如「迴避」、「轉進」),不是隨時可用的選項。

### 為什麼用卡片形式

- 跳過有機會成本(這張卡用了就沒)
- 避免養成「遇到不確定就跳」的逃避習慣
- 跳過本身可以是有趣的設計題

### 跳過卡的設計變化

- 一張只能跳當下
- 一張可以跳隊列中任何一個
- 一張跳過會讓敵人變強
- 一張跳過後該敵人在最終戰會出現

### 重要設計原則

**最終連戰中無法跳過,前期連戰可以跳過但有累積代價**——把跳過變成「拖到後面再算總帳」的策略選擇,而不是「逃避困難」的取巧。

## 十四、視覺語言要求

由於玩家可以「直接猜攻擊類型」,敵人的外觀必須提供足夠提示。

### 必須要做

- 同類敵人有共同視覺特徵
- 弱點類型可從外觀推論
- 戰鬥狀態(先攻 / 被襲)有視覺暗示

### 美術與機制的綁定

這不只是美學問題——遊戲機制本身依賴視覺溝通。**美術設計與機制設計必須同步進行**,不能分開做。

### 擊殺瞬間的儀式化

OHK 沒有經驗值或裝備獎勵,**「這一擊本身」就是獎勵**。如果擊殺感平淡,整個遊戲的核心體驗就垮了一半。

擊殺瞬間應該包含:
- 視覺強化(慢動作、特殊效果)
- 音效設計(明確的擊殺音)
- 節奏停頓(讓玩家「看見」勝利的瞬間)
- 不能讓玩家投牌後立刻看到結果——要讓他停頓一秒,看著各類型需求 bar 被填充逼近達標線,然後才崩解
